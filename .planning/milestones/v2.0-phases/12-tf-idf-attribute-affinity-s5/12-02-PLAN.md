<objective>
Land the one-shot backfill job that populates the Phase-12 schema additions for the ~3500 anime rows that already exist in production. After this plan ships and make backfill-attributes runs to completion, every anime row has its kind / rating / material_source columns populated where Shikimori has data, plus anime_studios and anime_tags join rows where Shikimori and AniList respectively return data. Anime without an AniList mapping in libs/idmapping ARM persist with empty tags — consistent with the spec §3.3 missing-attribute-equals-zero contract. The Wave-3 S5 plan can then TF-IDF over a non-empty dataset.

Purpose: Wave 1 (Plan 01) added the columns and the tables but only freshly-fetched anime carry the new data. Existing rows would force S5 to score against empty attribute vectors — making it indistinguishable from a no-op. The backfill closes that gap once. After it runs, S5 has real signal across all six locked attribute dimensions.

The job is intentionally a one-shot CLI tool, NOT a cron. Reasoning per CONTEXT.md §Backfill Strategy: the work is bounded (~3500 rows), idempotent, and we don't want a long-running goroutine baked into a service. Re-runs pick up newly-imported anime that arrive after this run; the Wave-3 plan does not need to coordinate with it.

Output:
- services/maintenance/cmd/backfill-attributes/main.go — CLI entry point with flag parsing, DB connection, parser construction, and a final summary log.
- services/maintenance/cmd/backfill-attributes/backfill.go — BackfillRunner with ShikimoriHalf (kind/rating/material_source/studios) and AnilistHalf (tags). Both halves are idempotent + per-anime error-isolated.
- services/maintenance/cmd/backfill-attributes/{main_test.go,backfill_test.go} — sqlite + mock-parser tests covering idempotency, skip-already-populated, and per-anime error isolation.
- services/maintenance/go.mod — new replace + require directives for libs/database, libs/idmapping, services/catalog (for the shikimori + anilist parsers + domain types).
- Makefile — make backfill-attributes invokes the host-native binary against production env. Plus build-backfill-attributes for CI.

After this plan ships, the Task 5 checkpoint runs `make backfill-attributes` against production, captures the output (rows updated / skipped / failed), and confirms via psql that at least one anime row has non-empty material_source AND non-empty kind AND at least one anime_studios join row AND at least one anime_tags join row with rank > 0.

Backfill scope:
- Shikimori half: re-fetch every anime missing kind / rating / material_source / studios. Idempotency predicate (the WHERE clause): (kind = '' OR kind IS NULL) OR (rating = '' OR rating IS NULL) OR (material_source = '' OR material_source IS NULL) OR NOT EXISTS (SELECT 1 FROM anime_studios WHERE anime_studios.anime_id = animes.id).
- AniList half: fetch tags for every anime missing tags. Idempotency predicate: NOT EXISTS (SELECT 1 FROM anime_tags WHERE anime_tags.anime_id = animes.id) AND shikimori_id != ''.
- Soft-deleted anime (deleted_at IS NOT NULL) are SKIPPED. The backfill operates on live rows only.
- Hidden anime (hidden = true) are INCLUDED. S5 will filter them at request-time via S11; backfilling them keeps the schema consistent for admins who toggle hidden later.

Rate limits:
- Shikimori: configured via ShikimoriConfig.RateLimit (defaults to 3 rps in production config). The backfill respects whatever the live config sets — does not override.
- AniList: 1 rps (60 req/min) with 33% headroom under the 90/min cap. The default in anilist.NewClient is 1 rps; the backfill exposes --anilist-rps to bump it to 2 if the operator wants ~30-min runtime instead of ~60-min, but the default is conservative.

Failure semantics:
- Per-anime fetch failure (timeout, 5xx, JSON parse): log + continue. Do NOT abort the run.
- Per-anime DB write failure: log + continue. Each anime is its own transaction.
- DB connection failure at startup: fatal exit (non-zero status). The job needs the DB to do anything.
- Config / flag parse failure: fatal exit (non-zero).
- Run-wide failure threshold: NONE. CONTEXT.md "Claude's Discretion" item suggests an optional > 10% in any 100-row window abort guard — we choose NOT to implement it for v1. Re-running the job is cheap (idempotent) so a fail-fast guard adds complexity without value. Document this decision inline.

Output / logging:
- Structured logging via libs/logger (matches existing services).
- Progress: log every --log-every rows processed (default 100) with: phase ("shikimori" / "anilist"), processed, succeeded, skipped, failed.
- Final summary: single Infow line with the full counts: shikimori_succeeded, shikimori_skipped, shikimori_failed, anilist_succeeded, anilist_skipped_no_anilist, anilist_skipped_already_done, anilist_failed, total_duration_seconds.

Production deploy:
- The job runs against production from the deploy host (where Makefile lives + DB env is loaded from docker/.env). It connects to the postgres container via the host-mapped port (typically 5432) using the same env vars docker compose uses.
- Run order on production: (1) ensure Wave 1 is deployed (catalog redeployed with new schema); (2) run make backfill-attributes; (3) confirm sample rows populated; (4) Wave-3 plan can deploy.
- Re-runs are safe and free.

Open verification items (decided in plan-phase):
1. Shikimori re-fetch granularity: per-anime via GetAnimeByID (1 fetch per row that needs Shikimori-half work). Batch fetch (GetAnimeByIDs, max 50 per call) is faster but the WHERE clause makes it harder to know which rows to fetch in a batch. Per-anime is simpler + sufficient at 3500 rows. Shikimori at 3 rps = ~20 minutes for the Shikimori half.
2. Tag deduplication: Tag.ID is the slugified name; multiple anime sharing "Slice of Life" all map to Tag{ID: "slice_of_life"}. The backfill uses INSERT ... ON CONFLICT (id) DO NOTHING for Tag rows so re-inserts are no-ops. The same idempotency applies to Studio rows.
3. AnimeTag write strategy: composite PK (anime_id, tag_id) so the join row uniquely identifies an anime+tag pair. The backfill uses INSERT ... ON CONFLICT (anime_id, tag_id) DO UPDATE SET rank = EXCLUDED.rank — this lets a re-run with a fresher AniList payload update the rank without orphaning the row.
4. Logger choice: structured libs/logger to match the rest of the services. Errors go to Errorw, progress goes to Infow.
5. Error path: errors.Wrap or fmt.Errorf — match the existing maintenance binary's pattern (it uses fmt.Errorf, so we do too).
6. The backfill is NOT containerized; it runs as a host-native binary via go build -> bin/backfill-attributes. CLAUDE.md confirms the maintenance service uses this pattern (see make build-maintenance line 51-53).

                - tagID := anilist.SlugifyTagName(t.Name)

           DryRun        bool
           Limit         int
           SkipShikimori bool
           SkipTags      bool
           LogEvery      int
           ShikimoriRPS  int
       }

       func parseFlags() cliConfig {
           var c cliConfig
           flag.BoolVar(&c.DryRun, "dry-run", false, "log what would be written without writing")
           flag.IntVar(&c.Limit, "limit", 0, "maximum rows to process per half (0 = no limit)")
           flag.BoolVar(&c.SkipShikimori, "skip-shikimori", false, "skip the Shikimori half (kind/rating/material_source/studios)")

           flag.Parse()
           return c
       }

       func main() {
           cli := parseFlags()

           log := logger.New("backfill-attributes")
           defer log.Sync()

           // DB env vars from CLAUDE.md — match the env shape used by every other service.
           dbCfg := database.LoadFromEnv()
           db, err := database.New(dbCfg)
           if err != nil {
               log.Fatalw("failed to connect to db", "error", err)
           }

           shikimoriCfg := config.ShikimoriConfig{
               GraphQLURL: getenv("SHIKIMORI_GRAPHQL_URL", "https://shikimori.one/api/graphql"),
               UserAgent:  getenv("SHIKIMORI_USER_AGENT", "AnimeEnigma/1.0 (https://animeenigma.ru)"),
               Timeout:    10 * time.Second,
               RateLimit:  cli.ShikimoriRPS,

           armClient := idmapping.NewClient()

               DryRun:        cli.DryRun,
               Limit:         cli.Limit,
               SkipShikimori: cli.SkipShikimori,
               SkipTags:      cli.SkipTags,
               LogEvery:      cli.LogEvery,
           })

           defer cancel()

           // Graceful Ctrl-C: cancel the in-flight context so the current
           // per-anime fetch finishes its rate-limited acquire and the loop
           // checks ctx.Err() between rows.
           sigCh := make(chan os.Signal, 1)
           signal.Notify(sigCh, syscall.SIGINT, syscall.SIGTERM)
           go func() {
               <-sigCh
               log.Warnw("shutdown signal received; cancelling backfill")
               cancel()
           }()

           started := time.Now()

           var shResult ShikimoriHalfResult
           if !cli.SkipShikimori {
               log.Infow("starting Shikimori half (kind/rating/material_source/studios)")
               r, err := runner.ShikimoriHalf(ctx)
               if err != nil {
                   log.Errorw("Shikimori half returned error (run continues)", "error", err)
               }
               shResult = r
           } else {
               log.Infow("skipping Shikimori half (--skip-shikimori)")
           }

           var alResult AnilistHalfResult
           if !cli.SkipTags {
               log.Infow("starting AniList half (tags)")
               r, err := runner.AnilistHalf(ctx)
               if err != nil {
                   log.Errorw("AniList half returned error (run continues)", "error", err)
               }
               alResult = r
           } else {
               log.Infow("skipping AniList half (--skip-tags)")
           }

           duration := time.Since(started)

           log.Infow("backfill complete",
               "duration_seconds", int(duration.Seconds()),
               "shikimori_succeeded", shResult.Succeeded,
               "shikimori_skipped", shResult.Skipped,
               "shikimori_failed", shResult.Failed,
               "anilist_succeeded", alResult.Succeeded,
               "anilist_skipped_no_anilist", alResult.SkippedNoAnilist,
               "anilist_skipped_already_done", alResult.SkippedAlreadyDone,
               "anilist_failed", alResult.Failed,
               "dry_run", cli.DryRun,
           )

               // Exit code 0 still — partial failure is expected (per-anime
               // errors are tolerated). The non-zero counts in the summary line
               // give the operator the signal to investigate.
               os.Exit(0)
           }
           os.Exit(0)
       }

           if v := os.Getenv(key); v != "" {
               return v
           }
           return fallback
       }
