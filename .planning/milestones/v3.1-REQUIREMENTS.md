# v3.1 Scraper Self-Healing — Requirements

**Milestone:** v3.1
**Status:** REOPENED 2026-05-19 (originally shipped 2026-05-13; reopened after regression + audit-gap + scope-expansion triggers)
**Last updated:** 2026-05-19

## Scope summary

Make the in-house scraper service **survive upstream-site regressions automatically and ship the user-facing surface that consumes it**. v3.1 originally delivered the *backend* self-healing infrastructure (playability gate, multi-URL fallback, daily canary, alerts, maintenance-bot pattern dispatch) but the *frontend* surface (`EnglishPlayer.vue`) was regressed away in the 2026-05-18 v3.0 Phase 20 cutover over-rotation. The reopening (Phases 24-26) restores the regressed surface, resolves the original 2026-05-13 audit gaps (BLK-INT-01 + W-INT-01..03), and adds provider expansion (AllAnime lift, 2026 candidate research, AnimeKai recovery).

**Decomposition:**
- **Phases 21-23** (SHIPPED 2026-05-13): backend self-healing infrastructure. Requirements SCRAPER-HEAL-01..16. SCRAPER-HEAL-08 (EnglishPlayer three-phase loader) regressed; restored in Phase 24.
- **Phase 24** (REOPENED): restore the regressed user-facing surface. Requirements SCRAPER-HEAL-17..20.
- **Phase 25** (REOPENED): resolve the original audit gaps (BLK-INT-01 + 3 warnings). Requirements SCRAPER-HEAL-21..24.
- **Phase 26** (REOPENED): expand provider pool. Requirements SCRAPER-HEAL-25..28.
- **Phase 27** (REOPENED): AnimePahe revival via stealth-Chromium sidecar. Requirements SCRAPER-HEAL-29..33.

**Non-goals (still):**

| Item | Reason |
|---|---|
| Headless browser / chromedp / utls / FlareSolverr | PoC 2026-05-13 proved real Chromium gets the same ad-decoy poisoning — egress IP is the limiting factor, not fingerprint. Anti-bot dependencies remain CI-rejected. |
| Multi-source switcher UI exposing every backend provider as a separate tab | The 2026-05-12 design landed one "English" tab with an in-player Source dropdown. The cleanup correctly removed multi-tab complexity; Phase 24 restores the *single* tab + in-player dropdown, not the multi-tab world. |
| WARP egress sidecar to revive VibePlayer | Separate v3.2+ effort, gated on operator appetite. |
| MinIO hot archival of EN streams | Separate v3.2+ effort. |
| 9anime/AniWave/Kaido mirrors | Confirmed dead 2026-05-12 + 2026-05-19 (no public mirror responds with episode lists). Out of v3.1 scope; replacement candidates evaluated in Phase 26. |

## Cross-cutting non-functional (inherits v3.0)

These were locked in v3.0 (`SCRAPER-FOUND-*`, `SCRAPER-NF-*`) and apply unchanged to v3.1's new phases. No re-statement; treated as invariants.

- 10-second hard timeout on every upstream HTTP call.
- `hashicorp/go-retryablehttp` exponential backoff (1s → 2s → 4s → 8s); no hand-rolled retry loops.
- `BaseHTTPClient` for all provider clients; no ad-hoc `http.Client`.
- Per-host `golang.org/x/time/rate.Limiter`; default 1 RPS, weight=2.
- `parser_requests_total`, `parser_request_duration_seconds`, `parser_fallback_total{from,to}`, `parser_zero_match_total{provider,selector}` emitted with `{provider}` labels.
- CI rejects anti-bot deps: `chromedp`, `go-rod`, `chromedp-rod`, `utls`, `tls-client`, `cloudscraper_go`, `flaresolverr`.
- Cache TTLs: 24h malsync IDs, 6h episodes, 15min search, ≤ min(parsed expiry − 30s, 5min) stream URLs.
