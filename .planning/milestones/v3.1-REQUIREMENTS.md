## Core value

The in-house EnglishPlayer / scraper service **survives upstream-site regressions automatically**. When VibePlayer / StreamHG / Earnvids / anitaku.to changes shape (regex misses, CDN rotates, ad-CDN poisoning appears), the system: (a) routes users around the broken provider transparently within one playback attempt, (b) detects the regression within 24 hours via a canary cron, (c) pages the existing maintenance bot with enough context to apply a known-pattern fix or escalate cleanly. No new operator burden.

## Architectural anchor

v3.1 keeps the v3.0 stance: **Go-native, no Puppeteer / headless-browser sidecar**. PoC 2026-05-13 proved real-browser extraction does not help VibePlayer (IP-level poisoning) and is unnecessary for StreamHG/Earnvids (Go regex on packed JS works). See `docs/plans/2026-05-13-scraper-self-healing-spec.md ┬¦2-┬¦3`.
