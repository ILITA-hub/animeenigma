<objective>
Extend the analytics collect handler so an FE-originated beacon row can be written into the ClickHouse `events` register with its own `source`, `operation`, `target`, `target_kind`, `requests`, `duration_ms`, and `accuracy` — instead of being silently mis-tagged as a backend (`source='be'`) clickstream row. Add a server-side whitelist on `source` so a forged public beacon cannot inject an arbitrary attribution origin.

Purpose: Without this mapping an FE "call" or "RUM" row lands as a generic `custom` clickstream event with `source='be'` (the `clickhouse_store.go:119` default), making it invisible to register pivots and contaminating authoritative byte aggregations (RESEARCH Pitfall 3).
Output: Updated `collect.go` field-mapping + whitelist; extended `collect_test.go` with the two register proofs (AR-FE-01 mapping + AR-FE-03 byte-poverty) and a forged-source rejection case.
</objective>
