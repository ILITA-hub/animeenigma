<objective>
Extend the existing axios request interceptor — the very block that already mints the W3C `traceparent` and calls `stampTrace(traceId)` — to ALSO emit a lightweight `source='fe'` register row carrying the call's `trace_id`, the current route, an optional caller-supplied semantic `action`, and the API path as `target`. Because the emitted row carries the SAME `traceId` that `stampTrace` back-fills onto the pending click, the click, the FE call row, and the downstream BE effect rows all share one `trace_id` — closing the FE→BE causation last mile (AR-FE-01) and giving the click its trace linkage (AR-FE-02).

Purpose: Today the interceptor mints a trace_id and stamps the click, but never writes an FE row, so there is no `source='fe'` register row to join against the BE effects. This plan emits it (RESEARCH Pattern 1/2, Open Q1).
Output: extended `client.ts` interceptor; extended `traceContext.spec.ts` asserting the click↔trace stamp end-to-end.
</objective>
