<objective>
Wire ClickHouse into the live analytics path and migrate the clickstream onto it the lowest-risk reversible way: a backend selector (`ANALYTICS_STORE_BACKEND`, default `postgres`) plus a dual-write `EventStore` wrapper that fans every `InsertBatch`/`UpsertIdentity` to both stores (Postgres stays source of truth; a CH failure is logged + best-effort, never propagated). Then add the NEW `aenigma-clickhouse` Grafana datasource, rewrite the 6 `product-analytics` panels in ClickHouse SQL against `events_resolved`, deploy to production, and prove a live clickstream event flows end-to-end into ClickHouse and renders on a panel (AR-STORE-04).

Purpose: Deliver AR-STORE-04 — the existing clickstream is migrated onto ClickHouse, still ingests, and the dashboards still render. The store impl (01-02) and the running container (01-01) are the inputs; this plan connects them to the live service and Grafana without breaking the existing Postgres-backed flow.
Output: a `DualWriteStore`, the config selector + CH connection wiring in `main.go`, the analytics→clickhouse compose dependency + `CLICKHOUSE_*` env, the `aenigma-clickhouse` datasource, the 6 rewritten panels, and a recorded live end-to-end smoke result.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Denial of Service | a CH outage blocking the live write path | mitigate | DualWriteStore is PG-authoritative + CH best-effort (CH error logged + swallowed); dualwrite degrades to PG-only if CH open/schema fails at boot — analytics never goes down for a CH blip |
| T-01-08 | Tampering | reusing a Grafana datasource uid | mitigate | brand-new `aenigma-clickhouse` uid; Postgres/Prometheus/Loki/Tempo uids untouched (datasources.yml header warning) |
| T-01-09 | Info Disclosure | CH credentials in compose/datasource | mitigate | password via `${CLICKHOUSE_PASSWORD}` env + `secureJsonData` (Postgres pattern); CH ports stay 127.0.0.1-bound (01-01) |
| T-01-10 | Tampering | panel rawSql against CH | mitigate | panels are admin-authored static SQL behind admin-only Grafana (auth-proxy SSO); read-only datasource |
| T-01-11 | Info Disclosure | GDPR erase missing CH rows after migration | mitigate | erase path extended to ClickHouse when CH backend active (Task 2) so right-to-erasure stays complete |
</threat_model>
