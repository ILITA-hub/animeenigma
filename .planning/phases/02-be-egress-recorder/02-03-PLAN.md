<objective>
Aggregate HLS proxy egress to one effect row per (stream-session, host) — never one row per ~6s segment — and capture both `bytes_out` (client egress) and `bytes_in` (upstream ingress). Inject a per-manifest `?sess=` token into the rewritten `.m3u8` (riding the existing `rewriteHLSURL` seam), tally returning segment GETs into an in-memory per-session map in the streaming process, and have an idle-timeout reaper emit ONE aggregated row at session end.

Purpose: AR-EGRESS-04 (one row per session,host) + AR-EGRESS-05 (dual byte counting). The register must not drown in per-segment noise (D-03/D-04/D-05). "One row per watch" is the milestone's awareness intent.
Output: A streaming-process-local session aggregator + dual byte counters; a single aggregated effect row per real watch session.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser segment GET → streaming proxy | untrusted ?sess= token + upstream URL cross here |
| streaming proxy → upstream CDN | proxy reads resp.Body (bytes_in) and writes client sink (bytes_out) |
| in-memory session map | unbounded growth risk from distinct tokens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-DOS | Denial of Service | HLS session map | mitigate | Reaper evicts idle sessions + hard map-size cap with oldest-eviction; mutex held only around the map mutation, never across io.Copy. |
| T-02-SESS | Tampering | ?sess= token | mitigate | crypto/rand per-manifest token; non-authoritative (carries NO authority — worst case is mis-attributed awareness data, not a security boundary — Security V3). |
| T-02-LOCK | Denial of Service | proxy concurrency | mitigate | Counting wrappers use atomic counters; no lock held across the copy → concurrent streams not serialized (Pitfall 5). |
| T-02-PII | Information Disclosure | session token | mitigate | Token NOT derived from user_id/PII (crypto/rand). |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new external packages (crypto/rand + sync/atomic are stdlib). |
</threat_model>
