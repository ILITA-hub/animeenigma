<objective>
Add the GORM after-callback set that records DB writes as `db_write` effect rows (always) and DB reads as `db_read` rows only when a SELECT exceeds its own per-`(operation, table)` P95 (D-01/D-02). Build the `ReadGate` as an in-memory threshold snapshot with a static cold-start default (the daily-P95 producer that REFRESHES this snapshot ships in plan 05). The hook emits to the Phase-2 global `EffectSink`, captures operation PCs cheaply (resolved async via plan 01's resolver), and issues ZERO DB queries (Pitfall 1 / gorm #7044).

Purpose: AR-EFFECT-01 — DB writes fact-rowed with `table`/`op`/`rows`; trivial reads NOT fact-rowed (the cost ledger stays sparse — D-04). The write path is deterministic; the read path is the P95-gated sparse capture.

Output: `RegisterEffectCallbacks`, the `ReadGate` snapshot, and their unit tests (Wave-0 scaffolds per VALIDATION.md).

Effort: `UXΔ = +1 (Better)` (operation-level DB cost attribution; the "expensive popular write" insight derives here) · `CDI = 0.05 * 8` (one callback registration + a gate snapshot in an existing sub-package; the gotcha discipline is the craft) · `MVQ = Griffin 85%/84%` (fuses GORM callbacks + the P95 gate + the shared resolver into one clean db-effect story; slop-resistant via the proven callback seam).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GORM statement ctx → effect row → analytics sink | Internal; user_id from private ctx only, never baggage |
| ReadGate snapshot (in-memory) ← plan-05 refresher | The refresh source (Redis/analytics) is consumed only by the ticker, never by the hot-path callback |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Denial of Service | GORM callback recursion / self-amplification (a query inside the callback re-fires the Query callback) | mitigate | Hook issues ZERO DB queries (Pitfall 1); reads statement fields only; emits to non-blocking sink |
| T-03-07 | Denial of Service | every SELECT does a synchronous threshold lookup (Redis round-trip per query) | mitigate | ReadGate is an in-memory atomic snapshot (Pitfall 4); refreshed off-path by the plan-05 ticker |
| T-03-08 | Information Disclosure | user_id mislabeled or leaked onto a db effect's wire | mitigate | UserID sourced only from UserIDFromContext (private ctx); db effects post to analytics /internal/effects over Docker network, never to 3rd parties |
| T-03-09 | Tampering | analytics' OWN ingestion writes recorded as db effects (self-referential sink amplification) | mitigate | RegisterEffectCallbacks is NEVER wired into services/analytics (D-16) — enforced in plan 06 boot wiring; documented here as the consumer constraint |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages; may add `gorm.io/driver/sqlite` ONLY as a test dependency if not already present — confirm it is an existing module in go.work before adding; if a new go.mod require is needed, run the legitimacy check against pkg.go.dev/gorm.io/driver/sqlite |
