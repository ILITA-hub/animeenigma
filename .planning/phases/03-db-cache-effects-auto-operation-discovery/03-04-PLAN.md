<objective>
Add an in-process cache hit/miss aggregator (byte-for-byte clone of the Phase-2 `HLSSessions` reaper) that counts `(key_class, result, operation)` and flushes summed `cache` effect rows on a ~10s interval (D-05/D-06/D-10), plus the key-class classifier derived from the `libs/cache/ttl.go` taxonomy (D-07). Hook it into the existing `RedisCache.Get`/`Set` hit/miss metering sites as an OPTIONAL nil-guarded field, so cache-less call paths and tests need no aggregator.

Purpose: AR-EFFECT-02 — cache effects recorded as hit/miss by key-class. Aggregation keeps volume low while preserving full per-class hit-rate fidelity. Per D-10 (RESEARCH A5 recommendation): aggregated cache rows use the COARSE baggage operation read cheaply from ctx (NOT a per-op stack-walk, which would violate D-11 on every Get/Set).

Output: `keyclass.go`, `aggregator.go`, the `cache.go` hooks, and their deterministic-clock tests (Wave-0 scaffolds per VALIDATION.md).

Effort: `UXΔ = +1 (Better)` (per-class cache hit-rate visibility; distinguishes hot list cache from cold similar cache) · `CDI = 0.04 * 8` (one classifier + one HLS-reaper clone + nil-guarded hooks in the cache lib) · `MVQ = Sprite 86%/85%` (a tight reuse of a production-proven aggregator; slop-resistance comes from cloning the exact lock/eviction/clock discipline).
</objective>
