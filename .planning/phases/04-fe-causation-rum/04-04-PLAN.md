<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    The FE causation + RUM instrumentation is deployed: the axios interceptor emits a source='fe' register row sharing the call's trace_id, clicks are stamped with that trace_id, and a PerformanceObserver beacons per-host source='fe_rum'/accuracy='approximate' rows carrying no bytes. The collect handler maps these into ClickHouse `events`.
  </what-built>
  <how-to-verify>
    1. Open the app in a real browser (the production web URL). Open DevTools → Network, confirm beacons POST to `/api/analytics/collect` (204).
    2. Perform a tracked action that triggers an API call (e.g. open an anime detail page, or click "play episode"). Note the approximate time.
    3. Query ClickHouse `events` for the AR-FE-01 join — replace {trace} with a recent trace_id (find one via `SELECT trace_id FROM events WHERE source='fe' ORDER BY timestamp DESC LIMIT 5`):
       `SELECT trace_id, source, origin, operation, effect_kind, target, requests, bytes_in, bytes_out FROM events WHERE trace_id = '{trace}' ORDER BY timestamp;`
       EXPECT: exactly one source='fe' (or 'fe_rum') row AND ≥1 source='be' effect row, all sharing the trace_id.
    4. AR-FE-02 click join: `SELECT event_type, source, trace_id FROM events WHERE event_type='click' AND trace_id != '' ORDER BY timestamp DESC LIMIT 5;` then confirm one of those trace_ids also appears on a source='be' effect row (re-run the step-3 query with that trace_id).
    5. AR-FE-03 RUM presence + byte-poverty:
       `SELECT source, accuracy, target_kind, target, requests, duration_ms, bytes_in, bytes_out FROM events WHERE source='fe_rum' ORDER BY timestamp DESC LIMIT 10;`
       EXPECT: accuracy='approximate', target_kind='host', target is a bare host (no path/query/token), bytes_in==0 && bytes_out==0.
    6. AR-FE-03 non-contamination:
       `SELECT sum(bytes_in) AS rum_bytes FROM events WHERE source='fe_rum';`  → EXPECT 0.
       `SELECT target, sum(bytes_in) AS authoritative_in FROM events WHERE effect_kind='egress' AND source='be' GROUP BY target ORDER BY authoritative_in DESC LIMIT 10;`  → authoritative totals exclude the RUM rows.
    7. Confirm `SELECT DISTINCT source FROM events;` now includes 'fe' and 'fe_rum' (not only 'be') — the mapping took (RESEARCH Pitfall 3 warning sign).
    Record the actual query outputs in the SUMMARY.
  </how-to-verify>
  <resume-signal>Type "approved" if the trace_id join, the click→effect join, and the source=be byte filter (with rum_bytes==0) all hold; otherwise describe the mismatch (e.g. only 'be' in DISTINCT source → mapping didn't take; or non-zero rum_bytes → a byte leaked onto an fe_rum row).</resume-signal>
