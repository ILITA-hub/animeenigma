<objective>
Run the one-time migration of all pre-existing admin content into the unified pool:
for each admin `library_episodes` row still on the legacy `{shikimori_id}/{ep}/` prefix,
`Move` the MinIO objects to `aeProvider/<mal>/RAW/<ep>/`, then repoint `minio_path`.
The migration is idempotent + restart-safe and runs once at boot after migrations 005/006
apply and before the service serves traffic. Also audit the catalog "ae" resolver path to
confirm it builds URLs only from per-row `minio_path` (so repointing is transparent).

Purpose: POOL-02 (one-time admin-content migration without interrupting playback) and the
spec §10 sequencing invariant — admin rows MUST be in the metered pool before the Phase 10
evictor goes live, or budget math is wrong.
Output: `autocache/migrator.go`, two new `EpisodeRepository` methods, boot wiring, audit note.

This phase lays the migration only — it does NOT implement eviction, serving, or triggers.
</objective>
