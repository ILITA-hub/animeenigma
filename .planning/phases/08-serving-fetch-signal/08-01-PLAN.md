<objective>
Build the library-side data + metrics foundation for the ae serve/fetch signal:
the durable `autocache_demand` table (migration 007), its GORM model + `DemandRepository.Record`
upsert, the `EpisodeRepository.BumpFetch` ledger update, and the
`library_autocache_serve_total{result}` Prometheus counter.

Purpose: Phase 8's serve path needs (a) a durable place to record backfill demand that Phase 9's
Planner can drain across restarts, (b) a way to bump the `last_fetch_at`/`fetch_count` ledger
columns Phase 7 added (SERVE-02), and (c) a hit/miss counter (SERVE-01/03 / Phase 11 charts it).
This plan delivers all three primitives; Plan 02 mounts the endpoints that call them.

Output: migration 007 + embed registration, `domain.AutocacheDemand`, `repo/demand.go`,
`EpisodeRepository.BumpFetch`, and the `serve_total` CounterVec quadruple (field/register/Inc/test-seam).
