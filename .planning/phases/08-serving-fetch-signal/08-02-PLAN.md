like recs `/internal/recs/recompute-hint` and notifications `/internal/notifications`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| catalog→library `/internal/*` (Docker network) | unauthenticated internal endpoints; trusted because unreachable from the gateway/public internet by construction |
| gateway→public | gateway proxies only /api/library/* + /api/library/health — never /internal/* |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Elevation of Privilege | /internal/* endpoints (unauthenticated) | mitigate | Mounted outside /api/library and never given a gateway HandleFunc; reachable only on the Docker network (same model as recs/notifications /internal/*) |
| T-08-05 | Spoofing | demand/fetch body (mal_id/episode) | accept | Internal trusted caller; mal_id/episode are non-secret identifiers; reason forced to backfill server-side so a spoofed reason cannot inject next_ep |
| T-08-06 | Tampering | master enabled switch bypass | mitigate | Demand recording + miss-count are gated server-side on AutocacheConfigRepository.Get(...).Enabled; a config-read error fails closed (skip side effects) |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (stdlib + existing chi/gorm/prometheus); slopcheck N/A |
</threat_model>
