<objective>
Wire the catalog ae-resolution seam to fire the serve signal: add `RecordFetch` / `RecordDemand`
best-effort POST methods to the catalog→library client, and call them from `GetLibraryStream`
(the strict ae path) — HIT just before returning the pool stream, MISS just before the NotFound
return — as non-blocking, drop-on-failure goroutines that never fail the playback resolution.

Purpose: This closes the SERVE loop. The ae provider (`GetLibraryStream`) is where HIT vs MISS is
observable; catalog stays ignorant of autocache config (the library side gates on `enabled`). The
fire-and-forget contract matches the recs recompute-hint producer so a library blip or client
disconnect never breaks an ae playback resolution — and SERVE-03 "no regression" is satisfied
because the AllAnime-raw failover control flow is untouched (only ADD the side-effect calls).

Output: two thin client POST methods + a small `postInternal` helper, the two best-effort calls
wired into `GetLibraryStream`, and tests proving non-blocking + no-error-propagation + failover-unchanged.
</objective>
