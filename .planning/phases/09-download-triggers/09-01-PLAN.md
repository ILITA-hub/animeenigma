<objective>
Lay the library-side schema + repository + metric + reason-attribution foundation the Phase-9 Planner
(Plan 09-02) and the two demand producers (Plans 09-03/04) all build on. This plan ships NO drain loop and
NO behavior change to the search/download pipeline — it widens the schema, adds the repo/metric primitives,
and corrects the Phase-8 demand handler so it HONORS the producer's wire reason.

Three idempotent migrations: 008 adds `autocache` to the `job_source` enum, 009 adds a nullable
`episode INT` column to `library_jobs` (the intended episode persisted at enqueue for single-flight
dedup + trigger attribution), and 010 adds `ongoing` to the `autocache_demand_reason` enum (Logic A,
distinct from Logic B's `next_ep`, per CONTEXT decision 7 / "Recommend (a)"). The GORM `Job` struct
gains the `Episode` field; the `DemandReason` enum gains `DemandReasonOngoing`. New repo methods:
`JobRepository.HasActiveForEpisode` (the TRIG-04 single-flight gate) and `DemandRepository.Drain` +
`DemandRepository.Delete` (the TRIG-03 drain primitives). A new
`library_autocache_downloads_total{trigger,result}` counter (TRIG-05 / OBS-04 series).

CRITICAL FIX (Task 4): Phase 8's `autocache_internal.go` Demand handler hard-codes
`domain.DemandReasonBackfill`, discarding the wire `reason`. With the Phase-9 producers (Logic B sends
`next_ep`, Logic A sends `ongoing`), that override would silently collapse EVERY demand to `backfill`,
making `downloads_total{trigger}` 100% backfill and the new `ongoing` enum unreachable (breaking CONTEXT
D7 A/B/backfill attribution). Task 4 changes the handler to ACCEPT + VALIDATE the wire reason against
`{backfill, next_ep, ongoing}`, defaulting to `backfill` ONLY when the field is absent or not a known
value. This is safe because the spoof-mitigation intent (T-08-05) was about untrusted callers — but
`/internal/*` is Docker-network-only (gateway never proxies it, per 08-02-SUMMARY T-08-04), so the
producers (player + scheduler) on that path are internal-trusted and their true reason is honored.

Purpose: every downstream Plan-9 task imports these contracts — defining them first prevents
executors from exploring the codebase to discover them — and the reason seam is correct by
construction so the producers never need a follow-up.
Output: 3 migrations + embeds, widened domain models, 3 new repo methods, 1 new metric, and the
reason-honoring demand handler.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| migration → live schema | DDL applied at library boot; must be idempotent + restart-safe |
