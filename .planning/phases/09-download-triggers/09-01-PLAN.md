<objective>
Lay the library-side schema + repository + metric foundation the Phase-9 Planner (Plan 09-02)
and the two demand producers (Plans 09-03/04) all build on. This plan ships NO drain loop and
NO behavior change — it only widens the schema and adds the repo/metric primitives.

Three idempotent migrations: 008 adds `autocache` to the `job_source` enum, 009 adds a nullable
`episode INT` column to `library_jobs` (the intended episode persisted at enqueue for single-flight
dedup + trigger attribution), and 010 adds `ongoing` to the `autocache_demand_reason` enum (Logic A,
distinct from Logic B's `next_ep`, per CONTEXT decision 7 / "Recommend (a)"). The GORM `Job` struct
gains the `Episode` field; the `DemandReason` enum gains `DemandReasonOngoing`. New repo methods:
`JobRepository.HasActiveForEpisode` (the TRIG-04 single-flight gate) and `DemandRepository.Drain` +
`DemandRepository.Delete` (the TRIG-03 drain primitives). A new
`library_autocache_downloads_total{trigger,result}` counter (TRIG-05 / OBS-04 series).

Purpose: every downstream Plan-9 task imports these contracts — defining them first prevents
executors from exploring the codebase to discover them.
Output: 3 migrations + embeds, widened domain models, 3 new repo methods, 1 new metric.
</objective>
