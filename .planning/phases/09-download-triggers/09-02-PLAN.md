<objective>
Build the library Planner — the periodic loop that drains `autocache_demand` and turns wanted
(mal_id, episode) rows into RAW download jobs, with single-flight dedup (TRIG-04), RAW/quality/seeder
gating (TRIG-05), and the backfill-drain semantics (TRIG-03). This is the core of Phase 9: producers
(Plans 09-03/04) and Phase-8's serve-miss path all funnel demand here; this loop consumes it.

The Planner runs a ctx-aware ticker (no new dep — `time.Ticker`, per RESEARCH recommendation) that
each tick re-reads `autocache_config` live (master `enabled` + `sweep_interval_min`), drains the oldest
demand rows, and for each: skips+clears if already present in the pool (`GetByShikimoriEpisode`), skips
if a non-terminal job already targets it (`HasActiveForEpisode`), searches via `TieredSearcher.FetchAll`,
filters to RAW ≤`quality_cap` ≥`min_seeders`, and on a winning release calls `jobRepo.Create` with
`source=autocache, episode=intended`. It increments `library_autocache_downloads_total{trigger,result}`
(trigger from the demand reason: ongoing→A, next_ep→B, backfill→backfill). main.go applies migrations
008/009/010 and wires the Planner's Start/Stop into the existing boot/shutdown lifecycle.

Purpose: autonomous RAW acquisition driven entirely by the durable demand table.
Output: `internal/autocache/planner.go` + the RAW filter + main.go wiring.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| autocache_demand → Planner | durable demand drained into download jobs; the only "who wants what" the library sees |
| search result → enqueue | untrusted magnet from a torrent indexer crosses into the job queue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Tampering | malicious magnet from a search result | mitigate | The existing enqueue path parses the magnet (`metainfo.ParseMagnetUri`, jobs.go:181) before the worker acts; the Planner uses the same `domain.Release.Magnet` shape an admin enqueue uses — no new trust surface |
| T-09-05 | Denial of Service | Jackett thundering-herd re-search | mitigate | Per-(mal,ep) last-searched-at backoff + bounded per-sweep fan-out + Drain batch limit; a not-yet-released ep is not re-searched every tick (RESEARCH Pitfall 4) |
| T-09-06 | Elevation of Privilege | Planner bypassing the master switch | mitigate | The loop re-reads `autocache_config.Enabled` live each tick; when disabled it drains/enqueues nothing (POOL-05) |
| T-09-07 | Tampering | wrong-track download (DUB instead of RAW) | mitigate | selectRAW negative-token + uploader allowlist + quality/seeder gate (TRIG-05) reject DUB/dual-audio/oversized releases before enqueue |
| T-09-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (stdlib time.Ticker + existing gorm/prometheus/errgroup); RESEARCH audit confirms zero new deps |
</threat_model>
