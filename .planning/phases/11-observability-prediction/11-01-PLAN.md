<objective>
Add the ONE new backend piece of Phase 11: a daily scheduler cron job that computes the §7 v1 storage-need heuristic and publishes it as `library_autocache_predicted_bytes{component="ongoing"|"nextep"}` on the scheduler `/metrics` endpoint, so the Grafana OBS-05 table panel (Plan 11-02) can join it against the library-exposed `library_autocache_budget_bytes`.

Purpose: OBS-05 requires a daily heuristic prediction of pool storage need (ongoing + next-episode components) charted against the budget. The prediction needs the same shared-DB watcher/combo counts Logic A enumerates, so it lives in the scheduler (the shared `animeenigma` DB), not library (separate DB). This plan is a leaner sibling of the Phase-9 Logic A job: same DISTINCT join, but it only COUNTS rows and sets a gauge — no per-row HTTP demand fan-out.

Output: a new `AutocachePredictedBytes` GaugeVec in `libs/metrics`, the `AutocachePredictionJob`, its config (cron + avg-ep-bytes env + a new `getEnvInt64` helper), and the wiring (registration in `job.go`, unconditional DI in `main.go`).
