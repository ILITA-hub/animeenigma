<objective>
Land Phase 13 (S6 Combo-Watched-After Pin) end to end: the synchronous seed update inside MarkEpisodeWatched, the local co-occurrence + Shikimori /similar cascade, the nightly co-occurrence materialization cron, the new catalog /similar endpoint, and the visually-distinct pinned tile in the Home.vue Up Next row. After this plan ships, finishing an anime with score >= 7 puts a "Because you finished X" pin at index 0 of the logged-in user's recs row within seconds.

This is implemented as a single multi-service plan rather than a wave-split because the work is sequenced, not parallelizable: the player's S6 cascade depends on the catalog /similar endpoint being live, the handler integration depends on S6 + the new repo method, the Home.vue card treatment depends on the API contract extension. A wave split would force three half-plans that can't be checkpointed independently. The plan honors the Phase 11/12 task-shape conventions (read_first, behavior, action, acceptance_criteria, verify) so each task remains executor-deterministic.

Per locked decisions §B1-B7 in the CONTEXT.md:
- B1 — synchronous seed update via narrow UpdateS6Seed (NOT full-row UPSERT) inside MarkEpisodeWatched, latency budget < 5 ms, fire-and-forget cache invalidation.
- B2 — S6 is NOT a SignalModule; it is a separate pin resolver invoked AFTER the ensemble ranks the row.
- B3 — new catalog /similar endpoint mirrors the existing /related shape, REST not GraphQL.
- B4 — nightly (24h) cron via new CoOccurrenceOrchestrator mirroring PopulationOrchestrator.
- B5 — cascade trigger to Shikimori is post-S11 candidate count < 5; score-7 → score-5 fallback; never fall to score > 0.
- B6 — pin treatment is colored left-border + top-right badge + row-header reason line.
- B7 — API contract: recs[0] gains pinned/pin_reason/pin_seed_anime_id/pin_source fields; Final is 0 (the JSON-zero approximation of "null" since RecItem.Final is float64 — frontend gates on Pinned, not Final).

- Catalog: GetSimilarAnime client method, SimilarAnime domain type, service-layer + handler + router + cache key.
- Player repo: UpdateS6Seed (narrow update), GetTopCoOccurrences.
- Player service/recs: CoOccurrenceOrchestrator (cron), signals/s6_combo_pin (resolver) + HTTPShikimoriSimilarClient (in same package).
- Player service: ListService.MarkEpisodeWatched extension (synchronous seed + cache bust), constructor change.
- Player handler: RecsHandler gains s6 + integrates pin in computeFreshForUser.
- Player main.go: three new wirings (co-occurrence orchestrator, S6 module, ListService new args).
- Frontend: Home.vue pin treatment, useRecs RecItem type extension, three locale files (recs.pinBadge), changelog.json entry.

After this plan ships, Task 9 (human-verify checkpoint) executes the production verification flow: capture pre-pin baseline, redeploy player + catalog + web, seed ui_audit_bot's s6_seed_*, hit GET /api/users/recs, assert recs[0].pinned===true and pin_reason starts with "Because you finished", time MarkEpisodeWatched p95 (must add < 5ms), verify pin expiry at 8 days, run the after-update skill.

INSERT INTO rec_completion_co_occurrence (seed_anime_id, candidate_anime_id, co_count, last_computed)
SELECT a.anime_id, b.anime_id, COUNT(DISTINCT a.user_id) AS co_count, NOW()

JOIN anime_list b ON a.user_id = b.user_id AND a.anime_id <> b.anime_id

       // for production; sqlite test fixture uses the same syntax (sqlite supports
       // ON CONFLICT DO UPDATE since 3.24).
       func (o *CoOccurrenceOrchestrator) RunOnce(ctx context.Context) error {
           const sql = `
               INSERT INTO rec_completion_co_occurrence (seed_anime_id, candidate_anime_id, co_count, last_computed)
               SELECT a.anime_id, b.anime_id, COUNT(DISTINCT a.user_id) AS co_count, NOW()
               FROM anime_list a
               JOIN anime_list b ON a.user_id = b.user_id AND a.anime_id <> b.anime_id
               WHERE a.status = 'completed' AND a.score >= 7
                 AND b.status = 'completed' AND b.score >= 7
               GROUP BY a.anime_id, b.anime_id
               HAVING COUNT(DISTINCT a.user_id) >= 1
               ON CONFLICT (seed_anime_id, candidate_anime_id) DO UPDATE
                 SET co_count = EXCLUDED.co_count, last_computed = EXCLUDED.last_computed`
           if err := o.db.WithContext(ctx).Exec(sql).Error; err != nil {
               return fmt.Errorf("recs: co-occurrence materialize: %w", err)
           }
           return nil
       }

       // exits the goroutine. Mirrors PopulationOrchestrator.Start (population.go).
       func (o *CoOccurrenceOrchestrator) Start(ctx context.Context, interval time.Duration) {
           go func() {
               if err := o.RunOnce(ctx); err != nil {
                   o.log.Errorw("co-occurrence cron failed (boot tick)", "error", err)
               } else {
                   o.log.Infow("co-occurrence cron boot tick complete")
               }
               ticker := time.NewTicker(interval)
               defer ticker.Stop()
               for {
                   select {
                   case <-ctx.Done():
                       o.log.Infow("co-occurrence cron stopped")
                       return
                   case <-ticker.C:
                       if err := o.RunOnce(ctx); err != nil {
                           o.log.Errorw("co-occurrence cron failed (tick)", "error", err)
                           continue
                       }
                       o.log.Infow("co-occurrence cron tick complete")
                   }
               }
           }()
       }
