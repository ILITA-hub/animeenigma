<objective>
Implement the AnimePahe provider — the first live `domain.Provider` registered with the scraper orchestrator. The provider:

- Resolves MAL IDs to AnimePahe anime IDs via `api.malsync.moe` (24h cache) with a fuzzy-title `/api?m=search` fallback (Jaro-Winkler ≥ 0.85).
- Lists episodes with `/api?m=release` pagination, caching the full list for 6 hours.
- Lists hoster servers by scraping `/play/{animeSession}/{epSession}` HTML for kwik.cx `data-src` URLs.
- Returns playable HLS streams by delegating kwik.cx URLs to the registry's Kwik extractor (Plan 16-02).
- Handles DDoS-Guard cookies via a single-handshake `check.ddos-guard.net/check.js` helper that populates the BaseHTTPClient cookie jar (Plan 16-01's Jar() accessor).
- Never caches session UUIDs across requests (per RESEARCH.md Pitfall 1).
- Caches stream URLs ≤ `min(parsed expires−30s, 5min)` so signed URLs don't outlive their signature (RESEARCH.md Pitfall 6).

Purpose: This is the core of Phase 16 — the first real provider satisfying SCRAPER-PAHE-01..04 + SCRAPER-NF-02. After this plan + 16-05's boot wiring, `GET /api/anime/{uuid}/scraper/episodes` returns real episode data instead of the Phase-15 503 stub.

Output: A self-contained `services/scraper/internal/providers/animepahe/` package with unit tests against the goldens captured in Plan 16-01.
</objective>
