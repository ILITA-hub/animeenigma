<objective>
Build the unified `EnglishPlayer.vue` component that replaces both HiAnime and Consumet player tabs end-to-end. The component is a fork of HiAnimePlayer.vue with three swaps:
1. API client: `hiAnimeApi` → `scraperApi` (Plan 16-04)
2. Accent color: purple → cyan (`--player-accent: #00d4ff`) per UI-SPEC §Color
3. New Source dropdown sub-component inside the toolbar (single-option collapse in Phase 16, multi-option in Phase 18+)

Then wire it into `Anime.vue`:
- Add an "English" tab button next to existing HiAnime+Consumet buttons
- Gate the HiAnime+Consumet tab buttons behind `route.query.legacy === '1'`
- Extend `videoProvider` type to include `'english'`
- Default EN-language users to `english` instead of `hianime`
- Add a v-else-if branch that mounts `<EnglishPlayer>`

Finally, ship a Playwright e2e spec that proves the wiring works end-to-end against the live deployed stack.

Purpose: This is the visible payoff of Phase 16. Without this plan, the backend changes from 16-01..05 are invisible to users. SCRAPER-UI-01 (EnglishPlayer.vue), SCRAPER-UI-02 (one English tab), SCRAPER-UI-04 (legacy=1 debug), and SCRAPER-NF-05 frontend-half (ReportButton provider) all land here.

Output: A working EnglishPlayer in production; `make redeploy-web` lands the new player; the Playwright spec demonstrates a green smoke against a real UUID via the ui_audit_bot test user.
</objective>
