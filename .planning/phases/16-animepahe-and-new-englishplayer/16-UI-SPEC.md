# Phase 16 — UI Design Contract: AnimePahe + new EnglishPlayer

> Visual and interaction contract for the unified EnglishPlayer.vue replacing the dual HiAnime+Consumet tabs end-to-end. Backed by the v3.0 scraper orchestrator (AnimePahe as the sole live provider in Phase 16; the "Source:" dropdown is single-option until Phase 18 adds 9anime). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This contract inherits the existing AnimeEnigma "Neon Tokyo" design system (Tailwind v4 CSS-first tokens in `frontend/web/src/styles/main.css`) and the per-player accent CSS-variable pattern already proven in HiAnimePlayer.vue / ConsumetPlayer.vue. Phase 16 introduces a NEW accent ($-cyan/blue family) for the unified English surface so it reads as a distinct identity from the legacy purple/green tabs during the soak.

## Typography

3 sizes, 2 weights. All values match existing player components verbatim so the new English tab is indistinguishable in rhythm from the surrounding chrome.

|------|------|--------|-------------|----------------|----------|

## Color

The project uses a Neon Tokyo dark theme (`--color-base: #121218`, `--color-surface: #1a1a24`). The per-player accent pattern (`--player-accent` CSS variable with `-rgb` companion + `accent-bg`, `accent-bg-muted`, `accent-text`, `accent-border` utility classes — proven in HiAnimePlayer.vue:1434-1446 and ConsumetPlayer.vue:1275-1286) is REUSED VERBATIM. Only the accent value changes.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#121218` (`--color-base`) | Page background, video element background, Source-dropdown panel background |
| Secondary (30%) | `rgba(255,255,255,0.05)` (`.glass-card` / `bg-white/5`) | All chrome surfaces inside the player: episode grid tiles, server rows, subtitle list, Source-dropdown trigger |
| Accent (10%) | `#00d4ff` (`--color-cyan-400`) → mapped to `--player-accent: #00d4ff` and `--player-accent-rgb: 0, 212, 255` | See "Accent reserved for" list below |
| Destructive | `#ff2d7c` (`--color-pink-500`) → `text-pink-400` / `bg-pink-500/20 border border-pink-500/30` | Inline `error` banner (existing pattern at HiAnimePlayer.vue:378-380); JP-subtitle activity highlight retains the existing `bg-red-500/20 text-red-400` from HiAnimePlayer.vue:278-279 (DO NOT change — would create cross-player inconsistency) |
| Warning | `#ffd600` (`--color-warning`) → `text-amber-400 / bg-amber-500/20` | `serverLoadWarning` banner (pattern at HiAnimePlayer.vue:368-374); reused unchanged |

**Accent reserved for (exhaustive list — do NOT extend without an explicit follow-up phase):**
1. Selected episode tile background (`accent-bg text-white`)
2. Selected server row background + checkmark badge (`accent-bg-muted border accent-border` and `accent-bg` for the badge circle)
3. Active sub/dub category tab pill (`accent-bg-muted accent-text border accent-border` for "Sub"; the existing blue `bg-blue-500/20 text-blue-400 border-blue-500/50` for "Dub" is PRESERVED VERBATIM from HiAnimePlayer.vue:193-195 for visual consistency)
4. Selected Source-dropdown item (`accent-bg-muted accent-text`)
5. Video.js progress bar + volume slider fill (`:deep(.video-js .vjs-play-progress) { background-color: var(--player-accent); }` — pattern at HiAnimePlayer.vue:1499)
6. Video.js big-play-button background hover (existing `:deep(.video-js .vjs-big-play-button:hover)` pattern)
7. Loading spinner ring (`accent-border border-t-transparent` — pattern at HiAnimePlayer.vue:5)
8. Focus rings (`:focus-visible` already routed through main.css:91-94 — uses `--color-cyan-400`, which matches the chosen accent for this player; no override needed)
9. Active state for "Mark as watched" success chip — REPLACES the green `bg-green-500/20 text-green-400 border-green-500/50` used in HiAnimePlayer.vue:101-103. Reason: green was tied to HiAnime's purple accent for contrast; with cyan as the new accent, green clashes. Use `accent-bg-muted accent-text border accent-border` instead — same pattern as ConsumetPlayer.vue:104-106 already does.

**60/30/10 audit:**
- 60% dominant `#121218` page bg + `#1a1a24` glass-card surface = 60% of pixels.
- 30% secondary `bg-white/5` translucent chrome (server rows, episode tiles in idle, subtitle list) = 30% of pixels.
- 10% accent cyan reserved for the 8 elements above. Compliant.

**Why cyan and not retain purple (HiAnime) or green (Consumet):**
The new "English" tab must read as a NEW identity, not a re-skin of either dying tab. Cyan (`--color-cyan-400`) is the project's primary brand color (used in main.css:91 for focus rings, in btn-primary, in glass-elevated shadow) — picking it for the unified English surface anchors the new tab to the brand spine rather than to either legacy tab. The brand color is already validated to pass WCAG AA contrast against `--color-base` (focus-ring usage in main.css proves this).

**API client** — new in `frontend/web/src/api/client.ts`:

```typescript
export const scraperApi = {
  getEpisodes: (animeId: string, prefer?: string) =>
    apiClient.get(`/anime/${animeId}/scraper/episodes`, {
      params: prefer ? { prefer } : undefined
    }),
  getServers: (animeId: string, episodeId: string, prefer?: string) =>
    apiClient.get(`/anime/${animeId}/scraper/servers`, {
      params: { episode: episodeId, ...(prefer && { prefer }) }
    }),
  getStream: (
    animeId: string,
    episodeId: string,
    serverId: string,
    category: 'sub' | 'dub',
    prefer?: string,
  ) =>
    apiClient.get(`/anime/${animeId}/scraper/stream`, {
      params: { episode: episodeId, server: serverId, category, ...(prefer && { prefer }) }
    }),
  getHealth: () => apiClient.get(`/anime/_/scraper/health`), // animeId is ignored for health
}
```

Do NOT repoint `hiAnimeApi` / `consumetApi` — they are deleted in Phase 20 (per SCRAPER-CUT-05).

**States:**
| State | Trigger appearance | Panel | Tooltip |
|-------|-------------------|-------|---------|
| Single-option (Phase 16 default) | Read-only chip "AnimePahe" (no caret) | Hidden | "Source: AnimePahe (only provider available)" on hover |
| Multi-option closed | `bg-white/5 border border-white/10`, caret pointing down | Hidden | "Change video source" |

| Item: healthy | Default item style | — | (provider name) |
| Item: selected | `accent-bg-muted accent-text` | — | "Current source" |
| Item: unhealthy (`provider_health_up{stage}=0`) | `opacity-40 cursor-not-allowed` | — | "Currently unreachable — orchestrator skipped this source" |
| Item: future (registered but Phase 16 doesn't know about it) | n/a in Phase 16 | — | n/a |

**Interactions:**
| Action | Behavior |
|--------|----------|
| Click trigger (single-option) | No-op. Cursor is `default`, no hover state. |
| Click trigger (multi-option) | Toggle panel open/closed. Focus moves to selected item. |
| Click item (healthy) | (1) Save current `vjsPlayer.currentTime()` as resume point; (2) update `selectedProvider`; (3) re-fetch `scraperApi.getStream(..., prefer: chosen)`; (4) restore time after stream loads; (5) write to `useWatchPreferences.preferredScraperProvider`; (6) close panel. |
| Click item (unhealthy) | No-op + show tooltip "Currently unreachable". |
| Escape key when open | Close panel; return focus to trigger. |
| Tab when open | Move focus through items sequentially; trap inside panel. |
| Click outside when open | Close panel; return focus to trigger. |

**Resume-on-switch implementation contract:**
```
async function switchProvider(next: string) {
  const resumeAt = vjsPlayer?.currentTime() ?? nativeVideoRef.value?.currentTime ?? 0
  selectedProvider.value = next
  await fetchStream() // honors prefer header
  // After stream loads + initPlayer fires:
  await nextTick()
  if (vjsPlayer) vjsPlayer.currentTime(resumeAt)
  else if (nativeVideoRef.value) nativeVideoRef.value.currentTime = resumeAt
}
```

If `fetchStream` returns an error (provider down), the toast `Couldn't switch source — staying on AnimePahe` fires AND `selectedProvider.value` is rolled back. Phase 16 has only one provider so this is a Phase-18+ concern, but the plumbing is in place.

## Edge Cases

| Case | Behavior |
|------|----------|
| User opens English tab on anime with NO malsync coverage | Empty state shows `player.englishNotAvailable.{heading,body}`. Source dropdown is NOT rendered (no providers to choose between). |
| `scraperApi.getHealth()` fails on mount | EnglishPlayer assumes all registered providers are healthy (fail-open). Dropdown shows all options. The orchestrator will still skip in-flight failures and return `tried[]`. |
| Scraper response has no `meta.tried` field | `triedChain.value = [activeProvider \|\| 'animepahe']`. ReportButton still gets a non-empty array. |
| User clicks Source item DURING stream load | Cancel in-flight request via AbortController (new — no existing pattern to inherit). Reset `loadingStream` to false on abort. Then fire new request. |
| User opens player with `?legacy=1` AND `?legacy=2` (or weird value) | Treat any truthy `legacy` query param as enabling debug tabs. Cast via `route.query.legacy === '1'` strictly — only `?legacy=1` activates debug. |
| Provider override stored in `useWatchPreferences` for a provider that is now unhealthy | On next visit: show the stored preference in the dropdown as "selected", but also show the unhealthy badge. Orchestrator will skip it on the next request and return real data from the next healthy provider; toast announces the silent fallback. |
| User on Phase 16 tries to switch to "9anime" via stored localStorage from a future visit | `availableProviders` only contains `['animepahe']` in Phase 16. Stored preference is ignored gracefully (no error toast — just falls through to AnimePahe). Persistence is preserved; will reactivate in Phase 18. |
| Anime.vue mounted with `videoLanguage='en'` and `videoProvider='english'` simultaneously with `?legacy=1` AND user picks `hianime` debug tab | English tab stays selected as the visible main tab. Debug tab `hianime` ALSO renders inline below the chrome (per CONTEXT.md "alongside English"). Both players exist in the DOM. The non-active one is `hidden` via `v-show="false"` — DO NOT unmount, as switching back loses state. (See Anime.vue's existing v-else-if chain at lines 416-465 — Phase 16 converts this from v-else-if to v-if + v-show.) |
