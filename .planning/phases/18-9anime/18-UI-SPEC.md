**Resume-on-switch contract (Phase 18 first activation):**

```ts
async function switchProvider(next: string) {
  const prior = selectedProvider.value
  const resumeAt =
    vjsPlayer?.currentTime() ??
    nativeVideoRef.value?.currentTime ??
    0

  selectedProvider.value = next
  setPreferredScraperProvider(next)

  try {
    await fetchServersAndStream() // honors `prefer: next`
  } catch (err) {
    // Provider switch failed ‚Äî roll back. Player keeps playing the prior stream.
    selectedProvider.value = prior
    setPreferredScraperProvider(prior)
    showToast(t('player.sourceSwitchFailed', { provider: capitalizeProvider(prior) }))
    return
  }

  await nextTick()
  if (vjsPlayer) vjsPlayer.currentTime(resumeAt)
  else if (nativeVideoRef.value) nativeVideoRef.value.currentTime = resumeAt
}
```

## Edge Cases

Phase 16 declared 8 edge cases (¬ßEdge Cases). Phase 18 ACTIVATES three of them for the first time AND introduces three new ones specific to the second provider:

### Phase 16 edge cases that ACTIVATE in Phase 18

| Case | Phase 18 first-activation behavior |
|------|-----------------------------------|
| User opens English tab on anime with NO malsync coverage | Empty state `player.englishNotAvailable.{heading,body}` shown. Source dropdown is hidden (no providers). **Phase 18 implication:** because Anitaku uses fuzzy title search as its PRIMARY ID-resolution path (per RESEARCH.md ‚Äî malsync has no Gogoanime key as of 2026-05-12), the "no coverage" condition triggers ONLY when Anitaku's fuzzy search ALSO fails (`<0.85` Jaro-Winkler) AND AnimePahe's malsync lookup also fails. The text is unchanged. |
| `scraperApi.getHealth()` fails on mount | Fail-open: `availableProviders` stays at the literal `['animepahe']`. **Phase 18 implication:** Anitaku is hidden from the dropdown until `getHealth` retries succeed. Users still get a working player (AnimePahe single-option). The dropdown auto-expands once `getHealth` succeeds on a subsequent request. |
| Provider override stored in `useWatchPreferences` for a now-unhealthy provider | On mount, `selectedProvider.value` is restored from storage; if the restored slug is in `availableProviders` but its `provider_health_up{stage="search"}=0`, the orchestrator skips it and the next-healthy provider serves the request. The dropdown shows the stored preference as `selected` with the `(offline)` badge so the user understands the silent fallback. Toast `player.sourceSwitchFailed` is NOT fired in this path ‚Äî the user didn't initiate the switch; the orchestrator did. Reserved for an explicit, future "stickiness toast" that we are NOT adding in Phase 18. |

### Phase 18 new edge cases

| Case | Behavior |
|------|----------|
| User clicks `Anitaku` item while a stream is still loading from AnimePahe | Cancel the in-flight `getServers`/`getStream` requests via the AbortController introduced in Phase 16 ¬ßEdge Cases. Reset `loadingStream` to false. Then fire new request with `prefer: gogoanime`. If user clicks fast enough to chain switches (A ‚Üí B ‚Üí A ‚Üí B), every prior request is aborted; the LAST click wins. |
| Anitaku returns 200 + episode list but every embed host fails extraction (`vibeplayer`/`streamhg`/`earnvids` all return `ErrExtractFailed`) | Orchestrator returns the FULL `tried[]` chain to the response envelope. Player shows in-video overlay copy `"Source unavailable. Tried: AnimePahe, Anitaku"` (multi-element chain). The Source dropdown remains usable ‚Äî user can pick a different provider ‚Äî but no other healthy EN provider exists in Phase 18, so the dropdown's only effect is the toast `player.sourceSwitchFailed`. Acceptable: this is the "EN-source coverage gap" empty path that Phase 19 (AnimeKai) is gated to fill. |
| User has `preferredScraperProvider='gogoanime'` set, but `availableProviders` only contains `['animepahe']` at mount (health response races, or Anitaku is temporarily skipped) | Same fail-open behavior as Phase 16's "future provider via localStorage" edge case: stored preference is ignored gracefully, no error toast, just falls through to AnimePahe. Once Anitaku reappears in `availableProviders`, the dropdown auto-restores the user's preference (because `selectedProvider.value` reads from the composable on every render, not just at mount). |
| User on a Phase-19+ build with AnimeKai opens an old session whose `triedChain` was `[animepahe, gogoanime]` | `triedChain.value` is overwritten on every new stream response. No special handling needed. The tried-chain debug line just renders whatever the LATEST response said. |
