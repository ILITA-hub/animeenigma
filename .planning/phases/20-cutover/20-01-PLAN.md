  <name>Task 1: Create executable cutover-preflight.sh</name>

  <action>
Create `scripts/cutover-preflight.sh` as a Bash script with `set -euo pipefail` and a leading shebang `#!/usr/bin/env bash`. The script implements four guardrail gates (per 20-CONTEXT.md decisions + ROADMAP success criterion 1) and emits a clear human-readable report. Use ONLY tools already in the host environment: `curl`, `jq`, `find`, `git`, `date`.

**Constants at top of script:**
```bash
PROM_URL="${PROM_URL:-http://localhost:9090/prometheus}"
EARLIEST_SHIP="2026-05-19"           # 7 days after EnglishPlayer first ship 2026-05-12 (commit 9e9d9a2)
ERROR_RATE_THRESHOLD=0.05            # 5% per-provider error rate ceiling
GUARDRAIL_WINDOW="7d"                # range vector for PromQL
PROVIDERS=(animepahe gogoanime)      # current EN providers as of 2026-05-12 (see STATE.md Phase 18 complete)
```

**Gate 1 — Date check (cheapest, fails first today):**
- `today=$(date -u +%Y-%m-%d)`
- If `today < $EARLIEST_SHIP`: print to stderr `[FAIL] Gate 1 (date): today=$today; earliest ship: $EARLIEST_SHIP (>= 7 days after EnglishPlayer shipped 2026-05-12)` and mark gate as failed.
- Continue evaluating remaining gates anyway (so the operator sees the complete report on the first run; do not short-circuit until the end). Use a `failed=()` array to collect failed gate names; exit code at the end is `${#failed[@]} != 0`.

**Gate 2 — Per-provider error rate <= 5% over 7d:**
- For each provider in `${PROVIDERS[@]}`:
  - Query: `sum(rate(parser_requests_total{parser="$provider",status="error"}[$GUARDRAIL_WINDOW])) / sum(rate(parser_requests_total{parser="$provider"}[$GUARDRAIL_WINDOW]))`
  - Use `curl -fsS --get --data-urlencode "query=<expr>" "$PROM_URL/api/v1/query"` and `jq -r '.data.result[0].value[1] // "0"'` for the scalar.
  - If the result is empty or `NaN`, treat the provider as having no traffic — that's a soft fail with message: `[WARN] Gate 2 (error_rate): provider=$provider has no recorded traffic in last 7d`. Add to `failed=()` because zero traffic means the guardrail "EnglishPlayer has served clean production traffic" is NOT satisfied for that provider.
  - If the ratio > `$ERROR_RATE_THRESHOLD`: `[FAIL] Gate 2 (error_rate): provider=$provider error_rate=<value> > 0.05`. Add to `failed`.
  - Otherwise `[PASS] Gate 2 (error_rate): provider=$provider error_rate=<value>`.

**Gate 3 — Zero Telegram alerts fired in 7d:**
The provider-health-stream-segment-down alert (Phase 17 17-04) is the gating alert. Query Prometheus for the alert's firing history:
- Query: `ALERTS_FOR_STATE{alertname="ProviderHealthStreamSegmentDown"}` — current state, AND
- Range query: `max_over_time(ALERTS{alertname="ProviderHealthStreamSegmentDown",alertstate="firing"}[$GUARDRAIL_WINDOW])`
- Use `curl --get --data-urlencode "query=..." "$PROM_URL/api/v1/query"`.
- If the range query returns ANY series with value `1` in the result vector, fail: `[FAIL] Gate 3 (telegram_alerts): ProviderHealthStreamSegmentDown fired in last 7d`. Add to `failed`.
- Otherwise `[PASS] Gate 3 (telegram_alerts): no firing alerts in last 7d`.
- IMPORTANT: do NOT rely on Telegram message history (no API available). Prometheus alert state is the source of truth — Phase 17 wired the alert to Telegram, so "alert did not fire" is sufficient proof "no Telegram notification was sent".

**Gate 4 — No new player-breakage entries in docs/issues/ in last 7d:**
- `cutoff=$(date -u -d '7 days ago' +%Y-%m-%d 2>/dev/null || date -u -v -7d +%Y-%m-%d)` (handle GNU vs BSD date).
- Find files in `docs/issues/` modified after the cutoff that mention any of: `EnglishPlayer`, `english player`, `english tab`, `animepahe`, `gogoanime`, `anitaku`, `scraper`. Use:
  ```bash
  matches=$(find docs/issues/ -type f -name '*.md' -newermt "$cutoff" 2>/dev/null | \
    xargs -r grep -lEi 'EnglishPlayer|english player|english tab|animepahe|gogoanime|anitaku|scraper' 2>/dev/null || true)
  ```
- Exclude files whose path matches `docs/issues/ui-audit-*` (audit reports are not breakage reports). Use a follow-up `grep -v 'ui-audit-'`.
- If `matches` non-empty: `[FAIL] Gate 4 (docs_issues): potential player breakage entries in last 7d: <files>`. Add to `failed`.
- Otherwise `[PASS] Gate 4 (docs_issues): no new player-breakage entries in last 7d`.

**Final report and exit:**
```bash
echo ""
echo "=========================================="
echo "Phase 20 Cutover Pre-flight Guardrail"
echo "=========================================="
if [[ ${#failed[@]} -eq 0 ]]; then
  echo "[PASS] All 4 gates satisfied. Cutover may proceed."
  exit 0
else
  echo "[FAIL] guardrail not met; earliest ship: $EARLIEST_SHIP"
  echo "Failed gates: ${failed[*]}"
  echo "Do NOT run Plans 20-02..20-05 until this script exits 0."
  exit 1
fi
```

**Make executable:** `chmod +x scripts/cutover-preflight.sh` (the task's verify step will assert the executable bit).

**Do NOT** add the script to .gitignore. **Do** add a header comment block documenting the four gates, the EARLIEST_SHIP rationale, and how to override `PROM_URL` for testing.
