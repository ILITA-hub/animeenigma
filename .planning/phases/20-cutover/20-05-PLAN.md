**Step 0 (mandatory gate):** Run `bash scripts/cutover-preflight.sh`. If exit code is non-zero, ABORT.

**Step 1 — Create the script:**
Write `scripts/cutover-purge-redis.sh` as an executable Bash script with `set -euo pipefail` and the shebang `#!/usr/bin/env bash`.

Script behavior:
1. Define the eight key patterns to purge:
   ```bash
   PATTERNS=(
     "search:hianime:*"
     "search:consumet:*"
     "stream:hianime:*"
     "stream:consumet:*"
     "episodes:hianime:*"
     "episodes:consumet:*"
     "hianime:*"
     "consumet:*"
   )
   ```
2. Resolve the redis container name. Prefer the docker-compose service if available:
   ```bash
   if command -v docker >/dev/null 2>&1; then
     CONTAINER=$(docker ps --format '{{.Names}}' | grep -E '^animeenigma-redis$' | head -1)
     if [[ -z "$CONTAINER" ]]; then
       CONTAINER=$(docker ps --format '{{.Names}}' | grep -i redis | head -1)
     fi
   fi
   if [[ -z "${CONTAINER:-}" ]]; then
     echo "[FAIL] cannot locate a running redis container — is the stack up?" >&2
     exit 1
   fi
   echo "[INFO] Using redis container: $CONTAINER"
   ```
3. For each pattern, use SCAN to enumerate keys and UNLINK to delete in batches. Pseudocode:
   ```bash
   for pattern in "${PATTERNS[@]}"; do
     echo "[INFO] Purging $pattern"
     total=0
     cursor=0
     while :; do
       resp=$(docker exec "$CONTAINER" redis-cli --no-raw SCAN "$cursor" MATCH "$pattern" COUNT 200)
       cursor=$(echo "$resp" | sed -n '1p' | tr -d '"')
       keys=$(echo "$resp" | sed '1d' | tr -d '"' | grep -v '^$' || true)
       if [[ -n "$keys" ]]; then
         count=$(echo "$keys" | wc -l)
         echo "$keys" | xargs -r docker exec -i "$CONTAINER" redis-cli UNLINK >/dev/null
         total=$((total + count))
       fi
       [[ "$cursor" == "0" ]] && break
     done
     echo "[DONE] $pattern: $total keys unlinked"
   done
   ```
   Robustness notes for the executor:
   - `redis-cli --scan --pattern <p>` would be simpler but its output streaming via `docker exec` can be fragile under high key counts; the explicit SCAN cursor loop above is bulletproof.
   - If `xargs -r` is unavailable (BSD), substitute with `[[ -n "$keys" ]] && echo "$keys" | docker exec -i "$CONTAINER" xargs redis-cli UNLINK`.
   - Use `redis-cli UNLINK` (non-blocking) not `DEL` — Redis 4.0+ ships UNLINK in the redis:7-alpine image.

4. After all patterns, run a final verification SCAN that should find ZERO matches in any of the eight namespaces:
   ```bash
   echo ""
   echo "[VERIFY] Post-purge SCAN over all patterns:"
   leftover=0
   for pattern in "${PATTERNS[@]}"; do
     hit=$(docker exec "$CONTAINER" redis-cli --scan --pattern "$pattern" | head -1 || true)
     if [[ -n "$hit" ]]; then
       echo "[FAIL] leftover key: $hit (pattern: $pattern)"
       leftover=$((leftover + 1))
     fi
   done
   if [[ $leftover -gt 0 ]]; then
     echo "[FAIL] $leftover leftover key(s) — purge incomplete"
     exit 1
   fi
   echo "[PASS] all dead namespaces empty"
   ```

5. Make the script executable: `chmod +x scripts/cutover-purge-redis.sh`. Idempotency: running the script a second time should report `0 keys unlinked` per pattern and exit 0.
