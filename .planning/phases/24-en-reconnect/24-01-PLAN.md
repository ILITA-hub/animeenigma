<objective>
Re-add the `"english": true` entry to the two player-service allow-list maps that gate which player IDs are accepted by the preference-save endpoint and the error-report endpoint. The 2026-05-18 cleanup (per CONTEXT.md risk inventory) likely stripped these entries when HiAnime/Consumet/EnglishPlayer.vue were deleted; confirmed via grep on 2026-05-19 — both maps currently contain only `"kodik"` and `"animelib"`. Without this fix, the restored EnglishPlayer in Wave 2 will 422 on preference save and report submit.

Purpose: Per SCRAPER-HEAL-18 acceptance, the EN tab must persist user preferences and accept error reports. The backend allow-list is the gate that decides whether a `player="english"` preference is accepted; without `english` in `ValidPlayers`, every save from the restored EnglishPlayer returns HTTP 422.

Output: Two-line edit to `preference.go` (add `"english": true` to ValidPlayers), two-line edit to `report.go` (add `"english": true` to allowedPlayerTypes), `make redeploy-player`, and curl smoke confirming both endpoints accept the new value.

This plan is intentionally backend-only. The frontend doesn't depend on the backend for the EN tab to render — it depends on it for save/report flows to succeed. Running this plan in Wave 1 (parallel with Wave 2's frontend restore) keeps the critical-path short.
</objective>
