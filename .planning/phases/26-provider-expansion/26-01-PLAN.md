<objective>
Create a self-contained `services/scraper/internal/providers/allanime/` package that implements `domain.Provider` against AllAnime's GraphQL persisted-query API (api.allanime.day) for the EN sub catalog, and register it in the scraper's orchestrator failover chain after gogoanime/animepahe and before the animekai gated block. The catalog-side parser at `services/catalog/internal/parser/allanime/` is NOT moved, NOT refactored, NOT shared via a new lib — it is copy-with-adaptation per CONTEXT.md D1. The scraper-side client owns its own GraphQL queries, DTOs, decrypt routine, and cache helpers, using the scraper's `BaseHTTPClient` (not the catalog's HTTP client).

Purpose: SCRAPER-HEAL-25 — lift AllAnime into the scraper failover pool so the orchestrator has three live EN providers (gogoanime, animepahe, allanime) instead of two. With Phase 24's three-provider verification still pending operator-side, AllAnime is the safety net that survives a single-provider outage. The lift is a copy-with-adaptation because (a) the catalog-side raw-jp consumption diverges from the scraper-side EN consumption (different category, different server preferences), (b) the GraphQL queries change slowly enough that duplication is cheaper than a shared boundary, and (c) extracting to a shared lib creates a new package for the sake of three GraphQL strings.

Output: A new Go package under `services/scraper/internal/providers/allanime/` (client.go, dto.go, queries.go, decrypt.go, cache.go, doc.go, client_test.go, testdata/*.json), a registration block in `services/scraper/cmd/scraper-api/main.go` that splices allanime between animepahe and animekai in the candidate-providers slice, and verification that `make redeploy-scraper` deploys cleanly with allanime appearing in `/scraper/health`.
</objective>
