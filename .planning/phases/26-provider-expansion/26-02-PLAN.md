<objective>
Land the browse-filter activation that CONTEXT.md D5 explicitly deferred from Phase 24 to Phase 26: a `has_english` GORM column on the Anime model, a repo `SetHasEnglish` setter, a filter switch case mapping `english` → `has_english = true`, the handler-level whitelist entry, an opportunistic setter call from the catalog-side scraper-episodes resolver, and the frontend filter row + i18n keys. The filter goes live in the same wave as 26-01's AllAnime lift so the column starts populating immediately — within ~7 days of ship, opportunistic backfill from real user traffic produces enough true rows to make the filter useful.

Purpose: SCRAPER-HEAL-25's "browse filter activation" sub-deliverable (per CONTEXT.md D5). Without this, the operator's "show me only anime with at least one English source" use-case has no UI. The on-demand catalog pattern (CLAUDE.md) means we don't pre-populate — the column fills lazily as users (or the maintenance bot's smoke runs) touch the EN tab. Acceptable for v3.1 per CONTEXT.md "Risks": the slow-populate risk is documented; backfill cron is a v3.2 polish item.

Output: Five backend file changes (domain + repo + handler + service + auto-migration), three frontend file changes (composable + sidebar + three locales), and verification that the column populates after a single curl hit and the filter narrows the listing correctly.
</objective>
