# Plan 27-02-PLAN.md: Parser Rewrite + UUID-Session Contract + Sidecar Transport

**Plan ID:** 27-02-PLAN

## Goal

The Go parser at `services/scraper/internal/providers/animepahe/` is rewritten so that the entire upstream-fetch transport routes through a new `resolverClient` HTTP client to the sidecar (`SCRAPER_ANIMEPAHE_RESOLVER_URL`); the API-contract migration to UUID `session` tokens (search → animeSession → release → episodeSession → play) is complete; `ddosguard.go` + `ddosguard_test.go` are deleted; MalSync gains single-strike invalidation on `/release` 404 (A9) backed by a persistent reverse-mapping cache key; existing tests are rewired to mock the resolver instead of upstream animepahe; `ANIMEPAHE_BASE_URL` env binding is removed from `config.go` and replaced with `SCRAPER_ANIMEPAHE_RESOLVER_URL`; three fresh Frieren goldens are captured against the deployed sidecar per D4.

`GetStream` is preserved semantically: it continues to hand the Kwik URL to `embeds.Registry.Find().Extract()` with the public animepahe origin as Referer. The only change inside `GetStream` is that the Referer string is sourced from a new package-level `kwikReferer` constant (`https://animepahe.pw/`, aligning with D2) instead of the now-deleted `p.baseURL` field.
