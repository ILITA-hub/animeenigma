### Task 2: Run the Frieren curl pipeline + capture verdict for all four stages

<action>
Run the exact curl pipeline from `docs/issues/scraper-provider-verification-2026-05-19.md` lines 38–57 verbatim, from `/data/animeenigma`. Set `BASE=http://localhost:8000` and `ANIME_ID=f0b40660-6627-4a59-8dcf-7ec8596b3623`. Sequence:

1. **Episodes** (animepahe explicit):
   ```bash
   RESP=$(curl -sS -w '\nHTTP=%{http_code} time=%{time_total}s' "$BASE/api/anime/$ANIME_ID/scraper/episodes?prefer=animepahe")
   echo "$RESP" | head -c 2000
   ```
   PASS gate: HTTP=200, response JSON parses with `data.episodes | length >= 28`. Capture an excerpt with the first 2 + last 1 episode for the verdict-log doc.

2. **Servers** (pick first episode session from the above response):
   ```bash
   EP=$(echo "$RESP" | jq -r '.data.episodes[0].id')
   curl -sS -w '\nHTTP=%{http_code} time=%{time_total}s' "$BASE/api/anime/$ANIME_ID/scraper/servers?episode=$EP&prefer=animepahe" | tee /tmp/27-04-servers.json | head -c 2000
   ```
   PASS gate: HTTP=200, `data.servers | length >= 1`, at least one entry with `name == "kwik"`.

3. **Stream** (pick first server, category sub):
   ```bash
   SRV=$(jq -r '.data.servers[0].id' /tmp/27-04-servers.json)
   curl -sS -w '\nHTTP=%{http_code} time=%{time_total}s' "$BASE/api/anime/$ANIME_ID/scraper/stream?episode=$EP&server=$(jq -rn --arg s "$SRV" '$s|@uri')&category=sub&prefer=animepahe" | tee /tmp/27-04-stream.json | head -c 2000
   ```
   PASS gate: HTTP=200, `data.sources | length >= 1`, `data.sources[0].url` ends in `.m3u8` (signed Kwik HLS URL).

4. **Stream fetchability** — verify the returned m3u8 actually loads with the Referer header from the response (`data.headers.Referer` should be `https://kwik.cx/` or similar):
   ```bash
   M3U8=$(jq -r '.data.sources[0].url' /tmp/27-04-stream.json)
   REF=$(jq -r '.data.headers.Referer // "https://kwik.cx/"' /tmp/27-04-stream.json)
   curl -sI -H "Referer: $REF" "$M3U8" | head -5
   ```
   PASS gate: response status `HTTP/2 200` (or 206). 4xx/5xx is a FAIL.

5. **Per-provider health gauge** — wait 30 seconds for the orchestrator's in-memory health cache to settle, then:
   ```bash
   curl -sS http://localhost:8088/scraper/health | jq '.providers.animepahe.stages'
   ```
   PASS gate: All four stages (`search`, `episodes`, `servers`, `stream`) report `up: true`. If `stream_segment` is also exposed, it does NOT count against this gate (it's owned by the probe runner — Plan 17 — not the provider). If after 5 minutes of waiting any of the four stages still reports `up: false`, mark the gate FAILED.

If ANY of the five gates FAILS, STOP and capture: (a) full response bodies, (b) `docker compose logs --tail=200 scraper animepahe-resolver` excerpt, (c) `/metrics` from the resolver (`docker exec animeenigma-animepahe-resolver wget -qO- http://localhost:3000/metrics | grep -E "stealth_|page_recycle_|upstream_403"`). Surface the failure to the operator as `## GATE BLOCKED: SCRAPER-HEAL-32 step <N>` with the captured diagnostics. Do NOT proceed to Task 3 until the gate is green.
</action>
