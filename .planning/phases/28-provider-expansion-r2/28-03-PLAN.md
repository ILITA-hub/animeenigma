    Key implementation points:
    (a) `var vidstreamVipHosts = []string{"am.vidstream.vip", "vidstream.vip"}` — case-insensitive comparison. Suffix-guard: `host == h || strings.HasSuffix(host, "."+h)`.
    (b) Set OUTGOING Referer header from `headers` arg (caller's choice, expected `https://animefever.cc/`); fall back to that constant if absent.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scraper service → am.vidstream.vip | Outbound HTTPS via embed extractor. Inline `sources` literal extracted via regex; no JS executed. Response untrusted. |
| extractor return → streaming proxy | Returned Stream URL host must be in `HLSProxyAllowedDomains` (added by Plan 28-02) or streaming returns 502. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-03-01 | SSRF | extracted m3u8 URL feeds streaming proxy | mitigate | HLS proxy fail-closed allowlist (Phase 25 SCRAPER-HEAL-24). Plan 28-02 adds `static-cdn-ca1.mofl.pro` to allowlist. |
| T-28-03-02 | Denial of Service | upstream returns multi-MB HTML | mitigate | `io.LimitReader(resp.Body, 2<<20)` (2 MiB) per BaseHTTPClient/embed convention. |
| T-28-03-03 | Tampering | malformed `sources: [{...}]` literal | mitigate | json.Unmarshal validates shape; non-absolute URL rejected; regex anchored to known prefix. |
| T-28-03-04 | Malicious Code | no JS execution path | mitigate | Plain regex; goja explicitly not used. |
| T-28-03-05 | Spoofing | suffix attack (`am.vidstream.vip.evil.com`) | mitigate | Matches() uses `host == h || strings.HasSuffix(host, "."+h)` — guards against suffix injection. Test 3 covers this. |
</threat_model>
