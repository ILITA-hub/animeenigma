## 2. Architecture

| # | Decision | Status | Rationale |
|---|---|---|---|
| 2.1 | **New `services/scraper/` microservice for the orchestrator + providers + embed registry + observability, called from a thin `services/catalog/internal/parser/scraper/client.go` HTTP wrapper.** Catalog stays the keeper of the external API surface (`/api/anime/{id}/scraper/*`) and UUID↔MAL ID translation; scraper does all the actual scraping. | 🔒 LOCKED (revised 2026-05-11 from "extend catalog") | Operational isolation: scraper iterates often (provider sites change), catalog is stable. Mirrors the existing thin-client pattern catalog already uses for `aniwatch`, `consumet-api`, `megacloud-extractor`. Catalog grows by ~150 LOC of HTTP client, not ~3,000 LOC of orchestrator. Gateway routing unchanged. |
| 2.2 | **Universal abstraction is at the EmbedExtractor + BaseHTTPClient + Provider interface layer** — NOT shared HTML scraping | 🔒 LOCKED | Empirically validated 2026-05-11: AnimePahe / 9anime / AnimeKai do NOT share HTML markup. They're independent codebases. |
| 2.3 | **Built from scratch in Go** — no wrapping of `@consumet/extensions`, `aniwatch` npm, or any other third-party scraper library | 🔒 LOCKED | The dependencies-that-died problem (aniwatch repo deleted, consumet enc-dec.app contract change) is the exact pattern we're escaping. |
| 2.4 | **Keep `docker/megacloud-extractor/` Node sidecar** for embed decryption | 🔒 LOCKED | It works today, decryption keys rotate at the upstream, JS-permissive runtime earns its keep here. Go layer calls it via HTTP. |
| 2.5 | **AnimeKai's MegaUp token generator runs inside our own megacloud-extractor sidecar** — extended in Phase 19 | 🔒 LOCKED | The `enc-dec.app` contract change killed Consumet. v3.0 introduces zero new external decryption dependencies. |
| 2.6 | **Sequential per-anime failover only** — orchestrator tries providers one at a time | 🔒 LOCKED | Parallel fan-out trips per-host rate limits at 10 users × 3 providers in seconds (PITFALLS §6). |
| 2.7 | **Provider clients never know about each other** — no in-client `FallbackProviders` list | 🔒 LOCKED | The Consumet anti-pattern we're killing. Failover lives in the orchestrator. |
| 2.8 | **Stream DTO has no `iframe_url` field at the Go type level** | 🔒 LOCKED | Type-level prevention of the Kodik silent-fallback trap (already shipped that bug twice — ISS-008, the `9347143` removal). |
| 2.9 | **Cache resolution work, not stream URLs** — stream cache TTL ≤ min(parsed expiry − 30 s, 5 min) | 🔒 LOCKED | Kwik/Megacloud URLs sign with short TTLs; caching too long causes 403s. ISS-001 already happened. |
| 2.10 | **HLS streams flow through the existing `libs/videoutils/proxy.go`** for CORS | 🔒 LOCKED | Allowed-domains list extended per phase; no new CORS proxy. |

## 8. Anti-Features (explicit non-goals)

🔒 **All LOCKED** — the executor must refuse to implement these and stop+ask if a task seems to require them:

- ❌ Preserving HiAnime / Consumet frontend identities
- ❌ Parallel fan-out across providers
- ❌ Headless browsers (Puppeteer, Playwright, chromedp, go-rod) for scraping
- ❌ TLS fingerprint spoofing (utls, tls-client)
- ❌ Proxy rotation / residential proxies
- ❌ Iframe-only EN streams (kills SubtitleOverlay + quality switching — same UX dead-end as Kodik)
- ❌ Cross-tier fallback EN → Kodik iframe (the trap we just escaped, twice)
- ❌ Pre-populating provider catalogs
- ❌ Auto-pick "first with results" ignoring per-anime preference (`feedback_watch_preferences.md`)
- ❌ Custom AES keys baked in from training data (the failure mode that killed aniwatch)
- ❌ Any new external decryption service dependency (no `enc-dec.app`, no equivalent)
- ❌ A "Zoro-family" shared HTML parser (empirically validated as nonexistent)

## 9. Auto-Mode Boundaries — When the Executor Must Stop & Ask

The autonomous executor proceeds without interruption for routine decisions inside the LOCKED scope. It MUST stop and request user input in any of the following situations:

1. **Phase 19 R&D outcome decision** — when the in-house AnimeKai token generator is implementable vs. has to carry over to v3.1. The executor proposes a recommendation; user decides.
2. **A provider site goes dead mid-implementation** — e.g. `9anime.org.lv` shuts down during Phase 18. Executor reports + proposes options (substitute / drop / defer).
3. **Discovery of a previously-unknown external decryption dependency** — if any provider's stream extraction can ONLY work via an external service. Decision 2.5 applies — user reapproves or scopes-out.
4. **Site requires headless / TLS spoofing / proxy rotation to scrape** — anti-bot scope creep is a 🔒 LOCKED non-goal. Stop, report, ask.
5. **Cutover soak fails** — if the 7-day post-Phase-19 traffic shows per-provider error > 5 % or alerts fire, Phase 20 cannot proceed automatically.
6. **Test failure that can't be explained by upstream-change** — if golden files diff in ways inconsistent with a benign markup tweak, escalate before "fixing" the test.
7. **Scope expansion request from within research** — if mid-phase research surfaces "we need a 4th provider" or "AniZone is reachable now," do not silently expand. Report and ask.

For everything else inside the LOCKED scope — implementation choices, test scaffolding, refactors, library minor-version bumps, docs writeups — the executor proceeds.
