## Root Cause (single mechanism, two layers)

1. **FE serialization:** `frontend/web/src/analytics/index.ts:76` тАФ `track(name, props)` always nests `props` under `properties`. The collector (`collect.go` `wireEvent`) reads `source`/`trace_id`/`operation`/`target`/`target_kind` only as top-level fields тЖТ they arrive empty. Fix: promote those keys to top-level on the wire (FE side) OR have `collect.go` read them out of `properties` (BE side).
2. **BE trace_id propagation:** the `libs/tracing` Effect chain has no `TraceID` field and never extracts the OTel span's trace id. Fix: add `TraceID` to `Effect` + `wireProducerEffect`, populate it from `trace.SpanFromContext(req.Context()).SpanContext().TraceID()` in `recordingTransport`, and set `domain.Event.TraceID` in `effects.go`.

Unit tests pass because each layer is tested in isolation with mocks across the serialization boundary; the live gate that would have caught it (Phase 04-04) was deferred.
