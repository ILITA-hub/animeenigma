behind their exact existing prop/event/v-model/slot surfaces. These are the two with real
behavioral deltas (portal + popper positioning for Select; focus-trap + portal +
scroll-lock for Dialog), so each ships with an explicit in-browser gate the orchestrator
must run. Rendering must be VISUALLY IDENTICAL; only the underlying primitive changes.

filename `Modal.vue` and re-skin in place; the barrel exports BOTH `Modal` (existing) AND a
new `Dialog` alias pointing at the same file. NO codemod, NO consumer edits — `vue-tsc`
proves the ~5 Modal + 2 Select consumers still compile.

Purpose: Move the portal/focus primitives onto Reka while holding the public API and the
rendered output constant, with scroll-lock + SubtitleOverlay-teleport regressions explicitly
gated in-browser (jsdom can't catch them — DS-NF-06).

Output: re-skinned `Select.vue` + `Modal.vue` on Reka, two specs, barrel exports `Dialog` alias.

Requirements: DS-LIB-05 (Select + Modal→Dialog swap behind same paths), DS-NF-04 (preserve API + alias rename).
