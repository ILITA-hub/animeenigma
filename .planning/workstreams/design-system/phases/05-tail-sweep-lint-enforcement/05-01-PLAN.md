<mapping_table>
The LOCKED DS-MIGRATE-02 off-palette → semantic-token mapping (from Phase 4, verbatim):
- red       → destructive
- amber / yellow → warning
- emerald / green → success
- blue / sky → info
- purple / violet → brand-violet
- gray / slate / zinc → muted / card / border / popover (context-dependent:
    a flat surface → card; an elevated menu/popover surface → popover; a hairline → border;
    secondary text → muted-foreground). `slate-900/95`/`zinc-900/95` elevated menus → `popover`.
Opacity modifiers (`/10 /20 /30 /50 /70 /80 /90`) are KEPT on the base semantic token when no
`-soft` token matches that exact alpha (Phase 4 precedent).
DEPRECATED-ALIAS repoints (DS-MIGRATE-04) — repoint the var NAME, do NOT flip the definition:
- var(--ink)   → var(--foreground)
- var(--ink-3) → var(--muted-foreground)
- var(--accent)→ var(--brand-cyan)
- var(--pink)  → var(--brand-pink)
DO NOT touch the literal aliases that have no canonical equivalent: `--ink-2`, `--ink-4`,
`--accent-soft`, `--accent-line`, `--accent-glow`, `--pink-soft`.
CYAN and PINK Tailwind utility classes (`bg-cyan-500`, `text-pink-400`, `text-orange-400`, etc.)
are BRAND/provider-identity colors — NOT off-palette. Leave them exactly as-is (Phase 4 rule).
`var(--token, #fallback)` patterns: repoint the token name; the `#fallback` after the comma is a
canonical-fine fallback (e.g. `var(--accent,#00d4ff)` → `var(--brand-cyan,#00d4ff)`).
</mapping_table>
