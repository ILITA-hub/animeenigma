<gate_rules>
The gate enforces EXACTLY these 3 rules over `frontend/web/src/**/*.vue` (excluding `*.spec.ts`/`__tests__`).
These ARE the field-proven Phase-4 greps — reuse them verbatim:

RULE 1 — Zero off-palette Tailwind color classes (ERROR). Pattern (Phase-4 verbatim):
  (text|bg|border|ring|from|to|via|fill|stroke|placeholder|divide|outline|decoration|shadow)-(red|amber|yellow|emerald|green|blue|sky|purple|violet|gray|slate|zinc)-(50|100|200|300|400|500|600|700|800|900)
  NOTE: `cyan`, `pink`, `orange`, `rose`, `indigo`, `teal`, `lime` are NOT in this set — cyan/pink are
  the Neon-Tokyo BRAND primitives and orange/rose are per-provider identity hues (Kodik cyan, AniLib
  orange, Hanime pink, Raw rose). Including them would fail the clean tree (Anime.vue, players, Navbar all
  legitimately use them). This brand-exemption rationale is documented in DESIGN-SYSTEM.md (DS-GOV-02).

RULE 2 — Zero hardcoded hex outside the allowlist (ERROR). Pattern:
  raw `#[0-9a-fA-F]{3,8}` literals in `.vue` (both `<style>` raw hex and Tailwind arbitrary `*-[#...]`),
  EXCLUDING any `path:hex` pair present in `scripts/design-system-allowlist.txt`. The allowlist match is
  per-(file,hex): a hex is allowed only in the file(s) that list it. (Implementation: for each offending
  `file:hex`, skip if a non-comment allowlist line contains both that file path and that hex.)

RULE 3 — Zero deprecated brand-alias usages (ERROR). Pattern:
  var(--ink) | var(--accent) | var(--pink)  (word-boundary), EXCLUDING the literal-alias survivors
  `--ink-2`, `--ink-4`, `--accent-soft`, `--accent-line`, `--accent-glow`, `--pink-soft`.
  After Wave-2, `var(--accent)` brand usage is itself a violation (the flip made --accent a hover surface).

OUT OF LINT SCOPE (Phase-6 governance docs only — do NOT add these rules): use-the-primitives, font-weight
scale, padding scale, cva variant usage, hex-in-`<style>`-beyond-allowlist niceties, arbitrary `text-[#...]`
beyond rule 2. The user chose the narrower roadmap scope; deferred items stay deferred.
</gate_rules>

<i18n_lint_pattern>

`SCRIPT_DIR`, RED/YELLOW/GREEN/NC colors, `ERRORS`/`WARNINGS` counters, per-rule `=== ... ===` section
headers, `OK`/`ERROR` lines, a `=== Summary ===` block, and `exit 1` when `ERRORS > 0` else `exit 0`.

`echo`/`cat` under pipefail; collect into temp files / use `grep -c ... || true`).
