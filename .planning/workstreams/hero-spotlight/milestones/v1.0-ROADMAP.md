## Outcome

Replaced the top-of-home `trendingRecs` row with a rotating 9-card hero
spotlight carousel. Eligibility-aware: anonymous visitors see up to 6
cards; logged-in users see up to 9.

**Live at:** https://animeenigma.ru/

## Key Accomplishments

1. **New backend aggregator** — `services/catalog/internal/service/spotlight/`
   with concurrent fan-out (`sync.WaitGroup + buffered chan`, NOT
   errgroup), 800ms per-card deadline, 2s overall budget, manual Redis
   cache discipline (`cache.Get` + `errors.Is(err, cache.ErrNotFound)`),
   eligibility filter, snapshot fallback.
2. **9 card resolvers** — `anime_of_day`, `random_tail`, `latest_news`,
   `platform_stats` (Phase 1) + `personal_pick`, `telegram_news`,
   `now_watching`, `not_time_yet`, `continue_watching_new` (Phase 3).
   Adaptive 1-2-3 layout rule for multi-item resolvers.
3. **Vue 3 carousel** — `HeroSpotlightBlock.vue` + 9 card SFCs +
   `useSpotlight.ts` composable + i18n parity (en/ru/ja) + Playwright
   e2e + axe-core 0 violations.
4. **Player internal endpoint** — `GET /internal/users/{user_id}/list?status=...`,
   OUTSIDE `/api` group, Docker-network-only (gateway defense-in-depth
   test pins NOT proxied).
5. **3 deliberate architectural divergences** baked into acceptance
   gates: manual cache discipline (NOT GetOrSet), bare JSON envelope
   (NOT httputil.OK), bare 404 (NOT httputil.NotFound).
6. **CLAUDE.md docs** — new "Adding a Spotlight Card Type" section under
   Common Tasks (HSB-NF-05).
