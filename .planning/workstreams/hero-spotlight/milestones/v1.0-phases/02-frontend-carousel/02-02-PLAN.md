<objective>
Build `CarouselControls.vue` — a stateless presentational component that renders the carousel's interactive chrome (prev chevron, next chevron, dot indicators). It is intentionally dumb: receives `currentIndex` + `cardCount` as props, emits `prev`, `next`, `goto(index)` events. Plan 03's `HeroSpotlightBlock.vue` owns the state machine and consumes these events.

Purpose: Decoupling the chrome from the state machine lets Plan 03's tests focus on state transitions, and lets Plan 02's tests focus on a11y + emit correctness — neither becomes a kitchen sink. Phase 3 can also reuse `CarouselControls` if/when other carousels appear.

- `frontend/web/src/components/home/spotlight/CarouselControls.spec.ts` — Vitest + @vue/test-utils
</objective>
