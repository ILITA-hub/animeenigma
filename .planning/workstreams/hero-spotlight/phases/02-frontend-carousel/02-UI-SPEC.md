# Phase 2 — UI Design Contract: HeroSpotlightBlock + Carousel

> Visual and interaction contract for the v1.0 HeroSpotlightBlock (4 static cards: `anime_of_day`, `random_tail`, `latest_news`, `platform_stats`). Generated by gsd-ui-researcher from approved design doc + CONTEXT.md + REQUIREMENTS.md (autonomous mode). Verified by gsd-ui-checker.

---

## Design System

AnimeEnigma uses no component framework (no shadcn / no Radix). The visual system is the **"Neon Tokyo"** Tailwind v4 setup defined in `frontend/web/src/styles/main.css` (lines 1–319) plus utility classes. The spotlight block reuses these tokens — no new tokens, no new CSS keyframes, no new utility classes are introduced. Card chrome borrows from `CollectionsRow.vue` / `ContinueWatchingRow.vue` / the existing `trendingRecs` markup in `Home.vue`.

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 + custom CSS layer via `@theme` in `main.css`) |
| Preset | not applicable |
| Component library | none (pure Vue 3 SFCs, Composition API + `<script setup lang="ts">`) |
| Icon library | inline SVG (existing convention — see `Home.vue` schedule link, `ContinueWatchingRow.vue` star icon) |
| Font | `Inter, 'Noto Sans JP', system-ui, sans-serif` (declared as `--font-sans` in `main.css:18`) |
| Theme | dark only ("Neon Tokyo") — base `#121218`, surface `#1a1a24`, accent cyan `#00d4ff`, pink `#ff2d7c` |

## Spacing Scale

Block + card layout uses the standard Tailwind 4-multiple scale (1 unit = 4px). All values below are tokens already present in Tailwind defaults.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between meta chips (`gap-1`) |
| sm | 8px | Dot-indicator gap (`gap-2`); CTA icon→label gap |
| md | 16px | Default card-content inner padding (`p-4`) |
| lg | 24px | Card outer padding desktop (`p-6`); section vertical margin (`mb-6`) |
| xl | 32px | Block bottom margin from next section (`mb-8`) |
| 2xl | 48px | Reserved — not used in this phase |
| 3xl | 64px | Reserved — not used in this phase |

**Specific block geometry:**

| Property | Desktop (`lg:`) | Tablet (`md:`) | Mobile (default) |
|----------|-----------------|----------------|------------------|
| Outer container | `max-w-7xl mx-auto px-4 lg:px-8` | same | `px-4` |
| Block bottom margin | `mb-8` | `mb-8` | `mb-8` |
| Block min-height | `min-h-[320px]` | `min-h-[340px]` | `min-h-[400px]` |
| Block max-height | `lg:max-h-[360px]` | none | none |
| Card inner padding | `p-6` | `p-4` | `p-4` |
| Card border-radius | `rounded-2xl` (16px) | same | same |

Exceptions:
- Touch targets (chevron buttons, dot buttons) use the project's `.touch-target` class (`min-h-[44px] min-w-[44px]`, declared in `main.css:236`) on mobile to satisfy WCAG 2.5.5 tap-target sizing. Visually they are smaller (chevrons ~40px, dots 8px), but their hit area is padded out.
- Card vertical content gap: `space-y-3` (12px) — a 4-multiple exception confirmed against `ContinueWatchingRow.vue`'s 12px gap between poster and title.
- 12px sub-step (`space-y-3`, `gap-3`, `bottom-3`) is permitted as a sub-step for tight inline groupings — precedent in `ContinueWatchingRow.vue`. Used by the dot-indicator `bottom-3` offset and the card content `gap-3` / `space-y-3` rhythm.

## Typography

The block uses exactly **4 sizes × 2 weights** drawn from existing Tailwind defaults. No new font-size custom properties are added.

| Role | Tailwind class | Computed px | Weight | Line height | Usage |
|------|----------------|-------------|--------|-------------|-------|
| Display | `text-2xl md:text-3xl` | 24px → 30px | `font-semibold` (600) | `leading-tight` (1.25) | Card headline (anime title in `AnimeOfDayCard` / `RandomTailCard`) |
| Heading | `text-lg md:text-xl` | 18px → 20px | `font-semibold` (600) | `leading-snug` (1.375) | Card section title ("Сегодня в фокусе", "Что нового", "Случайная находка", "Платформа за неделю") |
| Body | `text-sm md:text-base` | 14px → 16px | `font-medium` (500) | `leading-normal` (1.5) | Meta lines, changelog excerpts, news entry titles |
| Label / caption | `text-xs` | 12px | `font-medium` (500) | `leading-normal` (1.5) | Genre chips, score chip, "+ N" delta indicators, dot button aria-labels (visually `sr-only`), metric unit labels |

Hard rules:
- Only **two weight values** in use across the block: 500 (`font-medium`) for body / chips / labels, and 600 (`font-semibold`) for headings, the card display title, and CTAs.
- Anime titles use the project's localization helper `getLocalizedTitle(name, name_ru, name_jp)` from `@/utils/title` (precedent: `ContinueWatchingRow.vue:51`).
- Numerals in the stats card (`anime_added_7d`, etc.) use `tabular-nums` so transitioning counts do not jiggle column width.

## Color

Strictly reuses the Neon Tokyo palette from `main.css`. No new colors are introduced.

| Role | Value | Tailwind class / CSS var | Usage |
|------|-------|--------------------------|-------|
| Dominant (60%) — block surface | `rgba(255,255,255,0.05)` over `#121218` page bg | `.glass-card` (defined `main.css:110`) | Spotlight wrapper background |
| Secondary (30%) — interactive chrome | `rgba(255,255,255,0.10)` | `bg-white/10` (skeletons, dot inactive, chevron idle) | Skeleton blocks, inactive dot indicators, idle chevron buttons |
| Accent (10%) — cyan | `#00d4ff` (`--color-cyan-400`) / `#00b8e6` (`--color-cyan-500`) | `text-cyan-400`, `bg-cyan-500`, `border-cyan-400`, `ring-cyan-400` | Active dot, focus ring, primary CTA "Watch", hover state for links, score-chip border accent |
| Accent secondary (rare) — pink | `#ff2d7c` (`--color-pink-500`) | not used in Phase 2 | Reserved for Phase 3's `now_watching` "live" indicator — DO NOT use in Phase 2 |
| Destructive | `#ef4444` (Tailwind `red-500`) | not used in Phase 2 | No destructive actions exist in this phase |
| Success / live | `#00ff9d` (`--color-success`) | not used in Phase 2 | Reserved for Phase 3 |
| Warning | `#ffd600` (`--color-warning`) / Tailwind `yellow-400` | `text-yellow-400` | Score-chip star color (matches existing `trendingRecs` pattern in `Home.vue:93`) |

**Accent reserved for** (explicit list — never "all interactive elements"):
1. **Active dot indicator** — `bg-cyan-400` + `aria-current="true"`
2. **Focus ring** on chevrons, dots, CTAs — via `:focus-visible` global (`main.css:91`), which paints `box-shadow: 0 0 0 2px var(--color-cyan-400)`
3. **Primary CTA button** in `AnimeOfDayCard` / `RandomTailCard` ("Watch" / "Смотреть") — uses `.btn-primary` class from `main.css:162`
4. **Hover state** on internal links: `hover:text-cyan-400` (precedent: `ContinueWatchingRow.vue:51`)
5. **Score-chip star icon color** — `text-yellow-400` only (NOT cyan)

**60/30/10 verification**:
- 60% — base `#121218` page bg + glass-card `rgba(255,255,255,0.05)` surface
- 30% — white/10 utility chrome (skeletons, dot rest, chevron rest, divider lines)
- 10% — cyan-400 for the four reserved cases above + yellow-400 for score chip only

Card-specific color rules:

| Card | Headline accent | Body |
|------|-----------------|------|
| `AnimeOfDayCard` | `text-white` title; cyan-400 only on hover + score yellow | white meta on `glass-card` |
| `RandomTailCard` | `text-white` title; cyan-400 only on hover; `text-cyan-300/80` (`text-cyan-300` at 80% opacity) for the "Случайная находка" eyebrow label so it visually differentiates from `AnimeOfDay` without exceeding the 10% cyan budget |
| `LatestNewsCard` | `text-white` per-entry title; `text-gray-400` body; cyan-400 only on per-entry hover and on the "Read more →" footer link |
| `PlatformStatsCard` | `text-white` metric values; `text-gray-400` metric labels; delta indicators: `text-cyan-400` for positive delta (+N), `text-gray-400` for zero or null delta. No green/red — keeps the 10% cyan rule clean and avoids overloading the palette in Phase 2 |

## Visual Contract — Exact Tailwind Classes

### Block wrapper (`HeroSpotlightBlock.vue` root)

```html
<section
  v-if="enabled && cards.length > 0"
  ref="rootRef"
  role="region"
  :aria-roledescription="'carousel'"
  :aria-label="t('spotlight.regionLabel')"
  class="px-4 lg:px-8 max-w-7xl mx-auto mb-8 mt-8"
  tabindex="0"
  @mouseenter="stopCycle"
  @mouseleave="startCycle"
  @focusin="stopCycle"
  @focusout="startCycle"
  @keydown.left="prev"
  @keydown.right="next"
>
  <div class="relative glass-card rounded-2xl overflow-hidden min-h-[400px] md:min-h-[340px] lg:min-h-[320px] lg:max-h-[360px]">
    <!-- Slide region (aria-live) -->
    <div
      class="relative w-full h-full"
      aria-live="polite"
      aria-atomic="true"
    >
      <!-- Active slide (one of the 4 card components) -->
    </div>
    <!-- CarouselControls.vue mounts here, absolutely positioned -->
  </div>
</section>
```

### Slide cross-fade transition

```html
<transition
  :name="reducedMotion ? 'none' : 'spotlight-fade'"
  mode="out-in"
>
  <component :is="cardFor(active.type)" :key="active.id" :data="active.data" />
</transition>
```

Custom CSS appended to `main.css` (NEW additions for this phase — minimal, ~12 lines):

```css
/* Spotlight slide cross-fade. Honored only when no reduced-motion override. */
.spotlight-fade-enter-active,
.spotlight-fade-leave-active {
  transition: opacity 0.4s ease;
}
.spotlight-fade-enter-from,
.spotlight-fade-leave-to {
  opacity: 0;
}

/* prefers-reduced-motion override already global at main.css:311; the
   "none" transition name short-circuits Vue's <transition> entirely so
   no opacity animation runs. No additional CSS needed for that branch. */
```

### `CarouselControls.vue` — chevrons + dots

```html
<!-- Chevron PREV -->
<button
  type="button"
  class="absolute left-2 top-1/2 -translate-y-1/2 z-10
         w-10 h-10 touch-target
         flex items-center justify-center
         rounded-full bg-white/10 backdrop-blur-sm
         text-white hover:bg-cyan-500/20 hover:text-cyan-400
         transition-colors
         focus:outline-none"
  :aria-label="t('spotlight.prevSlide')"
  @click="$emit('prev')"
>
  <svg class="w-5 h-5" fill="none" stroke="currentColor" viewBox="0 0 24 24">
    <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M15 19l-7-7 7-7" />
  </svg>
</button>

<!-- Chevron NEXT — mirror of above, right-2, d="M9 5l7 7-7 7" -->

<!-- Dots -->
<div
  class="absolute bottom-3 left-1/2 -translate-x-1/2 z-10
         flex items-center gap-2"
  role="tablist"
>
  <button
    v-for="(card, i) in cards"
    :key="card.id ?? i"
    type="button"
    class="touch-target w-2 h-2 rounded-full transition-colors"
    :class="i === currentIndex
      ? 'bg-cyan-400'
      : 'bg-white/30 hover:bg-white/50'"
    :aria-label="t('spotlight.goToSlide', { n: i + 1 })"
    :aria-current="i === currentIndex ? 'true' : 'false'"
    @click="$emit('goto', i)"
  />
</div>
```

> Note: the dot button is visually 8px (`w-2 h-2`) but its **hit area** is enlarged to 44×44px via the `.touch-target` utility (min-height/min-width) plus a `padding` inset that doesn't change the visible dot. This satisfies WCAG 2.5.5 without compromising the visual density. The `bottom-3` (12px) offset is the documented 12px spacing sub-step exception.

## Interaction Contract

### Auto-cycle state machine (in `HeroSpotlightBlock.vue`)

| State | Transition | Trigger |
|-------|-----------|---------|
| `initial` (loading) | → `loaded` | `useSpotlight()` returns `loading: false` AND `cards.length > 0` |
| `initial` | → `hidden` (no-op) | `cards.length === 0` OR fetch error (5xx / network) |
| `loaded` | → `cycling` | `onMounted` after hydration if `!reducedMotion` |
| `cycling` | → `paused` | `mouseenter` OR `focusin` on root `<section>` |
| `paused` | → `cycling` | `mouseleave` AND `focusout` resolved (no element inside still focused) |
| `cycling` / `paused` | → `cycling` (index+1) | every 7000 ms while in `cycling` |
| any | → `seek(i)` | dot button click OR chevron click OR `ArrowLeft` / `ArrowRight` keydown on root |
| `cycling` | → `cycling` (after manual seek) | seek resets the 7000ms interval (clearInterval + setInterval) |
| any | → `cleanup` | `onBeforeUnmount` clears interval |

**Exact timing values:**

| Variable | Value | Rationale |
|----------|-------|-----------|
| `AUTO_CYCLE_INTERVAL_MS` | `7000` | Locked by HSB-FE-03 |
| `FADE_DURATION_MS` | `400` | CSS-side: `transition: opacity 0.4s ease` |
| Tap-target min | `44 × 44` px | WCAG 2.5.5 |
| Keyboard focus-ring offset | `2px` (cyan-400) + `2px` (base) | Inherits global `:focus-visible` from `main.css:91` |

### Event handlers (composable surface)

```typescript
// useSpotlight.ts
export function useSpotlight() {
  const cards = ref<SpotlightCard[]>([])
  const loading = ref(true)
  const error = ref<Error | null>(null)
  async function refresh() { /* GET /api/home/spotlight */ }
  onMounted(refresh)
  return { cards, loading, error, refresh }
}

// inside HeroSpotlightBlock.vue setup()
const currentIndex = ref(0)
const intervalId = ref<ReturnType<typeof setInterval> | null>(null)
const reducedMotion = useReducedMotion()

function startCycle() {
  if (reducedMotion.value) return
  if (intervalId.value) return // already cycling
  intervalId.value = setInterval(() => {
    currentIndex.value = (currentIndex.value + 1) % cards.value.length
  }, 7000)
}

function stopCycle() {
  if (intervalId.value) {
    clearInterval(intervalId.value)
    intervalId.value = null
  }
}

function next() { currentIndex.value = (currentIndex.value + 1) % cards.value.length; restart() }
function prev() { currentIndex.value = (currentIndex.value - 1 + cards.value.length) % cards.value.length; restart() }
function goTo(i: number) { currentIndex.value = i; restart() }
function restart() { stopCycle(); startCycle() }

watch(() => cards.value.length, (n) => {
  if (n > 0) {
    currentIndex.value = Math.floor(Math.random() * n)
    startCycle()
  }
}, { immediate: false })

onBeforeUnmount(stopCycle)
```

### Reduced-motion handling

- `useReducedMotion()` returns a reactive `Ref<boolean>` driven by `window.matchMedia('(prefers-reduced-motion: reduce)')`. Listens for `change` events so toggling the OS-level setting at runtime is honored without a reload.
- When `reducedMotion.value === true`:
  - `startCycle()` is a no-op (auto-cycle disabled).
  - Slide transitions use `<transition name="none">` which Vue treats as no animation (no opacity fade between slides).
  - Manual navigation (chevrons, dots, arrow keys) still works fully.
- The global rule at `main.css:311` (`prefers-reduced-motion: reduce` zeros out all animation/transition durations) is the secondary safety net; the explicit `<transition name="none">` is the primary mechanism so we are not relying purely on global CSS.

### Keyboard contract

| Key | Action | Notes |
|-----|--------|-------|
| `Tab` | Move focus into the block | Block has `tabindex="0"` on root `<section>` |
| `Tab` (within block) | Cycle through: CTA buttons → chevron prev → dot 1..N → chevron next | DOM order matches this; no `tabindex` overrides needed |
| `Shift+Tab` | Reverse traversal | standard |
| `ArrowLeft` | `prev()` | Bound on root `<section>` via `@keydown.left` |
| `ArrowRight` | `next()` | Bound on root `<section>` via `@keydown.right` |
| `Enter` / `Space` on dot/chevron | Activate (same as click) | Native button behavior |
| `Esc` | No effect | Block is not modal; nothing to dismiss |

### Hover / focus pause semantics

- `mouseenter` ANYWHERE inside the `<section>` → pause.
- `focusin` ANYWHERE inside → pause.
- Pause does NOT reset the carousel index — it just suspends the interval timer.
- Resume on `mouseleave` (truly leaves the section) OR `focusout` (no descendant still focused). Vue's `@focusout` fires before the new focus target is known, so the handler uses a microtask delay (`requestAnimationFrame`) to check whether `document.activeElement` is still inside the section before resuming. This avoids flickering pause/resume during internal Tab navigation.

## State Contract

| State | Block visibility | Skeleton visible | Cards rendered | Trigger |
|-------|------------------|------------------|----------------|---------|
| `loading` | yes (skeleton only) | YES (single full-block shimmer) | no | `useSpotlight()` initial mount |
| `loaded (≥1 card)` | yes | no | yes | fetch resolved with `cards.length > 0` |
| `loaded (0 cards)` | **NO** (block does not render) | no | no | fetch resolved with empty `cards` array — silent self-hide |
| `error (5xx / network)` | **NO** (block does not render) | no | no | fetch threw or non-2xx response; emits `console.warn('[spotlight] fetch failed', error)` once |
| `feature-flag-off` | **NO** (block does not mount) | no | no | `import.meta.env.VITE_HERO_SPOTLIGHT_ENABLED === 'false'`; component short-circuits with `v-if="enabled"` |
| `endpoint disabled` (backend `SPOTLIGHT_ENABLED=false`) | **NO** | no | no | gateway returns 404 → fetch throws → silent self-hide (same code path as error state) |

State transitions are unidirectional within a single mount cycle. On route re-mount (e.g. user navigates away and returns to `/`), the composable re-fetches and the state machine resets. There is no client-side retry within a single mount in Phase 2 (Phase 3's live cards will introduce a 30s refresh — out of scope here).

## Accessibility Contract

The carousel must pass **axe-core with zero violations** (Phase 2 success criterion #9). The block follows the [APG carousel pattern (auto-rotating, tabs)](https://www.w3.org/WAI/ARIA/apg/patterns/carousel/examples/carousel-1-prev-next/).

### Image alts

- Anime posters: `alt={localizedTitle}` (NOT empty string — the poster IS the primary visual identifier of the anime; assistive tech users benefit from the title). Precedent: the existing `trendingRecs` row uses `alt=""` because the title text follows immediately and would create duplicate announcements; in the spotlight card the title is in a separate region so the redundancy concern doesn't apply equivalently. Decision: use the localized title for the spotlight cards.
- Skeleton block: wrapper has `aria-hidden="true"`.
- Decorative SVG icons inside CTAs / chevrons / score chip: `aria-hidden="true"`.

### Focus indicators

The project's `:focus-visible` rule (`main.css:91`) paints a 2px cyan-400 ring with a 2px base-color spacer, providing WCAG-compliant 3:1 contrast against both the glass-card surface and the page background. The block must NOT override this with `focus:outline-none` anywhere except as paired with an explicit `focus-visible:` replacement (chevrons / dots / CTAs use the global `:focus-visible` directly — see code samples above where `focus:outline-none` is present, paired with the global box-shadow).

### Color contrast (WCAG AA)

| Pair | Ratio | Status |
|------|-------|--------|
| white (#fff) on glass-card surface (~#26262d effective) | 14.0:1 | PASS AAA |
| gray-400 (#9ca3af) on glass-card | 6.4:1 | PASS AA |
| cyan-400 (#00d4ff) on glass-card | 7.8:1 | PASS AAA |
| yellow-400 score on `bg-black/70` | 13.2:1 | PASS AAA |
| cyan-400 focus ring on base (#121218) | 11.9:1 | PASS AAA |
| white CTA label on cyan-500 button (#00b8e6) | 4.6:1 | PASS AA (large text); `.btn-primary` uses `color: var(--color-base)` so the label is dark on cyan = 11.9:1 AAA |

### Screen reader announcement flow

1. User Tab-focuses the section → SR announces "Today's spotlight, carousel, region. 1 of 4: Frieren: Beyond Journey's End, slide."
2. Auto-advance (7s elapses) → `aria-live="polite"` fires → SR announces the new slide's `aria-label` ("2 of 4: What's new, slide").
3. User presses ArrowRight → manual next → SR announces the next slide's label.
4. User presses Tab → focus moves to first interactive control inside slide (CTA or chevron prev).

### Hard a11y constraints

- **No `aria-roledescription="carousel"` without `role="region"`** — both must coexist (axe rule `aria-allowed-attr`).
- **`aria-live="polite"` only on the slide container** — NOT on the whole section (would announce every focus change on every dot button).
- **Dot buttons MUST be `<button>` elements**, not `<div>` with click handlers — focusable + Enter/Space activation come for free.
- **Pause-on-focus is mandatory** — APG carousel pattern requires that keyboard users get the same hover-pause benefit so they can read the active slide without it advancing under them.

## Responsive Contract

Three breakpoint tiers. AnimeEnigma uses Tailwind v4 defaults: `sm: 640px`, `md: 768px`, `lg: 1024px`. The spotlight block targets:

| Tier | Range | Block height | Card layout strategy |
|------|-------|--------------|----------------------|
| Mobile | `< 768px` (default classes) | `min-h-[400px]` | Vertical stack within card; full-width content |
| Tablet | `768px ≤ w < 1024px` (`md:`) | `min-h-[340px]` | Horizontal split for poster cards; 3-col grid for multi-item cards |
| Desktop | `≥ 1024px` (`lg:`) | `min-h-[320px] max-h-[360px]` | Full horizontal layouts; larger padding (`p-6`) |

### Per-card responsive matrix

| Card | Desktop (`lg:` ≥ 1024px) | Tablet (`md:` 768–1023px) | Mobile (< 768px) |
|------|---------------------------|----------------------------|-------------------|
| **AnimeOfDayCard** | Poster left (`w-48`), meta right (`flex-1`). Eyebrow above title. CTAs row. Padding `p-6`. | Poster left (`w-40`), meta right. Padding `p-4` (16px — matches mobile to keep the standard 4/8/16/24 scale). | Poster top (centered, `w-32`), meta below (full width). Eyebrow shown above poster (mobile-only block). CTAs wrap to a column on very narrow widths. Padding `p-4`. |
| **RandomTailCard** | Same poster-left layout as AnimeOfDay; eyebrow uses `text-cyan-300/80` so it visually contrasts. Single "Open" CTA only. | Same. | Same as AnimeOfDay mobile (poster top, meta below) — single CTA. |
| **LatestNewsCard** | `<ul class="grid grid-cols-3 gap-4">` — 3 entries side by side. Header row has title + "Read full changelog →" link at right. | `grid-cols-3 gap-3` — slightly smaller gap. | `grid-cols-1` — entries stack vertically. Header row stays at top with title + link wrapping to a second line if needed. |
| **PlatformStatsCard** | `grid-cols-3` (when 3 metrics) — chips in a row. Value text size `text-4xl`. | `grid-cols-3` when 3 metrics, `grid-cols-2` when 2. | `grid-cols-1` always — chips stack vertically. Value text `text-3xl`. |

### Carousel-chrome responsive behavior

| Element | Desktop | Tablet | Mobile |
|---------|---------|--------|--------|
| Chevrons | Visible, 40×40 px, positioned `left-2` / `right-2` | Visible, 40×40 px | Visible BUT with reduced opacity `opacity-70 hover:opacity-100` to reduce visual clutter on small screens; hit area still 44×44 via `.touch-target` |
| Dots | Bottom-center, 8×8 px visual, 8px gap between | Same | Same — `bottom-3` (12px from bottom, see Spacing Scale exception) so they sit comfortably above the card's content padding |
| Tap → seek | OK | OK | OK |
| Swipe gestures | Not supported in Phase 2 | N/A | N/A — Phase 4+ enhancement if user demand arises |

### Mobile-specific design rules

1. **No "+ N more →" pattern in Phase 2** — that pattern is for `PersonalPickCard` (Phase 3 multi-poster card). The Phase 2 multi-item cards (`LatestNewsCard`) instead vertically stack all 3 entries on mobile, since text rows stack compactly without overwhelming the hero height.
2. **CTAs wrap, never truncate** — on narrow widths the "Watch" + "Add to list" pair flows to a column. Use `flex-wrap` not `overflow-hidden`.
3. **Posters always have visible aspect-ratio** — `aspect-[2/3]` is preserved across all breakpoints; mobile reduces width (`w-32`) not aspect ratio.
4. **Block content NEVER scrolls horizontally** on mobile — unlike `CollectionsRow` / `ContinueWatchingRow` which are horizontally scrollable rows, the spotlight is a single-slide hero. The whole carousel advances; the slide itself does not have an inner horizontal scrollbar.

## Registry Safety

The project does not use shadcn or any external component registry. Every UI primitive is either:
- A native HTML element styled with Tailwind v4 utility classes, OR
- A custom CSS class declared in `frontend/web/src/styles/main.css` under the `@theme` block (and visible to a project-wide diff), OR
- An inline SVG icon (no icon-library dependency).

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|

| third-party | none | not applicable — no third-party visual components introduced |
