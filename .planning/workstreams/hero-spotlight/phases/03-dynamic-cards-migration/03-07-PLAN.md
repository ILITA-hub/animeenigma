Final verification gate for Phase 3. After Plans 01-06 ship, this plan:

1. **Redeploys** the 3 affected services (catalog, player, web) via `make redeploy-*`.
2. **Smoke-tests** the full stack via a new `scripts/spotlight-phase3-smoke.sh` that curls `/api/home/spotlight` anonymously AND with the `ui_audit_bot` API key, asserting the card-count and card-type contracts.

5. **Gates on a human-verify checkpoint** for the visual regression — the auto-cycle, focus management, and login-gated card visibility need a human "looks right" before declaring Phase 3 done.

This plan is the ONLY checkpoint in Phase 3 (other plans are fully autonomous). After the checkpoint passes, the user runs `/animeenigma-after-update` per project convention.
