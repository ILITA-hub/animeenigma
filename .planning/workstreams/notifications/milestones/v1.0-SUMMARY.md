## Goal achieved

> A user with 3-8 ongoing shows in `anime_list.status = 'watching'` notices when a new episode drops without ever having to refresh the anime page. Within ~1 hour of upstream availability on the player and translation they were already using, a red badge appears on the header bell. If the tab is foregrounded, a toast slides in. One click → right episode on the right player with the right translation.

✅ Sensing, engine, UX, type-pluggability — all four DoD elements live in production.

## What shipped

### Backend

- **New microservice** `services/notifications/` on port **8090** (8087 was taken by host-native maintenance binary — same precedent as library v0.2 at 8089).
- **Two new tables** in shared `animeenigma` Postgres:
  - `user_notifications` with `(user_id, dedupe_key) WHERE dismissed_at IS NULL` partial unique index for UPSERT semantics
  - `parser_episode_snapshots` with `(anime_id, player, language, watch_type, translation_id)` composite unique for diff cache
- **6 public CRUD routes** behind gateway JWT auth: list / unread-count / read / mark-all-read / dismiss / click
- **1 internal producer endpoint** `POST /internal/notifications` (gateway-non-routing security model)
- **2 admin trigger endpoints** `/internal/detector/run-once` + `/internal/cleanup/run-once` for deterministic verification
- **Hourly cron detector** with ±5min boot jitter — DISTINCT hot-combo SQL → kodik/animelib parser fan-out via 5-worker errgroup with 10s per-call timeout → diff vs snapshot → batched UPSERT with first-fire/aggregation semantics. **Bootstrap protection** unit-tested.
- **Daily cleanup cron** at 03:30 with 30-day retention.
- **Catalog gains** `GET /internal/anime/{id}/episodes?player=&translation_id=&watch_type=&language=` with 5-min Redis cache. `kodik | animelib` only for v1.0; English players (scraper) deferred to v1.0.x.
- **6 Prometheus metrics** + structured per-run logging.

### Frontend

- **Pinia store** `useNotificationsStore` with 60s polling (paused on `document.hidden`), single-flight guard, auth-state lifecycle.
- **NotificationBell** in Navbar with pink-500 badge (`99+` cap).
- **NotificationDropdown** cloning language-switcher styling (`w-[380px]`, `bg-gray-950/95 backdrop-blur-xl`).
- **NotificationToast** at desktop bottom-right / mobile top-16 (avoiding collision with existing Toaster.vue), 8s auto-hide, pause-on-hover, route-suppression when on the matching anime page.
- **NewEpisodeCard** (poster, range line, translation source, dismiss ×) + **UnknownNotificationCard** graceful fallback.
- **Type-pluggable renderer registry** at `lib/notification-renderers.ts` — adding v1.1 types is additive only.
- **i18n keys** in en/ru/ja.
- **watch_url translator** helper + router redirect alias `/anime/:id/watch` → `/anime/:id` for defense in depth.
- **VITE_NOTIFICATIONS_ENABLED** env flag for cheap rollback.
- **Playwright e2e** (8 TCs: 3 always-on + 5 seed-gated).
