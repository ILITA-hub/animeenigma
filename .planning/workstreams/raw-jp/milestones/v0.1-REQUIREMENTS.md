# Requirements: AnimeEnigma `raw-jp` workstream

**Milestone:** v0.1 Raw Provider MVP
**Defined:** 2026-05-18
**Core value:** Watch anime in original Japanese audio with native-language subtitles from any of multiple aggregated subtitle providers. New "RAW JP" provider chip joins the existing RU and EN groups.
**Source design doc:** `/data/animeenigma/docs/superpowers/specs/2026-05-18-raw-jp-provider-design.md`

## v0.1 Requirements

### Backend — raw video source

- [ ] **RAW-01**: New parser at `services/catalog/internal/parser/allanime/` queries AllAnime's GraphQL API (`api.allanime.day`) using `translationType: raw` and returns episode lists + HLS stream URLs + (when available) embedded subtitle tracks. Persisted-query SHA hashes are read from env (`ALLANIME_QUERY_SEARCH_SHA`, `ALLANIME_QUERY_EPISODES_SHA`, `ALLANIME_QUERY_SOURCES_SHA`) — never hardcoded.

- [ ] **RAW-02**: Raw video source resolution path. Catalog exposes `GET /api/anime/{shikimori_id}/raw/episodes` and `GET /api/anime/{shikimori_id}/raw/stream?episode={n}&quality={q}`. Cache stream URLs for 1 hour (matching the existing pattern). Rotating-domain support iterates `[allanime.day, allmanga.to, allanime.to]` and caches the first-success domain in memory for the process lifetime.

### Backend — subtitle aggregation

- [ ] **RAW-03**: New parser at `services/catalog/internal/parser/opensubtitles/` queries the OpenSubtitles v1 REST API (`api.opensubtitles.com/api/v1`) keyed by IMDb ID + episode number (or query string fallback). Auth via `OPENSUBTITLES_API_KEY` env var. Returns subtitle file URLs with language metadata. New aggregator at `services/catalog/internal/service/subs_aggregator.go` fans out to Jimaku + OpenSubtitles in parallel, dedupes by content hash, groups by ISO language code. Exposed at `GET /api/anime/{shikimori_id}/subtitles?lang=ru,en,jp,...` and `GET /api/anime/{shikimori_id}/subtitles/all` (no language filter — for the "Other subs" panel).

- [ ] **RAW-04**: Extend `libs/idmapping/` to resolve IMDb and TMDB IDs in addition to the existing AniList/AniDB/Kitsu. Source: Kitsu mappings table fetched lazily on first OpenSubtitles query, cached in catalog DB. Schema: `ALTER TABLE animes ADD COLUMN IF NOT EXISTS imdb_id TEXT;` and `ALTER TABLE animes ADD COLUMN IF NOT EXISTS tmdb_id TEXT;`.

### Frontend — new player + subs panel

- [ ] **RAW-05**: New component `frontend/web/src/components/player/RawPlayer.vue` plays HLS streams returned by the raw resolver. Mirrors `HiAnimePlayer.vue` structure (Video.js + HLS.js switchable, `SubtitleOverlay.vue` mounted at fullscreen target, `subtitle-parser.ts` for ASS/SRT/VTT). Subtitle picker is promoted to a primary toolbar control (not buried in a settings flyout) since there is no dub option to compete for the surface.

- [ ] **RAW-06**: New component `frontend/web/src/components/player/OtherSubsPanel.vue`. Modal/sheet UI triggered by an "Other subs" button in the RawPlayer toolbar. Lists every subtitle track returned by `/api/anime/{id}/subtitles/all`, grouped by language with a provider attribution chip per row (e.g. "Jimaku JP", "OpenSubtitles RU"). On selection, emits a `select-track` event up to RawPlayer which loads it into `SubtitleOverlay`.

### Frontend — wiring + rollout

- [ ] **RAW-07**: Extend `frontend/web/src/views/Anime.vue` to add `'raw'` to the `preferred_video_provider` localStorage type union and add a new "RAW JP" language group in the provider chip switcher (third group alongside RU and EN). New `preferred_raw_provider` localStorage key for future-proofing v0.2+. Provider chip wiring is gated by feature flag `VITE_RAW_PROVIDER_ENABLED` (default false until live testing on `ui_audit_bot` passes).

- [ ] **RAW-08**: Playwright e2e test under `frontend/web/e2e/` covers the happy path: log in as `ui_audit_bot`, navigate to a known AllAnime-covered anime, switch to RAW JP provider, assert raw video loads, assert subtitle picker populates with ≥1 JP track, open "Other subs" panel, switch to a different language track, assert overlay updates.

### Non-functional

- [ ] **RAW-NF-01**: All new backend code respects existing error-handling conventions (`libs/errors` package). AllAnime parser failures return `errors.Unavailable` (not 500) so the frontend can show a graceful "raw provider unavailable" state. OpenSubtitles rate-limit (429) errors fall back to Jimaku-only results without surfacing the error to the user.

- [ ] **RAW-NF-02**: User-facing changelog entry in `frontend/web/public/changelog.json` describing the new RAW JP provider + "Other subs" feature. Informative + enthusiastic tone with emoji per the after-update convention. Three locales (en/ru/ja) per the existing translation expectation.

## Phase mapping

## Out of Scope

| Feature | Reason |
|---------|--------|
| Self-hosted MinIO library | Deferred to v0.2 — explicit user decision to split milestones |
| Auto-download for ongoings | Deferred to v0.3 |
| HiAnime / Consumet replacement | Separate workstream concern; the dead-HiAnime container will be addressed elsewhere |
| Kage Project / fansubs.ru integration | Cloudflare-protected, no API — too much friction for v0.1; revisit in a later milestone |
| User-uploaded subtitle files | Out of scope for all v0.x milestones |
| Subtitle quality scoring or auto-pick best track | Deferred to v0.4+ |
