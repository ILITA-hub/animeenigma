# Requirements: AnimeEnigma `raw-jp` workstream — v0.2

**Milestone:** v0.2 Self-Hosted Library
**Defined:** 2026-05-18
**Core value:** Admin-controlled self-hosted raw-JP source — Nyaa/AnimeTosho → BitTorrent → HLS in MinIO → transparently preferred over AllAnime by the existing `/raw/stream` endpoint when present.
**Source design doc:** `/data/animeenigma/docs/superpowers/specs/2026-05-18-raw-jp-provider-design.md`

## v0.2 Requirements

### Backend — library service scaffold

- [ ] **LIB-01**: New Go microservice at `services/library/` on port 8087. Standard project layout (`cmd/library-api/main.go`, `internal/{config,domain,handler,repo,service,transport}`, `migrations/`). Health endpoint `GET /health` returns 200 `{"status":"ok"}`. Prometheus `/metrics` registered via `libs/metrics`. Dedicated Postgres DB `library` auto-created via `libs/database.New()` + `db.AutoMigrate(...)` for the new tables. Service builds in the existing multi-module workspace (`go.work` extended) and ships as `services/library/Dockerfile` mirroring `services/catalog/Dockerfile`'s shape.

- [ ] **LIB-02**: Gateway proxy `/api/library/*` → `library:8087` in `services/gateway/internal/router/routes.go` (mirror the `/api/themes/*` proxy pattern). Public routes pass through; admin routes are guarded by the gateway's existing AdminMiddleware. New `Makefile` targets `redeploy-library`, `logs-library`, `restart-library`. `make health` includes `library:8087`.

### Backend — search

- [ ] **LIB-03**: Nyaa.si RSS client at `services/library/internal/parser/nyaa/`. Hits `https://nyaa.si/?page=rss&q={query}&c=1_2&f=0`. Parses RSS items into `Release{Title, Magnet, Uploader, Quality, SizeBytes, FoundAt}`. Quality parsed from title (`1080p`, `720p`, `480p`). Uploader extracted from the `<dc:creator>` field. Wrapped failures use a typed error so the merger can downgrade to AnimeTosho-only on outage.

- [ ] **LIB-04**: AnimeTosho JSON-feed client at `services/library/internal/parser/animetosho/`. Hits `https://feed.animetosho.org/json?show=mal&id={mal_id}` (primary) or `https://feed.animetosho.org/json?q={query}` (fallback when no MAL ID). Parses entries into the same `Release` type. Surfaces a `MALID` field so the merger can rank AnimeTosho results above Nyaa when MAL IDs match.

- [ ] **LIB-04b**: Search aggregator at `services/library/internal/service/search.go`. Runs both clients in parallel via `errgroup.Group{SetLimit(2)}`. Merges results, dedupes by `(uploader, magnet-infohash)` extracted via `metainfo.ParseMagnetURI`, and ranks AnimeTosho-with-MAL-ID hits to the top. Exposed at `GET /api/library/search?q=&mal_id=&limit=` (default limit 50). Returns 200 even when one provider fails (the other's results survive).

### Backend — torrent + job queue

- [ ] **LIB-05**: Embedded BitTorrent client at `services/library/internal/torrent/`. Wraps `github.com/anacrolix/torrent` with a clean interface: `Add(ctx, magnet) (torrentID string, err error)`, `Progress(torrentID) (downloadedBytes, totalBytes int64, peers int)`, `Cancel(torrentID)`, `Close()`. Configurable via env: `LIBRARY_TORRENT_DOWNLOAD_DIR` (default `/data/torrents`), `LIBRARY_TORRENT_MAX_PEERS` (default 80), `LIBRARY_TORRENT_MAX_UPLOAD_RATE_KBPS` (default 1024). Seeds completed torrents for `LIBRARY_TORRENT_SEED_DURATION` (default 24h) before stopping.

- [ ] **LIB-06**: Postgres-backed `library_jobs` queue. Schema in `migrations/001_library_jobs.sql` per the design doc. Worker checkout via `SELECT ... FOR UPDATE SKIP LOCKED LIMIT 1` so multiple workers don't race. State machine: `queued → downloading → encoding → uploading → done|failed|cancelled`. Worker count tunable via `LIBRARY_DOWNLOAD_WORKERS` (default 2). Progress updates every 5s while in `downloading`. Endpoints:
  - `POST /api/library/jobs` — admin only; body `{magnet, title, source, uploader?, quality?, size_bytes?, shikimori_id?}` → returns created job.
  - `GET /api/library/jobs?status=&limit=` — admin only; list jobs, optional status filter.
  - `GET /api/library/jobs/{id}` — admin only; single job detail.
  - `DELETE /api/library/jobs/{id}` — admin only; transitions `queued|downloading → cancelled`.

### Backend — encoding + storage

- [ ] **LIB-07**: ffmpeg HLS transcoder at `services/library/internal/ffmpeg/`. Wraps `os/exec` with a structured command: `-c:v libx264 -c:a aac -b:a 128k -hls_time 6 -hls_playlist_type vod -hls_segment_filename .../segment_%03d.ts .../playlist.m3u8`. Source bitrate capped at 5 Mbps via `-b:v` chosen from ffprobe duration + file size. ffmpeg stderr is captured to a bounded ring buffer (last 2 KB) and stored in `library_jobs.error_text` on non-zero exit. Working dir under `LIBRARY_ENCODE_TMPDIR` (default `/tmp/encode`) — cleaned up on success, retained on failure for debugging. Worker count tunable via `LIBRARY_ENCODE_WORKERS` (default 2).

- [ ] **LIB-08**: MinIO writer at `services/library/internal/minio/`. New bucket `raw-library` bootstrapped at service start (idempotent, `mb` if absent). Uploads HLS playlist + segments to `raw-library/{shikimori_id}/{episode_number}/playlist.m3u8` and `.../segment_NNN.ts`. Concurrent segment uploads via worker pool (default 8). On successful upload, insert `library_episodes` row keyed `(shikimori_id, episode_number)` with `minio_path`, `duration_sec`, `size_bytes`. Migrations `002_library_episodes.sql` + `003_library_filename_patterns.sql` per the design doc; seed `library_filename_patterns` with regex patterns for Ohys-Raws, SubsPlease, Erai-raws, Leopard-Raws, ARC-Raws variants.

### Frontend — admin UI

- [ ] **LIB-09**: New view `frontend/web/src/views/admin/RawLibrary.vue` mounted at `/admin/raw-library` (guarded by router beforeEnter against `authStore.isAdmin`). Three sections:
  - Search input + MAL ID picker → table of results from `GET /api/library/search`.
  - Active jobs panel polling every 5s via `GET /api/library/jobs?status=queued,downloading,encoding,uploading`, with cancel buttons + progress bars per job.
  - Stats strip showing disk-free + active-torrents from a new `GET /api/library/health/extended`.
  - "Episodes pending link" panel: jobs with `done` + `shikimori_id=NULL` get an inline anime-picker dropdown that calls `PATCH /api/library/jobs/{id}` (new endpoint to update `shikimori_id` post-hoc), then triggers a re-link flow to write `library_episodes`.

  i18n strings in `frontend/web/src/locales/{en,ru,ja}.json` under `player.adminLibrary.*`. Reuses existing `Modal.vue`, `Button.vue`, `Badge.vue`, `Input.vue`. New types in `frontend/web/src/types/library.ts`.

### Backend — hybrid resolver

- [ ] **LIB-10**: Extend `services/catalog/internal/service/raw_resolver.go`'s `GetStream(animeID, episode, quality)` to first call a new thin client `services/catalog/internal/parser/library/client.go` (`GET http://library:8087/api/library/episodes/{shikimori_id}/{episode}`). When the library returns 200 with a MinIO HLS URL, the catalog returns that URL with `source: "library"` in its response. When the library returns 404 / 5xx / times out, the catalog falls back to the v0.1 AllAnime path (existing behavior). Cache the per-`(animeID, episode)` source decision in Redis for 1h. The library service POSTs to a new catalog endpoint `POST /internal/cache/invalidate/raw/{shikimori_id}` after each `done` job to bust the cache eagerly. The catalog handler protects this endpoint with the same `Internal` middleware the existing `/internal/resolve-api-key` uses.

### Non-functional

- [ ] **LIB-NF-01**: Disk-free guard. `services/library/internal/service/diskguard.go` checks free space on `LIBRARY_TORRENT_DOWNLOAD_DIR` at every job enqueue. When free < `LIBRARY_DISK_FREE_MIN_PCT` (default 20), the enqueue handler returns HTTP 507 with body `{"error":"disk_full"}` and increments `library_enqueue_rejected_total{reason="disk_full"}`.

- [ ] **LIB-NF-02**: Job stall detection. Worker emits a heartbeat counter `library_torrent_no_peers_seconds{job_id}` whenever a job has had zero peers for >30s continuously. When the counter exceeds `LIBRARY_TORRENT_STALL_TIMEOUT` (default 30m), the worker transitions the job to `failed` with `error_text="stalled: no peers for 30 minutes"`. Admin UI surfaces the failure with a "Retry" button that re-enqueues a fresh job with the same magnet.

- [ ] **LIB-NF-03**: Prometheus metrics on `/metrics` per the design doc: `library_jobs_total{status}` (counter, increments on transition), `library_download_bytes_total` (counter, by job), `library_encode_duration_seconds` (histogram, by quality), `library_active_torrents` (gauge), `library_disk_free_bytes` (gauge, polled every 30s), `library_torrent_seed_count` (gauge), `library_enqueue_rejected_total{reason}` (counter), `library_upload_bytes_total` (counter, MinIO PUT bytes). Grafana dashboard `infra/grafana/dashboards/library.json` with panels covering each metric.

- [ ] **LIB-NF-04**: Operator runbook. Append an `## ISS-013` (or next free) entry to `docs/issues/README.md` documenting: env var glossary, where to look in Grafana when jobs stall, how to manually delete a `library_episodes` row to force re-encode, how to extend `library_filename_patterns` for a new uploader. Update `CLAUDE.md` "Service Ports" table with `library | 8087 | /metrics | Library service (BitTorrent → HLS → MinIO)`. Update `CLAUDE.md` "Gateway Routing" with `/api/library/* → library:8087`.

## Out of Scope (v0.2)

| Feature | Reason |
|---------|--------|
| Auto-download for ongoing series | Explicit v0.3 milestone — needs real watch data first. |
| User-facing library listing / request-to-add | Defer until admin curation is proven and the library has critical mass. |
| Subtitle extraction from torrent files | Out of scope for all v0.x; the subs aggregator (v0.1 Phase 2) is the single source for subtitle metadata. |
| Multi-season torrent splitting / season-pack auto-detection | v0.2.1 followup if pain materializes; admins split season packs manually for v0.2. |
| Kage Project / fansubs.ru integration | Cloudflare-protected, deferred indefinitely (deferred from v0.1 as well). |
| Subtitle quality scoring or auto-pick best track | Cross-cuts the subs aggregator; deferred to v0.4+. |
| Library historical cleanup (delete old episodes) | v0.2.1 once disk pressure manifests. |
| Cloudflare WARP egress for library service | Library service downloads from public torrent peers, not Cloudflare-fronted CDNs. No need. |
