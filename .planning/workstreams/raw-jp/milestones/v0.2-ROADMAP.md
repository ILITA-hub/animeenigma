# Roadmap: v0.2 Self-Hosted Library

**Workstream:** raw-jp
**Milestone:** v0.2 (active)
**Created:** 2026-05-18
**Source design doc:** `/data/animeenigma/docs/superpowers/specs/2026-05-18-raw-jp-provider-design.md`
**Requirements:** `v0.2-REQUIREMENTS.md`

## Goal

Stand up a self-hosted raw-JP library managed by admins: Nyaa.si / AnimeTosho search → BitTorrent download → ffmpeg HLS transcode → MinIO storage → hybrid resolver that prefers the self-hosted copy over AllAnime when both exist. Backend-heavy milestone with one admin-only UI; user-facing behavior is unchanged until an admin queues a job and an episode lands in MinIO, after which the existing `/api/anime/{id}/raw/stream` endpoint transparently switches the served source.

## Phases

### Phase 1: Library Service Scaffold

**Goal:** Stand up a new Go microservice `services/library/` on port 8087 with the standard project layout (`cmd/library-api/main.go`, `internal/{config,domain,handler,repo,service,transport}`, `migrations/`). Wire it into `docker-compose.yml` (build context + service block + new persistent volumes `library_torrents`, `library_minio_staging`) and the gateway router (`/api/library/*` → `library:8087`). Bootstrap a dedicated Postgres DB `library` via the shared `libs/database` helper. Service responds 200 on `/health` and exposes Prometheus `/metrics` (initially just the standard HTTP middleware metrics). Service prefix `library` is registered with the existing metrics + logger conventions.

**Depends on:** Nothing — additive backend module.
**Requirements:** LIB-01, LIB-02, LIB-NF-04

### Phase 2: Nyaa + AnimeTosho Search Clients

**Goal:** Two parsers under `services/library/internal/parser/{nyaa,animetosho}/`. Both implement a small `Searcher` interface: `Search(ctx, params SearchParams) ([]Release, error)`. AnimeTosho is the primary source because its JSON feed at `feed.animetosho.org` supports a `?mal_id=` filter — when the catalog has a MAL ID for the anime, this is the most reliable identifier. Nyaa.si is the fallback: pure RSS, query-string search (title-keyed), broader catalogue including some titles AnimeTosho misses. Both return a normalized `Release{Title, Magnet, Uploader, Quality, SizeBytes, SeedersFromFeed, FoundAt}`. New endpoint `GET /api/library/search?q=&mal_id=&limit=` runs both searches in parallel, merges results, dedupes by `(uploader, infohash-from-magnet)`, and ranks AnimeTosho hits above Nyaa when MAL ID matches.

**Depends on:** Phase 1 scaffold.
**Requirements:** LIB-03, LIB-04
**SPEC:** `v0.2-phases/02-nyaa-animetosho-clients/02-SPEC.md`
**Touches:**
- `services/library/internal/parser/nyaa/client.go` + `client_test.go` (new)
- `services/library/internal/parser/animetosho/client.go` + `client_test.go` (new)
- `services/library/internal/service/search.go` (new — merge + dedupe + rank)
- `services/library/internal/handler/search.go` (new — `GET /search`)
- `services/library/internal/transport/router.go` (route)
- `services/library/internal/config/config.go` (extend with `NYAA_BASE_URL`, `ANIMETOSHO_BASE_URL`, `LIBRARY_SEARCH_TIMEOUT`)

**Success criteria:**
1. `curl 'http://localhost:8087/api/library/search?q=bocchi+the+rock&mal_id=52082'` returns 200 with ≥1 hit from each provider (assumes upstream reachable).
2. AnimeTosho hits surface ABOVE Nyaa hits in the response when MAL ID is provided.
3. Duplicate magnet links (same infohash) appear once in the merged response.
4. Unit tests against `httptest` mocks cover both parsers and the merger.
5. Upstream timeout / 5xx returns the other provider's results (graceful degradation).

### Phase 3: Torrent Client + Job Queue + Metrics

**Goal:** Embed `github.com/anacrolix/torrent` behind `services/library/internal/torrent/`. Build a `library_jobs` Postgres-backed queue (state machine: `queued → downloading → encoding → uploading → done|failed|cancelled`) with `FOR UPDATE SKIP LOCKED` for safe concurrent worker checkout. Worker count tunable via `LIBRARY_DOWNLOAD_WORKERS` env (default 2). Emit Prometheus metrics on `/metrics`: `library_jobs_total{status}`, `library_download_bytes_total`, `library_active_torrents`, `library_disk_free_bytes`, `library_torrent_seed_count`. Surface job enqueue + status endpoints:
- `POST /api/library/jobs` — body `{magnet, title, source, uploader, quality, size_bytes, shikimori_id?}` → returns the created `library_jobs` row.
- `GET /api/library/jobs?status=&limit=` — list, filterable by status.
- `GET /api/library/jobs/{id}` — single job detail.
- `DELETE /api/library/jobs/{id}` — cancel (transition to `cancelled` from `queued|downloading`).

Workers transition `queued → downloading`, run the torrent to completion, update progress every 5s, then transition to `encoding` (no work yet — Phase 4). Disk-free guard at the enqueue handler refuses new jobs with HTTP 507 when free disk drops below `LIBRARY_DISK_FREE_MIN_PCT` (default 20).

**Depends on:** Phase 1 scaffold; Phase 2 search clients are used by the admin UI but not by the queue path itself (admin builds a magnet via search, then POSTs to /jobs).
**Requirements:** LIB-05, LIB-06, LIB-NF-01, LIB-NF-02, LIB-NF-03
**SPEC:** `v0.2-phases/03-torrent-client-job-queue/03-SPEC.md`
**Touches:**
- `services/library/internal/torrent/client.go` + `client_test.go` (new — anacrolix wrapper)
- `services/library/internal/domain/job.go` (new — `Job`, `JobStatus` types)
- `services/library/internal/repo/job.go` (new — CRUD + `Claim()` with `FOR UPDATE SKIP LOCKED`)
- `services/library/internal/service/worker.go` (new — downloader loop)
- `services/library/internal/handler/jobs.go` (new)
- `services/library/internal/metrics/library_metrics.go` (new — counters/gauges)
- `services/library/migrations/001_library_jobs.sql` (new)
- `infra/grafana/dashboards/library.json` (new dashboard with the new metrics)

**Success criteria:**
1. POST a known small public-domain magnet → job lands in `queued`, worker picks it up, status transitions to `downloading`, then to `encoding` (no-op for now), progress visible via the GET endpoint.
2. `library_jobs_total{status="downloading"}` increments and decrements.
3. Two workers concurrently claim two different queued jobs without one stomping the other (verified via integration test).
4. Disk-free probe returns 507 when manually constrained.
5. Cancel transitions a running job to `cancelled` and the torrent client stops fetching peers.

### Phase 4: ffmpeg HLS Transcoder + MinIO Writer

**Goal:** Workers progress `encoding` and `uploading` states. ffmpeg subprocess wrapper at `services/library/internal/ffmpeg/` transcodes the downloaded file to H.264 + AAC + HLS (6s segments, VOD playlist, audio bitrate 128k, video bitrate matched to source up to 5 Mbps cap). The wrapper extracts duration from ffprobe before transcode for ETA, then runs ffmpeg with a stable command (no creative flags — `-c:v libx264 -c:a aac -hls_time 6 -hls_playlist_type vod ...`). MinIO writer uploads playlist + segments under `raw-library/{shikimori_id}/{episode_number}/`. Auto-detect `episode_number` from the source filename via per-uploader regex patterns stored in `library_filename_patterns` (seeded with Ohys-Raws, SubsPlease, Erai-raws, Leopard-Raws, ARC-Raws variants). When `shikimori_id` is missing (admin queued raw torrent without linking), the job lands `done` with `shikimori_id=NULL` and the admin UI surfaces it for manual linking (Phase 5). On successful encode + upload, insert a `library_episodes` row keyed `(shikimori_id, episode_number)`.

**Depends on:** Phase 3 job queue.
**Requirements:** LIB-07, LIB-08
**SPEC:** `v0.2-phases/04-ffmpeg-minio-transcoder/04-SPEC.md`
**Touches:**
- `services/library/internal/ffmpeg/transcoder.go` + `transcoder_test.go` (new — Cmd construction, exit-code handling, stderr capture)
- `services/library/internal/minio/writer.go` + `writer_test.go` (new — `Upload(bucket, prefix, files)`)
- `services/library/internal/parser/filename/detector.go` (new — episode-number heuristics)
- `services/library/internal/service/encoder_worker.go` (new — encoding+uploading loop)
- `services/library/migrations/002_library_episodes.sql`
- `services/library/migrations/003_library_filename_patterns.sql`
- `services/library/internal/handler/episodes.go` (new — `GET /api/library/episodes/{shikimori_id}/{episode}`)
- `services/library/internal/config/config.go` (new ffmpeg path + workers + MinIO knobs)
- `docker/docker-compose.yml` (`ffmpeg` binary baked into library Docker image; new MinIO bucket bootstrap)

**Success criteria:**
1. A queued job for a known small public-domain torrent completes the full pipeline: `queued → downloading → encoding → uploading → done`.
2. MinIO bucket `raw-library` has `<shikimori_id>/1/playlist.m3u8` + segments.
3. `curl` against the MinIO HLS playlist plays in `mpv` or `vlc`.
4. Job with no `shikimori_id` finishes with `done` + `shikimori_id=NULL` and a row in `library_jobs` flagged for admin link.
5. ffmpeg failure → job `failed` with `error_text` populated from stderr tail.
6. Unit tests cover episode-number regex extraction across all five seeded uploader patterns.
7. `library_encode_duration_seconds` histogram populated.

### Phase 5: RawLibrary.vue Admin UI

**Goal:** New admin-only Vue view `frontend/web/src/views/admin/RawLibrary.vue` mounted under `/admin/raw-library`. Three sections:
1. **Search** — title input + optional MAL ID prefill (autocomplete from catalog search). Submitting calls `/api/library/search` and renders a table: provider chip (AnimeTosho / Nyaa), uploader, quality, size, magnet preview, "Queue" button.
2. **Active jobs** — polls `/api/library/jobs?status=queued,downloading,encoding,uploading` every 5s and shows progress bars per job. Cancel button per row. "Episodes pending link" panel highlights `done`-but-NULL-shikimori_id jobs with an inline dropdown to bind them to a catalog anime.
3. **Stats** — top strip showing `library_disk_free_bytes`, `library_active_torrents` (from `/api/library/health/extended` summary endpoint added in this phase).

Gated by the existing AdminMiddleware on the backend and by `authStore.isAdmin` in the router. New `adminLibraryApi` module in `frontend/web/src/api/client.ts` wraps the endpoints. i18n strings in `player.adminLibrary.*` for en/ru/ja.

### Phase 6: Hybrid Resolver

**Goal:** Extend the v0.1 catalog raw resolver at `services/catalog/internal/service/raw_resolver.go` so that `GetStream(animeID, episode, quality)` first checks the library service for a MinIO HLS URL via a new thin client `services/catalog/internal/parser/library/client.go` (`GET /api/library/episodes/{shikimori_id}/{episode}`). If the library has the episode, the catalog returns the MinIO URL directly with `source: "library"` in the response envelope. If the library lookup misses, returns 404, or the service is unhealthy, the catalog falls back to AllAnime (current v0.1 behavior).

Add a `source` field to the `RawStream` response shape; frontend already ignores unknown fields. Cache the library-vs-allanime decision per `(animeID, episode)` for 1h so we don't double-hit two backends every stream resolve. Bust the cache via a webhook from library service (`POST /internal/cache/invalidate/raw/{shikimori_id}`) when a new `library_episodes` row is written — keeps the cache fresh after admin actions.

**Depends on:** Phase 4 (`library_episodes` populated).
**Requirements:** LIB-10
**SPEC:** `v0.2-phases/06-hybrid-resolver/06-SPEC.md`
**Touches:**
- `services/catalog/internal/parser/library/client.go` + `client_test.go` (new)
- `services/catalog/internal/service/raw_resolver.go` (extend `GetStream` with library-first lookup)
- `services/catalog/internal/service/raw_resolver_test.go` (new — table-driven cases for hit/miss/unhealthy)
- `services/catalog/internal/config/config.go` (new `LibraryConfig{APIURL, Timeout}`)
- `services/catalog/internal/handler/internal_cache.go` (new — `POST /internal/cache/invalidate/raw/{id}`)
- `services/library/internal/service/encoder_worker.go` (extend — fire invalidation webhook after `done`)

**Success criteria:**
1. With a `library_episodes` row present, `curl /api/anime/{uuid}/raw/stream?episode=1` returns the MinIO HLS URL with `source: "library"`.
2. Without a `library_episodes` row, the same endpoint returns the AllAnime URL with `source: "allanime"`.
3. Library service stopped → catalog falls back to AllAnime within the per-request timeout (no 503 surfaces to the frontend).
4. Webhook invalidation flushes the catalog's `raw:stream:*` Redis keys for the affected shikimori_id.
5. Unit + integration tests cover all routing branches.

## Out of Scope (v0.2)

- Auto-download for ongoing series — v0.3.
- User-facing surface for the library (read-only listing, request-to-add, etc.) — v0.4+.
- Subtitle extraction from torrent files — out of scope for v0.x; subs continue to come from Phase 02 aggregator.
- Multi-season torrent handling — single-season per job for v0.2; admins split season packs manually.
- Cloudflare-protected Russian fansub indexers (Kage Project, fansubs.ru) — not part of v0.2.

## Risks / Followups

- **anacrolix/torrent licensing + portability:** confirmed pure-Go, BSD-3-clause; safe.
- **Disk consumption:** uncontrolled growth is the biggest operational risk; `LIBRARY_DISK_FREE_MIN_PCT` guard rejects new jobs, but historical clean-up (delete old episodes) is a separate followup deferred to v0.2.1 if needed.
- **Magnet → infohash dedupe:** anacrolix's `metainfo.ParseMagnetURI` is the safe path; never hand-roll the parse.
- **ffmpeg container size:** baking ffmpeg into the library Docker image bloats it by ~80MB but avoids a sidecar pattern; acceptable trade-off.
- **MinIO bucket policy:** new bucket `raw-library` needs the standard server-side-only ACL (no public read). Bootstrap script lives in `docker/minio/init.sh` (extend in Phase 4).
