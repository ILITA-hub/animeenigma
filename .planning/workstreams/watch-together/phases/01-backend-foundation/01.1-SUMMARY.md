## Deviations from Plan

1. **chi pinned to v5.0.12 (plan said v5.2.5)** — The watch-together workspace
   peers (12 of 13 services) all use `chi v5.0.12`; only `services/notifications`
   uses `v5.2.5`. To avoid `go work sync` propagating a chi upgrade across the
   entire workspace (and dragging 13 unrelated services into this commit), the
   new module pins `v5.0.12` to match the majority. Rule 3 (auto-fix blocking
   issues) — this prevents a cross-service workspace consistency cascade outside
   01.1's scope. Functionally equivalent: chi v5 is API-stable across patch
   versions, both expose `chi.Router`, `middleware.RequestID`, `middleware.RealIP`,
   etc.

2. **Test collector wired via `sync.Once`** — `libs/metrics.NewCollector` uses
   `promauto` against the global Prometheus registry, which panics on duplicate
   registration in a single test binary. Solved with a package-level
   `sync.Once`-guarded shared collector in `router_test.go` so both tests use one
   instance. Standard Go test-isolation pattern; documented in code comment.

3. **`go run` smoke test bound port 18091 (not 8091)** — Plan acceptance suggests
   `curl http://localhost:8091/health`; production port is 8091 (per CLAUDE.md
   Service Ports row to be added in 01.8). Local smoke used 18091 via
   `SERVER_PORT=18091` to avoid clashing with whatever might already bind 8091 on
   the dev host. Functional check is identical.

4. **`AuthMiddleware` exported eagerly** — The plan explicitly anticipates this
   (`<action>` step says "Define AuthMiddleware even though 01.1 doesn't use it
   — downstream plans (01.4) import this directly"). Implemented as a verbatim
   copy from `services/notifications/internal/transport/router.go`. Not a
   deviation — flagging for visibility.

5. **Pre-existing workspace `ambiguous import` repro under `go work sync`** —
   When the workspace is active (`GOWORK` unset), `go build` against any service
   in the repo fails on `google.golang.org/genproto/googleapis/rpc/status` due
   to an old `genproto` module being pinned in `go.work.sum` alongside the new
   split-out `genproto/googleapis/rpc`. Reproducible against `services/themes`
   and `services/notifications` independently of watch-together. **NOT
   introduced by this plan.** All builds in this PR (and the Docker build) use
   `GOWORK=off GOFLAGS=-mod=mod` which produces clean compiles. Flagging here
   so future workstreams know the workaround; deferring an actual fix as
   out-of-scope (logged via this summary instead of a `deferred-items.md`
   since there is no phase-level deferred file yet).
