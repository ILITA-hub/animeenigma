<schema_contract>
<!-- From CONTEXT.md / WT-FOUND-02 / design doc § Data Model — all 4 key families. -->

wt:room:{roomId}                 HASH
  ├─ id                            string
  ├─ created_at                    unix sec (int64 stored as string)
  ├─ anime_id                      string
  ├─ episode_id                    string
  ├─ player                        "kodik" | "animelib" | "ourenglish" | "hanime" | "raw"
  ├─ translation_id                string
  ├─ playback_state                "playing" | "paused"
  ├─ playback_time                 float64 (stored via strconv.FormatFloat)
  ├─ playback_time_updated_at      unix ms (int64)
  └─ host_user_id                  string

wt:room:{roomId}:members         HASH    user_id → MemberMeta-JSON

wt:room:{roomId}:messages        LIST    LPUSH new, LTRIM 0 99 (cap 100, newest at head)

wt:room:{roomId}:events          PUBSUB  channel for forward-compat fanout

TTL: 900s sliding on all 3 persistent keys (the room HASH, members HASH, messages LIST)
     refreshed atomically via a single Redis pipeline on every state-mutating op
</schema_contract>

    Implementation rules:
    - CreateRoom uses `HSet` with a map and follows up by piping `Expire` on all 3 keys (HASH + members + messages). Use `client.TxPipelined`.
    - UpdateRoomState ONLY writes the fields passed in the map (so callers can update just `playback_state` + `playback_time` + `playback_time_updated_at` without overwriting other fields). Allowed field names are validated against a whitelist:
      `id, created_at, anime_id, episode_id, player, translation_id, playback_state, playback_time, playback_time_updated_at, host_user_id`. Unknown field → return `errors.New("invalid field name")`.
    - AppendMessage:
      ```
      pipe.LPush(ctx, KeyRoomMessages(roomID), jsonBytes)
      pipe.LTrim(ctx, KeyRoomMessages(roomID), 0, 99)  // cap 100
      pipe.Expire(ctx, KeyRoom(roomID), r.ttl)
      pipe.Expire(ctx, KeyRoomMembers(roomID), r.ttl)
      pipe.Expire(ctx, KeyRoomMessages(roomID), r.ttl)
      ```
    - GetMessages(limit): `LRange(0, limit-1)` to read newest-first → reverse before returning (oldest-first).
    - GetRoom on missing key: detect via `client.Exists` first; return `(nil, ErrNotFound)` if not present. ErrNotFound = `errors.New("watch-together: room not found")` — define it as a package-exported sentinel so handlers can `errors.Is(err, repo.ErrNotFound)`.
    - Subscribe returns the raw `*redis.PubSub` (the hub in 01.3 manages lifecycle).
    - Use `strconv.FormatFloat(playback_time, 'f', -1, 64)` to encode the float on writes; `strconv.ParseFloat` on reads. Use `strconv.FormatInt(int64Field, 10)` / `ParseInt` for the int64 fields.
    - All public methods MUST refresh TTL on the 3 persistent keys via a single TxPipelined block on every successful state-mutating call. GetRoom / ListMembers / GetMessages / CountMembers / Exists do NOT refresh (per CONTEXT.md: "sliding TTL refreshed on any inbound event" — reads are not inbound events).
    - Log every state mutation via `log.Infow("watch_together repo op", "op", "create_room", "room_id", roomID, ...)` per WT-NF-03 grep-ability requirement.
