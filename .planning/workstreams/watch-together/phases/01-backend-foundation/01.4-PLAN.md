<api_contract>
POST /api/watch-together/rooms
  Request body:
    {
      "anime_id":       string (UUID, required),
      "episode_id":     string (required),
      "player":         "kodik" | "animelib" | "ourenglish" | "hanime" | "raw" (required),
      "translation_id": string (required)
    }
  Validation:
    - anime_id non-empty
    - episode_id non-empty
    - player ∈ allowed set (use domain.PlayerKodik/... constants from 01.1)
    - translation_id non-empty
  Auth: required (AuthMiddleware applied at route group)
  Response 200 (NOT 201 to match design doc convention):
    {
      "room_id":    string (newly generated UUID),
      "invite_url": string (e.g. "https://animeenigma.ru/watch/room/{room_id}"),
      "ws_url":     string (e.g. "wss://animeenigma.ru/api/watch-together/ws?room={room_id}")
    }
  Errors:
    - 400 if validation fails (specific field listed)
    - 401 if no JWT
    - 500 on Redis failure

GET /api/watch-together/rooms/{id}
  Auth: required
  Response 200:
    RoomSnapshot { room: {...}, members: [...], messages: [...last 50...], protocol_version: "1.0" }
  Errors:
    - 401 if no JWT
    - 410 Gone if room TTL has expired or DeleteRoom was called

DELETE /api/watch-together/rooms/{id}
  Auth: required
  Authorization: only the host_user_id of the room can delete (per WT-FOUND-03 "host-only force-close")
  Response 204 on success
  Errors:
    - 401 if no JWT
    - 403 if caller is not host_user_id ("not_host" code)
    - 410 if room already gone
    - 500 on Redis failure
  Side effects:
    - Deletes wt:room:{id}, wt:room:{id}:members, wt:room:{id}:messages
    - (NB: broadcast room:closed to all connected members is wired in 01.5/01.6 — not in this plan, because we don't yet have the hub plumbed into main.go. Add a TODO comment in the handler at the deletion point: `// TODO(01.5/01.6): hub.Broadcast(roomID, MsgRoomClosed) before DeleteRoom`)
</api_contract>
