# Phase 01 Plan 01.4: REST handlers + service layer Summary

## What was built

Three REST endpoints for the room lifecycle (POST/GET/DELETE under `/api/watch-together/rooms`) backed by a `RoomService` that owns every Redis mutation for room state. Wired into `main.go` with a real `redis.NewClient` PING-checked at boot.

## Endpoint contract (as shipped)

```
POST /api/watch-together/rooms
  Auth: JWT Bearer (AuthMiddleware)
  Body: {"anime_id", "episode_id", "player", "translation_id"}  // all required
  200 OK: {"success":true, "data": {"room_id","invite_url","ws_url"}}
  400: validation error (missing field, unknown player like "vlc", malformed JSON)
  401: missing/invalid JWT
  500: Redis failure

GET /api/watch-together/rooms/{id}
  Auth: JWT Bearer
  200 OK: {"success":true, "data": RoomSnapshot{room, members[], messages[], protocol_version:"1.0"}}
  410 Gone: room expired or never existed (body: {"success":false,"error":{"code":"GONE","message":"room expired or does not exist"}})
  401: missing/invalid JWT

DELETE /api/watch-together/rooms/{id}
  Auth: JWT Bearer
  Authorization: only room.host_user_id may delete (WT-FOUND-03)
  204 No Content: success
  403 Forbidden: caller is not host
  410 Gone: room already gone
  401: missing/invalid JWT
```

## Sample POST /rooms response

Given `PublicBaseURL = "https://animeenigma.ru"` and `room_id = "abc-123"`:

```json
{
  "success": true,
  "data": {
    "room_id": "abc-123",
    "invite_url": "https://animeenigma.ru/watch/room/abc-123",
    "ws_url": "wss://animeenigma.ru/api/watch-together/ws?room=abc-123"
  }
}
```

For dev (`PublicBaseURL=http://localhost:8000`): `invite_url=http://localhost:8000/watch/room/abc-123`, `ws_url=ws://localhost:8000/api/watch-together/ws?room=abc-123`.

## Auth flow used in tests

The handler tests bypass the real `AuthMiddleware` and instead inject a synthesized `*authz.Claims` directly via a small test-only middleware (`injectClaims`). This is equivalent to what `AuthMiddleware` would do after a successful JWT validation, but avoids the JWT-minting round-trip in unit tests.

One canary test (`TestCreate_NoAuth_Returns401`) uses a router with NO claims injection to verify the handler's defensive `userID == ""` check fires and returns 401 — independent of `AuthMiddleware` behavior, which is exercised in the package-level `router_test.go` (Plan 01.1).

For end-to-end auth verification, the gateway smoke test (Plan 01.7) and the wscat smoke (Plan 01-CONTEXT.md §Smoke Test) exercise the full path with a real JWT.

## wsURLFromBase scheme-swap implementation

The frontend constructs `new WebSocket(ws_url)` directly against the value returned by POST /rooms. Browser WebSocket APIs require `ws://` / `wss://` schemes (HTTP schemes are rejected), so the handler explicitly swaps:

| PublicBaseURL                 | ws_url                                                          |
| ----------------------------- | --------------------------------------------------------------- |
| `https://animeenigma.ru`      | `wss://animeenigma.ru/api/watch-together/ws?room={id}`          |
| `http://localhost:8000`       | `ws://localhost:8000/api/watch-together/ws?room={id}`           |
| `animeenigma.ru` (no scheme)  | `ws://animeenigma.ru/api/watch-together/ws?room={id}`           |

The scheme-swap uses `strings.HasPrefix` rather than `net/url.Parse` because:

1. Input space is bounded by config — three known shapes, not arbitrary user input
2. A `url.Parse` error on a config value would be a startup-time bug, not a runtime concern
3. The prefix check is easier to grep for and reason about

If 01.7 reveals that the gateway needs a different WS path (e.g. proxying via a sub-path like `/ws/wt`), update the constant in `wsURLFromBase` and add a test case to `TestWSURLFromBase_SchemeSwap`. The helper is the sole source of truth for the WS URL shape — handlers don't construct it from anywhere else.

## Deviations from Plan

None — plan executed exactly as written. Two minor implementation-level decisions worth flagging:

1. **Test injection points were exported via a separate file** (`internal/service/testing_hooks.go`) instead of exposing the unexported `newID`/`now` fields on `RoomService` directly. This was necessary because the handler package's tests can't reach unexported fields across package boundaries. The `SetIDProviderForTest` / `SetClockForTest` method names + the dedicated file make the test-only contract loud at the point of use.

2. **`go mod tidy` would have stripped the `google.golang.org/genproto` legacy-pin block**, which is required to resolve the workspace-wide `ambiguous import: googleapis/rpc/status` conflict (carried over from 01.1's go.mod). The pin was preserved through the linter pass by re-adding the require + comment block immediately after `go mod tidy`. Future maintainers: if you see a similar `ambiguous import` build error, check that the `require google.golang.org/genproto v0.0.0-20240528184218-...` line is still in `go.mod`.
