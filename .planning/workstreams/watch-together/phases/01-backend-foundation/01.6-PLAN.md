<drift_contract>
<!-- From design doc § Sync engine + WT-FOUND-07. Pure logic — testable without Redis or hub. -->

```
expected_time = room.playback_time + (now_ms - room.playback_time_updated_at_ms) / 1000.0
                if room.playback_state == "playing"
                else room.playback_time
drift = abs(member.reported_time - expected_time)
```

Decision table:
| drift             | action                                                               |
|-------------------|----------------------------------------------------------------------|
| <= 1.5s           | no action                                                            |
| 1.5s < d <= 5s    | send playback:correction {time: expected_time, server_ts: now_ms}    |
|                   | reset that member's consecutive-hard-drift counter to 0              |
| > 5s              | send playback:correction (same payload)                              |
|                   | increment that member's consecutive-hard-drift counter               |
|                   | if counter reaches 5 → send error:PERSISTENT_DRIFT, suspend          |
|                   | further corrections for that member until reconnect                  |

Member state (per (roomID, userID) tuple):
- consecutiveHardDrifts: int
- suspended: bool (once true, OnTimeTick returns immediately without action)

Reset triggers:
- Connection disconnect: clean up the member's entry from the map (called from the WS handler's OnClose hook).
- Any drift <= 5s: reset counter to 0 (drift recovered).
</drift_contract>

<rate_limit_contract>
- Per-user token-bucket via `rate.NewLimiter`.
- Seek limit: 1/sec, burst 1 (`rate.Every(time.Second), 1`).
- Chat limit: 5/sec, burst 5 (`rate.Limit(5), 5`).
- Identified by user_id (NOT by connection — multi-tab same-user shares the bucket).
- Disconnect cleanup: when MemberUserIDs(roomID) no longer contains userID, garbage-collect the bucket entry (call from OnClose).
- In-process only. WT-NF-02 explicitly scopes to in-process; Redis-backed limiting is deferred.
</rate_limit_contract>

<dispatch_table>
<!-- One handler per message type. All non-trivial logic lives in the handler; the
     Dispatch method is a 10-case switch. -->

| type                        | handler                  | side effects                                                                                       |
|-----------------------------|--------------------------|----------------------------------------------------------------------------------------------------|
| playback:play               | handlePlay               | repo.UpdateRoomState(state=playing, time=data.time, ts=now_ms); broadcast playback:event{kind:play}|
| playback:pause              | handlePause              | repo.UpdateRoomState(state=paused, time=data.time, ts=now_ms); broadcast playback:event{kind:pause}|
| playback:seek               | handleSeek               | rate-check; repo.UpdateRoomState(time=data.time, ts=now_ms); broadcast playback:event{kind:seek}    |
| playback:time_tick          | handleTimeTick           | NO broadcast; drift.OnTimeTick; may SendTo(self) playback:correction OR error:PERSISTENT_DRIFT     |
| state:change_episode        | handleChangeEpisode      | repo.UpdateRoomState(episode_id=value, time=0, state=paused, ts=now_ms); broadcast room:state_changed |
| state:change_player         | handleChangePlayer       | repo.UpdateRoomState(player=value, time=0, state=paused, ts=now_ms); broadcast room:state_changed   |
| state:change_translation    | handleChangeTranslation  | repo.UpdateRoomState(translation_id=value, time=0, state=paused, ts=now_ms); broadcast room:state_changed |
| chat:message                | handleChat               | rate-check; len(body)>500 → SendTo error:CHAT_TOO_LONG; repo.AppendMessage; broadcast chat:message  |
| chat:reaction               | handleReaction           | NO persistence; broadcast chat:reaction{user_id, emoji}                                            |
| presence:heartbeat          | handleHeartbeat          | repo.AddMember (re-PUT MemberMeta with last_seen_at=now); NO broadcast                              |

Every handler also bumps `wt_ws_messages_received_total{type=<msg type>}` counter (which was registered in 01.3 but called from the readPump in 01.3 — verify whether the readPump already bumps this OR if the dispatcher does. If the readPump bumps it on every read, the dispatcher should NOT double-bump. Decide once, document in 01-06-SUMMARY.md.)

Broadcast envelopes:
- playback:event data → `{kind: "play"|"pause"|"seek", time: float64, by_user_id: string, server_ts: int64_ms}`
- room:state_changed data → `{field: "episode_id"|"player"|"translation_id", value: string, by_user_id: string}`
- chat:message data → `{message: {id, user_id, username, body, ts}}` (note: wrapped in `message` per WT-FOUND-10 success criterion #4)
- chat:reaction data → `{user_id, emoji}`
- error data → `{code, message?, hint?}`
- playback:correction data → `{time, server_ts}`
</dispatch_table>

    Per-handler implementation rules:

    1. **handlePlay / handlePause / handleSeek**: Parse `{time float64}` from env.Data. handleSeek calls `r.rl.AllowSeek(conn.UserID)` first — if false, send error:RATE_LIMITED to self via hub.SendTo and return. After UpdateRoomState succeeds, build a playback:event envelope `{kind, time, by_user_id, server_ts}` and call `r.hub.Broadcast(ctx, conn.RoomID, env, conn.UserID)` (exclude self — sender already knows they played). NB: the design doc says playback:event excludes sender; the test should verify this.

    2. **handleTimeTick**: Parse `{time float64}`. Call `r.drift.OnTimeTick(ctx, r.repo, conn.RoomID, conn.UserID, time, now_ms)`. If returned Correction != nil:

       - Severity == DriftSoft or DriftHard → r.hub.SendTo(conn.RoomID, conn.UserID, correctionEnvelope(c.Time, c.ServerTS))
       - Severity == DriftNone → no-op
       NEVER broadcast (corrections are per-recipient).

    3. **handleStateChange**: Parse `{value string}` (the value name varies per type — extract via map[string]interface{} → look for episode_id/player/translation_id). Call repo.UpdateRoomState with the corresponding field + reset playback_time=0 + playback_state=paused + playback_time_updated_at=now. Broadcast room:state_changed{field, value, by_user_id} to ALL members (including sender — per design doc § server-outbound table).
