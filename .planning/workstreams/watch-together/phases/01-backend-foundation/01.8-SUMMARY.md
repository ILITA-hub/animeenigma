### `docker/docker-compose.yml`

Two edits:

1. Added `WATCH_TOGETHER_SERVICE_URL: http://watch-together:8091` to the gateway service env block, immediately after `NOTIFICATIONS_SERVICE_URL` (line ~351).
2. Inserted a new `watch-together:` service block immediately after `notifications:` and before `library:`. Verbatim:

```yaml
  # workstream watch-together, v1.0 Phase 1 — co-watch service on port 8091.
  # Redis-only state (no Postgres). REST endpoints for room lifecycle,
  # WebSocket /ws for sync/chat/reactions, drift detection engine.
  # See .planning/workstreams/watch-together/phases/01-backend-foundation/
  watch-together:
    build:
      context: ..
      dockerfile: services/watch-together/Dockerfile
    container_name: animeenigma-watch-together
    restart: unless-stopped
    environment:
      SERVER_PORT: 8091
      REDIS_HOST: redis
      REDIS_PORT: 6379
      JWT_SECRET: ${JWT_SECRET:-dev-secret-change-in-production}
      # WT-FOUND-02 — sliding TTL on Redis keys + 5min grace.
      WATCH_TOGETHER_ROOM_TTL: ${WATCH_TOGETHER_ROOM_TTL:-15m}
      WATCH_TOGETHER_GRACE_PERIOD: ${WATCH_TOGETHER_GRACE_PERIOD:-5m}
      # WT-NF-02 — capacity per room.
      WATCH_TOGETHER_MAX_MEMBERS: ${WATCH_TOGETHER_MAX_MEMBERS:-10}
      # Invite URLs in POST /rooms response. Defaults to production origin;
      # override for dev/staging via .env.
      WATCH_TOGETHER_PUBLIC_BASE_URL: ${WATCH_TOGETHER_PUBLIC_BASE_URL:-https://animeenigma.ru}
      # CORS / Origin allowlist gate. Set to true in dev if connecting from
      # a non-canonical origin. Production should leave this false.
      WATCH_TOGETHER_ALLOW_ALL_ORIGINS: ${WATCH_TOGETHER_ALLOW_ALL_ORIGINS:-false}
    ports:
      - "127.0.0.1:8091:8091"
    depends_on:
      redis:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "wget", "--quiet", "--tries=1", "--spider", "http://localhost:8091/health"]
      interval: 30s
      timeout: 5s
      retries: 3
```

Cross-checked env-var names against `services/watch-together/internal/config/config.go` (Load() reads `SERVER_PORT`, `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`, `REDIS_DB`, `JWT_SECRET`, `WATCH_TOGETHER_MAX_MEMBERS`, `WATCH_TOGETHER_ROOM_TTL`, `WATCH_TOGETHER_GRACE_PERIOD`). `REDIS_PASSWORD` / `REDIS_DB` not set in the block — config defaults (empty / 0) match dev Redis. The `WATCH_TOGETHER_PUBLIC_BASE_URL` and `WATCH_TOGETHER_ALLOW_ALL_ORIGINS` vars are surfaced ahead of 01.5/01.7 plans that will consume them.

Generic `redeploy-%`, `logs-%`, `restart-%` pattern rules still match, but explicit targets surface in `make help` for discoverability:
   ```makefile
   .PHONY: redeploy-watch-together logs-watch-together restart-watch-together

   redeploy-watch-together: ## Rebuild and restart watch-together service
   	cd docker && docker compose build watch-together && docker compose up -d watch-together

   logs-watch-together: ## Follow watch-together service logs
   	cd docker && docker compose logs -f watch-together

   restart-watch-together: ## Restart watch-together (no rebuild)
   	cd docker && docker compose restart watch-together
   ```
