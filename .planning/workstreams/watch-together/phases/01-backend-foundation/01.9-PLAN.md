RESEARCH STEP: check `services/auth/cmd/auth-api/` for any mint command (`grep -rn "MintJWT\|mint-jwt" services/auth/`); if none exists, use the Python-style HS256 approach: write JWT inline via openssl. The HS256 JWT for ui_audit_bot can be constructed as:
      ```bash
      mint_jwt() {
          local secret="$1"
          local user_id="$2"
          local username="$3"
          local now=$(date +%s)
          local exp=$((now + 900))  # 15min
          local header='{"alg":"HS256","typ":"JWT"}'
          local payload=$(printf '{"uid":"%s","username":"%s","role":"user","iss":"animeenigma","iat":%d,"exp":%d}' \
              "$user_id" "$username" "$now" "$exp")
          local b64_header=$(echo -n "$header" | openssl base64 -A | tr '+/' '-_' | tr -d '=')
          local b64_payload=$(echo -n "$payload" | openssl base64 -A | tr '+/' '-_' | tr -d '=')
          local sig=$(echo -n "${b64_header}.${b64_payload}" | openssl dgst -binary -sha256 -hmac "$secret" | openssl base64 -A | tr '+/' '-_' | tr -d '=')
          echo "${b64_header}.${b64_payload}.${sig}"
      }
      ```
      Use this inline. Pull `JWT_SECRET` via `source docker/.env`. Pull the ui_audit_bot user_id by querying `docker compose -f docker/docker-compose.yml exec -T postgres psql -U postgres -d animeenigma -tA -c "SELECT id FROM users WHERE username='ui_audit_bot'"`.

    Script structure:
    ```
    #!/usr/bin/env bash
    set -euo pipefail

    REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
    cd "$REPO_ROOT"

    # 0. Pre-checks: jq, openssl, docker, websocat-or-wscat, docker/.env exists
    require_tools jq openssl docker
    discover_ws_client  # sets WS_CLIENT to "websocat" or "wscat"

    # 1. Load secrets
    set -a; source docker/.env; set +a

    # 2. Make sure stack is up and watch-together is healthy.
    echo "[1/8] health check"
    curl -fsS http://localhost:8091/health | jq -e '.status=="ok"'

    # 3. Mint JWT for ui_audit_bot.
    UI_AUDIT_USER_ID=$(docker compose -f docker/docker-compose.yml exec -T postgres psql -U postgres -d animeenigma -tA -c "SELECT id FROM users WHERE username='ui_audit_bot'" | tr -d '[:space:]')
    JWT=$(mint_jwt "$JWT_SECRET" "$UI_AUDIT_USER_ID" "ui_audit_bot")

    # 4. POST /rooms via gateway
    echo "[2/8] POST /rooms via gateway"
    ROOM=$(curl -fsS -X POST http://localhost:8000/api/watch-together/rooms \
        -H "Authorization: Bearer $JWT" \
        -H "Content-Type: application/json" \
        -d '{"anime_id":"00000000-0000-0000-0000-000000000001","episode_id":"1","player":"animelib","translation_id":"test-translation"}')
    ROOM_ID=$(echo "$ROOM" | jq -r .room_id)
    [[ -n "$ROOM_ID" && "$ROOM_ID" != "null" ]] || { echo "FAIL: no room_id"; exit 1; }
    echo "  room_id=$ROOM_ID"

    # 5. WS connect — receive room:snapshot
    echo "[3/8] WS connect → room:snapshot"
    ws_url="ws://localhost:8000/api/watch-together/ws?token=$JWT&room=$ROOM_ID"
    first_frame=$(timeout 5 $WS_CLIENT -1 "$ws_url" || true)
    echo "$first_frame" | grep -q "room:snapshot" || { echo "FAIL: no room:snapshot"; exit 1; }

    # 6. Two-connection chat broadcast (run two background ws sessions)
    echo "[4/8] two-connection chat broadcast"
    # ... use bash subshells + fifos; capture each client's received frames
    #     send {"type":"chat:message","data":{"body":"hi"}} from one, assert both see it

    # 7. No-token → 401
    echo "[5/8] no-token → 401"
    rc=$(curl -s -o /dev/null -w "%{http_code}" "http://localhost:8000/api/watch-together/ws?room=$ROOM_ID" -H "Connection: Upgrade" -H "Upgrade: websocket" -H "Sec-WebSocket-Version: 13" -H "Sec-WebSocket-Key: dGVzdGtleQ==")
    [[ "$rc" == "401" ]] || { echo "FAIL: expected 401, got $rc"; exit 1; }

    # 8. Bogus room → ROOM_NOT_FOUND or 404
    echo "[6/8] bogus room"
    bogus_url="ws://localhost:8000/api/watch-together/ws?token=$JWT&room=does-not-exist-$(date +%s)"
    frame=$(timeout 3 $WS_CLIENT -1 "$bogus_url" 2>&1 || true)
    echo "$frame" | grep -qE "ROOM_NOT_FOUND|404" || { echo "FAIL: expected ROOM_NOT_FOUND or 404, got: $frame"; exit 1; }

    # 9. Spam seek → RATE_LIMITED
    echo "[7/8] rate limit on seek"
    # ... open WS, send 2× {"type":"playback:seek","data":{"time":10}} within 1s, assert second yields error:RATE_LIMITED

    # 10. TTL expiry → 410 Gone (uses overridden TTL=5s)

    # Only run if WATCH_TOGETHER_FAST_TTL=1 in env (operator opt-in: requires restart with TTL=5s).
    if [[ "${WATCH_TOGETHER_FAST_TTL:-0}" == "1" ]]; then
        # create room, sleep 7s, GET → assert 410
        ...
    else
        echo "  SKIP (set WATCH_TOGETHER_FAST_TTL=1 + restart watch-together with WATCH_TOGETHER_ROOM_TTL=5s to enable)"
    fi

    # Cleanup
    echo "Cleanup: DELETE /rooms/$ROOM_ID"
    curl -fsS -X DELETE -H "Authorization: Bearer $JWT" "http://localhost:8000/api/watch-together/rooms/$ROOM_ID" > /dev/null

    echo "✓ smoke complete"
    ```

    Implementation notes:
    - websocat `-1` flag reads a single frame then exits — perfect for first-frame assertions.
    - Two-connection chat test: spawn two backgrounded `websocat ... < pipe1 > out1.log &` processes, each fed via FIFO, capture stdout to log files, parse logs to assert both saw the broadcast.
    - Don't rely on `wscat` interactive mode — `websocat` is the better tool for scripting. The discovery step prefers websocat.
    - On any FAIL, print the last 20 lines of `docker compose -f docker/docker-compose.yml logs --tail 20 watch-together` for debugging context.
    - Make the script idempotent — re-running cleans up rooms it created (DELETE in trap EXIT).
    - The TTL-expiry test is gated behind an opt-in env flag because it requires reconfiguring the running container; it's a "deep" check that runs occasionally, not on every CI/dev cycle.
