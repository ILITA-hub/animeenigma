Codepoint preservation notes:
- `❤️` keeps U+2764 + U+FE0F (variation selector) — the dressed-as-emoji form most clients send. Stripping VS16 would silently fail the server-side whitelist match.
- `⚡` is U+26A1 (HIGH VOLTAGE SIGN).

## API client surface

- `createRoom(payload: CreateRoomRequest): Promise<CreateRoomResponse>` — POST `/watch-together/rooms`; unwraps `{success, data}` envelope.
- `getRoom(id: string): Promise<RoomSnapshot>` — GET `/watch-together/rooms/{encodeURIComponent(id)}`; throws `RoomGoneError` on 410, propagates other axios errors.
- `deleteRoom(id: string): Promise<void>` — DELETE `/watch-together/rooms/{encodeURIComponent(id)}`; throws `RoomForbiddenError` on 403, `RoomGoneError` on 410.

API client re-exports every public type + constant from `@/types/watch-together` so consumers can do:

```ts
import {
  createRoom, getRoom, deleteRoom,
  RoomGoneError, RoomForbiddenError, REACTION_WHITELIST,
  type RoomSnapshot, type Envelope, type CreateRoomRequest,
} from '@/api/watch-together'
```

## Deviations from Plan

None — plan executed exactly as written. Two minor refinements applied during implementation:

1. **REACTION_WHITELIST formatted one-emoji-per-line.** Plan-level verification gate `grep -c "❤️\|🔥\|⚡"` counts MATCHING LINES, not matches. The eight-emoji-per-row formatting initially yielded 2 matching lines (🔥+❤️ on one, ⚡ on another). Reformatting to one-per-line makes the array trivially readable and satisfies the ≥3 gate (3 matching lines). No semantic change.

2. **API client re-exports.** Plan's <behavior> specified "Re-exports the type symbols callers need." The implementation re-exports the FULL public surface (every constant, every interface, both errors) from `@/api/watch-together` so consumers have a single canonical import path. This is strictly additive — no consumer is forced to use the re-exports, and the original `@/types/watch-together` import path remains valid.
