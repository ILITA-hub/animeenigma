## Interpolation tokens

vue-i18n interpolation templates are preserved verbatim across both locales:

- `{n}` ‚Äî used by `chat_char_count` ("{n}/500" / "{n}/500")
- `{username}` ‚Äî used by `member_joined` and `member_left`

The parity test contains three explicit assertions that each interpolation token appears in both locale versions of the relevant key. Translators dropping a placeholder (e.g. translating `{n}` to a literal character) will fail the test.

## Deviations from `CONTEXT.md` required-list

`CONTEXT.md` ¬ßi18n listed 15 required keys. The plan's Task 1 action block (cross-checked against component needs by the planner) expanded that to 27. The 12 extras are:

| Added key | Why |
|-----------|-----|
| `chat_input_aria` | a11y ‚Äî vue-template SFCs need a separate aria-label distinct from placeholder |
| `chat_char_count` | Chat input enforces 500-char cap (per design doc); needs visible counter |
| `reaction_palette_aria` | a11y ‚Äî reaction palette needs aria-label distinct from heading |
| `invite_button_label` | InviteButton on WatchView needs its own CTA label |
| `invite_copy_manual` | Non-HTTPS environments need a manual-copy fallback (per CONTEXT.md "Claude's Discretion") |
| `reconnect_failed_title` | Permanent-disconnect UX (per CONTEXT.md "Auto-reconnect UX" ¬ß) |
| `reconnect_failed_button` | "Refresh" CTA on permanent-disconnect page |
| `capacity_full_title` | Renamed from CONTEXT.md `capacity_full_error` to clarify it's a full-page state, not a toast |
| `capacity_full_back_button` | "Back to anime" CTA on capacity-full page |
| `loading` | "Loading room‚Ä¶" before snapshot resolves |
| `member_joined` | Inline event line when a member joins (per design doc smoke spec) |
| `member_left` | Inline event line when a member leaves (per design doc smoke spec) |
| `room_id_label` | Admin/debug surface in sidebar footer |

(One key was renamed: `capacity_full_error` ‚Üí `capacity_full_title` because the design treats capacity-full as a dedicated page, not an inline error.)

## Insertion placement

Top-level keys in `en.json` / `ru.json` are NOT strictly alphabetical (they follow insertion order). I inserted `watch_together` between `notifications` and `spotlight` (the last block) to preserve existing ordering ‚Äî no other top-level keys were reordered. Both files end with the `spotlight` namespace as before.

## Downstream impact

2. Update `watch-together-keys.spec.ts` to import `ja` and assert tri-parity (en/ru/ja).
