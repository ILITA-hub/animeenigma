    Internal behaviors:
    1. **Snapshot replay:** Every (re)connect, the FIRST inbound frame MUST be `room:snapshot`. On receipt, OVERWRITE `room.value`, `members.value`, `messages.value` (NOT merge — replay-from-scratch is the project's reconnect model). If the first frame is NOT `room:snapshot`, log a warning and treat as protocol violation (force-close + reconnect once, then fail).
    2. **Reconnect backoff:** `delays = [1000, 2000, 4000, 8000, 16000, 30000]`. Capped at 30s; index clamped at `delays.length - 1`. Resets to 0 on successful `room:snapshot` receipt. After 30s cap, transition to `connectionStatus='failed'` and stop reconnecting (UI shows "Unable to reconnect — refresh").
    3. **Re-emission guard:** Each subscribed handler is called UNLESS the inbound event's `by_user_id` matches `useAuthStore().user?.id`. Applies to `onPlaybackEvent`, `onStateChanged`. Does NOT apply to chat/reaction (sender sees their own echo — server explicitly broadcasts to all-including-sender per Phase 1 chat-echo rule from 01-PHASE-SUMMARY.md).
    4. **WS URL construction:** Use `snapshot.room` for the room ID, and the JWT from `useAuthStore().token`. The full WS URL is computed client-side from `window.location` because Phase 1 returns `ws_url` from POST only, not GET; for GET (the join path), construct it: `${location.protocol === 'https:' ? 'wss:' : 'ws:'}//${location.host}/api/watch-together/ws?token=${token}&room=${roomId}`. Mention `?token=` query-param auth (browsers can't set Authorization headers on WS upgrade).
    5. **Echo suppression for emits:** When you call `emitPlay(time)`, the server WILL broadcast `playback:event{by_user_id: <you>}` back to all members INCLUDING you. The re-emission guard above filters that local echo automatically.
    6. **sendChat:** Reject if `body.trim().length === 0` OR `body.length > 500` (mirror server-side cap from WT-FOUND-05). Returns silently (no throw).
    7. **sendReaction:** Reject if not in `REACTION_WHITELIST` from `@/types/watch-together`. Returns silently.
    8. **ReactionEvent ring buffer:** Track the last 20 reactions in `reactions.value` for the BurstOverlay. Each entry: `{ id: number; emoji: string; user_id: string; ts: number }`. After 5000ms, prune via a single shared `setInterval` (or rAF loop) so BurstOverlay can iterate without managing its own state.
    9. **onUnmounted hook:** Automatically calls `disconnect()`. The composable supports being created outside a setup() (e.g. unit tests) — only register `onUnmounted` if `getCurrentInstance()` returns truthy.
    10. **JSON parse errors / unknown message types:** Log via `console.warn('[watch-together] bad frame', ...)` once per type per session (use a Set to dedupe noise). Do NOT crash the composable.
    11. **AUTH_EXPIRED error code:** When received via `onError`, set `connectionStatus='failed'` and emit via the `onError` subscribers but do NOT reconnect (handled by the view → router redirect).
    12. **CAPACITY_FULL / ROOM_NOT_FOUND:** WS will close after the error frame (per Phase 1 semantics). Set `connectionStatus='failed'`. Do NOT reconnect.
