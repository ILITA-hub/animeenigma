  <behavior>
    Props:
    - `messages: ChatMessage[]`
    - `sendChat: (body: string) => void` (passed in from parent; the parent extracts it from the composable's return)

    Renders:
    - `<section aria-label="…">` with vertical-flex layout
    - Heading `<h3 class="font-semibold">{{ $t('watch_together.title') }}</h3>` (or a separate chat heading if the design needs it — use `watch_together.title` for v1.0 since chat IS the room view)
    - Scrollable list `<ul ref="listRef" class="flex-1 overflow-y-auto">` of messages. Each message:
      - `<li class="…">`
      - Sender: `<span class="font-medium">{{ msg.username }}</span>`
      - Body: `<p class="text-sm">{{ msg.body }}</p>`
      - Timestamp: `<time>{{ formatTs(msg.ts) }}</time>` (short HH:MM format)
    - Empty state: `<p>{{ $t('watch_together.empty_chat') }}</p>` when `messages.length === 0`
    - Input row at bottom:
      - `<textarea v-model="draft" maxlength="500" rows="1" :placeholder="$t('watch_together.chat_input_placeholder')" :aria-label="$t('watch_together.chat_input_aria')" />` (auto-grow optional)
      - Send button `<button type="button" :disabled="!canSend">{{ $t('watch_together.send_button') }}</button>` — disabled when draft is empty after trim OR draft.length > 500
      - Char counter `<span aria-live="polite">{{ $t('watch_together.chat_char_count', { n: draft.length }) }}</span>` ONLY when `draft.length > 400` (don't clutter at low counts)
    - Send on Enter (not Shift+Enter — Shift+Enter inserts a newline): keydown handler on the textarea
    - On send: call `props.sendChat(draft.trim())`, clear `draft`, scroll list to bottom
    - Auto-scroll: when `messages.length` increases AND user was already at-bottom (within 50px tolerance), scroll to bottom on `nextTick`. If user scrolled up, do NOT yank.
    - Padding: `p-4 md:p-6` on outer section
    - Font weights: font-medium and font-semibold only

    Behaviors validated in spec:
    - Test 1: 3 messages → 3 `<li>` items rendered with username + body + time
    - Test 2: Empty messages → `watch_together.empty_chat` rendered
    - Test 3: Typing in textarea updates draft; clicking send calls `props.sendChat(trimmedBody)` exactly once
    - Test 4: After send, the textarea is cleared
    - Test 5: Empty draft → send button is disabled
    - Test 6: Whitespace-only draft (e.g. `'   '`) → send button is disabled
    - Test 7: 501-char draft → send button disabled (the textarea's `maxlength=500` should prevent this in real life; the test asserts the disabled state)
    - Test 8: Enter key (without Shift) calls sendChat; Shift+Enter does not call sendChat
    - Test 9: Char counter visible only when `draft.length > 400`
    - Test 10: No `font-bold` in rendered HTML
  </behavior>
