# Phase 02 Plan 02.4: MemberList.vue + ChatPanel.vue Summary

Shipped the two "social-information" sidebar components for Watch Together — the member roster and the chat round-trip surface. Pure presentational components that consume `useWatchTogetherRoom` reactive refs from a parent and stay decoupled from the WebSocket layer.

## Prop Signatures (exact)

### MemberList.vue
```typescript
defineProps<{
  members: Member[]      // from composable's members.value
  hostUserId: string     // from composable's room.value?.host_user_id
}>()
```

**No emits.** Reads `useAuthStore().user?.id` internally for the `(you)` badge.

### ChatPanel.vue
```typescript
defineProps<{
  messages: ChatMessage[]   // from composable's messages.value
  currentUserId: string     // own user id for bubble alignment
}>()

defineEmits<{
  send: [body: string]      // trimmed body; parent wires this to composable.sendChat
}>()
```

## Key Behaviors

### MemberList
- Heading renders `Members ({count})` using `t('watch_together.members_heading')`
- Each `<li>`: avatar (`<img>` if `meta.avatar_url`, else first-letter chip) + username + optional `(host)` and `(you)` badges
- A single member can show BOTH badges if they're the host AND the local user
- Empty state renders a subdued `—` (no i18n key for a placeholder dash, per plan)

### ChatPanel
- Layout: `flex flex-col h-full` so scrollable list + input row sit cleanly inside a sidebar
- Message list `<ul ref="listRef" class="flex-1 min-h-0 overflow-y-auto">`
- Each message: username (font-medium) + locale-aware HH:MM timestamp + body (whitespace-pre-wrap so newlines render)
- Send-button disabled state: empty or whitespace-only OR over 500 chars (textarea `maxlength` enforces the over-limit case)
- Send on Enter; Shift+Enter inserts a newline (`event.preventDefault()` only when Shift is unset)
- After successful send: emit `send`, clear `draft`
- Char counter visible only when `draft.length > 400` — uses `t('watch_together.chat_char_count', { n: draft.length })` (i18n message has `{n}/500`)
- Auto-scroll: when `props.messages.length` increases AND `isAtBottom(listRef)` was true before the update, scroll to bottom on `nextTick`. If user scrolled up beyond 50px tolerance, do not yank.

## Auto-scroll tolerance: 50px

A reader is considered "at bottom" when `scrollHeight - scrollTop - clientHeight < 50`. This tolerance gives a small margin so a user who is reading near (but not exactly at) the bottom still gets auto-scrolled, while a deliberate scroll-back (more than 50px up) blocks auto-scroll.

## Char counter threshold: 400

The char counter is hidden until `draft.length > 400`. At under 80% of the cap, the counter is noise; near the cap, it becomes useful warning.

## Deviations from Plan

None — plan executed as written, with one harmless adaptation:

**Adaptation: ChatPanel emits `send(body)` instead of receiving `sendChat` as a prop.**
The user prompt explicitly directed `Emits: send: (body: string) => void` while the plan suggested a `sendChat` prop. Emit-based design is cleaner (parent wires `@send="sendChat"` in RoomSidebar) and matches Vue 3 idioms more naturally. Both achieve the same architecture (composable owns WS, panel is presentational).
