  <behavior>
    Props:
    - `animeId: string`
    - `episodeId: string`
    - `player: PlayerKind`
    - `translationId: string`

    Renders:
    - `<button type="button" :disabled="loading" :aria-label="$t('watch_together.invite_button_label')" class="…">{{ $t('watch_together.invite_button_label') }}</button>`
    - On disabled state (during createRoom flight), show a spinner inside the button
    - Tailwind: `font-medium` text, padding `px-4 py-2`, color e.g. `bg-cyan-500/20 text-cyan-300 hover:bg-cyan-500/30 rounded-lg`
    - No `font-bold`

    Behavior on click:
    1. Set `loading=true`
    2. Call `createRoom({ anime_id: props.animeId, episode_id: props.episodeId, player: props.player, translation_id: props.translationId })`
    3. On success:
       a. `router.push(\`/watch/room/${response.room_id}\`)`
       b. Attempt `navigator.clipboard.writeText(response.invite_url)`. On success → `useToast().push($t('watch_together.invite_copied_toast'), 'success', 4000)`.
       c. On clipboard failure (no `navigator.clipboard` OR HTTPS-required denial) → `useToast().push($t('watch_together.invite_copy_manual') + ' ' + response.invite_url, 'info', 8000)` (fallback per CONTEXT.md §Clipboard write).
    4. On createRoom failure: `useToast().push($t('errors.serverError'), 'error', 4000)` (use the existing `errors.serverError` key — already in en.json/ru.json — to avoid adding more i18n)
    5. Set `loading=false` in a finally clause

    Behaviors validated in spec:
    - Test 1: Click triggers `createRoom` once with the props as payload
    - Test 2: On success, `router.push('/watch/room/<room_id>')` called once
    - Test 3: On success, `navigator.clipboard.writeText(invite_url)` called
    - Test 4: On clipboard success, toast.push called with the `invite_copied_toast` message and `'success'` type
    - Test 5: On clipboard absent (`vi.stubGlobal('navigator', {})`), toast falls back to manual-copy message
    - Test 6: On createRoom failure (rejected promise), toast.push called with `'error'` type
    - Test 7: While createRoom is in flight, button is disabled
    - Test 8: After createRoom resolves OR rejects, button is re-enabled (loading=false)
    - Test 9: No `font-bold` in rendered HTML
  </behavior>
