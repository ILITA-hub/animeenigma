Added a stable consumer-facing type alias to the composable from Plan 02.3:


## Deviations from Plan

None — the plan was followed exactly. The only minor stylistic choice: the void-reference + comment was placed immediately AFTER the `defineProps<…>()` closing line (not "near other top-of-script comments") because every player's script-setup has a different layout of `defineEmits`, `useAuthStore`, `useI18n`, etc. right after `defineProps`, and inserting the reference there would have created an inconsistent location across the 5 files. Placing it directly after `defineProps` gives Phase 3 a uniform anchor across all 5 players.
