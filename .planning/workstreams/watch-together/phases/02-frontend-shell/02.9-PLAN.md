From the existing `Anime.vue` (script setup state inspected via grep — these are the reactive sources for the props):
- `anime.id` — the UUID (animeId)
- `selectedEpisode` (in player components) — Anime.vue needs to track the active episode. From the existing reactive state for the active player provider, the episode is held by the child player. For Phase 2 we'll surface a simple bridge: emit / prop from the player back up, OR mount InviteButton with a "current episode" sourced from `resumeStartEpisode` (the parent-owned hint). Conservative choice: use `String(resumeStartEpisode ?? 1)` as the episode_id placeholder for v1.0. Phase 4 (State Switching) will revisit this when actual episode propagation lands.
- `videoProvider` — one of `'kodik' | 'animelib' | 'ourenglish' | 'hanime' | 'raw'` — matches `PlayerKind` exactly. Note the lowercase value in Anime.vue:383-431.
- Translation ID: not always tracked at the Anime.vue level. For Phase 2, pass an empty string `""` if no specific translation is known. The backend stores `translation_id` as a free string and we don't actively switch on it in this phase. Phase 4 wires the real value.
