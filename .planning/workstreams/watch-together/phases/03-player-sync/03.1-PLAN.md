// PlaybackEventData shape (server → client):
{ kind: 'play' | 'pause' | 'seek', time: number, by_user_id: string, server_ts: number }

// PlaybackCorrectionData shape (server → client, per-recipient):
{ time: number, server_ts: number }

NOTE: The composable's existing re-emission guard suppresses `playback:event` echoes where `by_user_id === ownUserId` BEFORE calling subscribers. So `onPlaybackEvent` handlers receive ONLY remote events. The bridge still needs its OWN guard against the local-DOM-event storm: when we call `video.play()` programmatically, the browser fires a native `play` event which the bridge MUST NOT re-emit back to the room.

    - emit-on-local-pause: same pattern for `pause` → `room.emitPause(video.currentTime)`

    - apply-on-remote-seek: `kind: 'seek'` → set guard, `video.currentTime = event.time`, clear guard after next native `seeked` OR 250ms timer
    - time-tick-heartbeat: while video is playing (NOT paused, NOT ended, NOT seeking), emit `room.emitTimeTick(video.currentTime)` once per second via a `requestAnimationFrame` loop with a `lastTick = 0` gate. Stops on pause/ended/unmount.

    - cleanup-on-unmount: removeEventListener for all native handlers, cancel RAF loop, restore playbackRate to 1.0, unsubscribe room handlers, clear any pending guard timeout.
    - null-videoRef-tolerance: if videoRef.value is null at any point (player unmounts before bridge), bridge is a no-op. NEVER throw on null.
    - null-room-tolerance: signature accepts non-null room only — players gate the call site with `if (props.room) usePlayerSyncBridge(...)`. Bridge does NOT defend against null room (would be a programming error).

Create `frontend/web/src/composables/usePlayerSyncBridge.ts` exporting `usePlayerSyncBridge(videoRef: Ref<HTMLVideoElement | null>, room: WatchTogetherRoomHandle): void`.

Module shape: a single named export, no default export. Co-locate small helpers (`computeTargetTime`, `now`) inside the file; do NOT extract to a separate utility module — keeps the bridge self-contained and audit-friendly.

Internal state (closure-scoped per invocation; one bridge instance per player):
- `let applyingRemote = false` — guard flag set true around programmatic video calls
- `let applyingRemoteClearTimer: ReturnType<typeof setTimeout> | null = null` — fallback clear if native event never fires
- `let lastAppliedTime = -1` — last currentTime we applied from a remote event; native `seeked` events firing within ±0.5s of this value AND while `applyingRemote=true` are swallowed
- `let lastTickAt = 0` — wall-clock ms of last emitTimeTick
- `let rafId: number | null = null` — requestAnimationFrame handle
- `let playbackRateRestoreTimer: ReturnType<typeof setTimeout> | null = null` — pending soft-correction restore
- `const unsubscribers: Array<() => void> = []` — collected on subscribe, drained on unmount

Constants at the top:
- `const DRIFT_SOFT_THRESHOLD = 1.0` (seconds)
- `const TICK_INTERVAL_MS = 1000`
- `const APPLY_GUARD_TIMEOUT_MS = 250`
- `const SOFT_CORRECTION_RATE_SLOW = 0.97`
- `const SOFT_CORRECTION_RATE_FAST = 1.03`
- `const SOFT_CORRECTION_DURATION_MS = 5000`
- `const SEEK_GUARD_TOLERANCE = 0.5` (seconds)

Helper `withApplyGuard(fn: () => void)`:
1. `applyingRemote = true`
2. If `applyingRemoteClearTimer` is non-null, clearTimeout(it)
3. Call `fn()`
4. Schedule `applyingRemoteClearTimer = setTimeout(() => { applyingRemote = false }, APPLY_GUARD_TIMEOUT_MS)`

Helper `computeTargetTime(correctionTime: number, serverTs: number): number`:
- Returns `correctionTime + (Date.now() - serverTs) / 1000`
- Used by both soft and hard correction paths

Native event handlers (attach in onMounted via `watch(videoRef, ...)` since videoRef is a template ref that mounts after the component, OR use the `videoRef` watcher pattern from `HanimePlayer.vue` lines 416-417 — both work; pick the watcher pattern for symmetry with existing players):
- `handlePlay`: if `applyingRemote || video.paused` (defensive) → no-op. Else `room.emitPlay(video.currentTime)` and start the tick loop.
- `handlePause`: if `applyingRemote` → no-op. Else `room.emitPause(video.currentTime)` and stop the tick loop.
- `handleSeeked`: if `applyingRemote` → no-op. Else if `Math.abs(video.currentTime - lastAppliedTime) < SEEK_GUARD_TOLERANCE` → no-op (lingering native event from a recent apply). Else `room.emitSeek(video.currentTime)`.
- `handleEnded`: stop the tick loop. No emit (the room can derive "ended" from time vs duration if it ever needs to; v1 doesn't).

Remote subscription handlers (registered immediately on bridge invocation, even if videoRef.value is still null — the handler dereferences videoRef.value lazily and no-ops on null):
- `room.onPlaybackEvent(handler)`:
  - `play`: `withApplyGuard(() => { if (Math.abs(v.currentTime - e.time) > DRIFT_SOFT_THRESHOLD) { v.currentTime = e.time; lastAppliedTime = e.time }; v.play().catch(() => {}) })`. Start tick loop.
  - `pause`: `withApplyGuard(() => { v.pause(); v.currentTime = e.time; lastAppliedTime = e.time })`. Stop tick loop.
  - `seek`: `withApplyGuard(() => { v.currentTime = e.time; lastAppliedTime = e.time })`. Tick loop state unchanged.
- `room.onCorrection(handler)`:
  - `target = computeTargetTime(c.time, c.server_ts)`
  - `drift = Math.abs(v.currentTime - target)`
  - If `drift < DRIFT_SOFT_THRESHOLD`:
    - If `v.currentTime < target`: `v.playbackRate = SOFT_CORRECTION_RATE_FAST`
    - Else: `v.playbackRate = SOFT_CORRECTION_RATE_SLOW`
    - Clear any prior `playbackRateRestoreTimer`, schedule restore to 1.0 after `SOFT_CORRECTION_DURATION_MS`
  - Else (hard): `withApplyGuard(() => { v.currentTime = target; lastAppliedTime = target })`. NO toast — corrections are silent per WT-SYNC-06.

Tick loop:
- `tickLoop = () => { rafId = requestAnimationFrame(tickLoop); const v = videoRef.value; if (!v || v.paused || v.ended || v.seeking) return; const now = Date.now(); if (now - lastTickAt < TICK_INTERVAL_MS) return; lastTickAt = now; room.emitTimeTick(v.currentTime) }`
- `startTickLoop()`: if rafId === null, rafId = requestAnimationFrame(tickLoop)
- `stopTickLoop()`: if rafId !== null, cancelAnimationFrame(rafId); rafId = null

Lifecycle:
- Watch `videoRef` with `immediate: true`: when value transitions null→element, attach all 4 native listeners. When element→null (replacement), remove all 4 listeners from old element.
- `onBeforeUnmount`: stopTickLoop, clearTimeout(applyingRemoteClearTimer), clearTimeout(playbackRateRestoreTimer), restore `videoRef.value.playbackRate = 1.0` if it exists and is not 1.0, drain `unsubscribers` array.

Import surface (top of file):
```typescript
import { onBeforeUnmount, watch, type Ref } from 'vue'
import type { WatchTogetherRoomHandle } from '@/composables/useWatchTogetherRoom'
```
