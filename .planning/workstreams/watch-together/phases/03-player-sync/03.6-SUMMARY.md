# Phase 03 Plan 03.6: Two-browser sync end-to-end Playwright spec — Summary

End-to-end Playwright spec exercising Watch Together play/pause/seek mirroring across two browser contexts for all 5 player kinds, plus drift correction under simulated CPU throttling. The spec is the canonical Phase 3 regression gate (WT-SYNC-09).

### Spec structure mirrors Phase 2 shell

- **Pre-flight gateway probe** (`test.beforeAll` → `isStackUp(request)`): `GET /api/anime/_/scraper/health` + a probe `POST /api/auth/login`. Whole suite `test.skip`s when the local stack is down, identical pattern to `watch-together-shell.spec.ts`. CI-runnable without a hard fail when there's no backend.
- **Auth** (`loginAs`): `POST /api/auth/login` with `ui_audit_bot` / `audit_bot_test_password_2026` from CLAUDE.md's "UI Audit Test User" section. Response shape matches the project's `{ data: { access_token, user } }` envelope (NOT the `{ token, user }` shape the plan's draft snippet assumed — the spec adapts to the real envelope).
- **Ephemeral B-user** (`registerEphemeralUser`): `POST /api/auth/register` with `wt_sync_e2e_<ms>_<rand>` username + a long fixed password. Same shape as Phase 2 spec.
- **Dynamic anime UUID** (`resolveSeededAnimeId`): `GET /api/users/me/anime-list?status=watching` on the bot's JWT, takes first item's `anime_id`/`anime.id`/`id`. Module-level memoization (`cachedAnimeId`) so the 6 tests reuse one lookup.
- **Room creation** (`createRoom`): `POST /api/watch-together/rooms` with `{ anime_id, episode_id: '1', player, translation_id: '' }`. Accepts both `data: { room_id, invite_url }` and bare-top-level shapes.

### Selectors and adaptive points

- **HTML5 video selector:** `page.locator('video').first()` — works across all 4 HTML5 player components since each renders exactly one `<video>` element at the surface (subtitle overlay is `<div>`, controls are siblings).
- **Kodik iframe:** `page.locator('iframe').first()` (NOT `iframe[src*="kodik"]`) — the Watch Together room view loads the player by `kind` and the iframe is the only iframe in the player wrapper; widening the selector future-proofs against bundle URL changes the way the kodik-rpc-probe canary doesn't need to (the canary scopes by `iframe[src*="kodik"]` to be defensive against non-Kodik iframes appearing elsewhere on the catalog page; in the WT room view the player is alone).
- **Kodik fallback banner:** matches both `Kodik sync unavailable` (en) and `Синхронизация Kodik недоступна` (ru) — exact i18n strings will be wired in 03.5 (sender-attribution toasts + connection overlay landed in 03.5 commits `baaec58` + `696c1e7`; the banner text will live in the 03.5 i18n bundle once it merges). If the i18n key name shifts during 03.5 integration, the regex needs a quick refresh — the spec catches that as a `.isVisible({ timeout: 500 })` false-negative which lands in the `else` branch (continue with pause/seek assertions) rather than failing, so it degrades gracefully.

### 2. [Rule 1 — Bug] Auth response envelope shape

**Found during:** Adapting plan's draft snippet to the project's actual auth API.

**Issue:** The plan's draft assumed `body.token` + `body.user`, but the AnimeEnigma auth service responds with `{ data: { access_token, user } }` (verified against `watch-together-shell.spec.ts`'s `loginAs` and the gateway envelope helpers). Using the plan's shape would yield "no token/user in response" errors at runtime.

**Fix:** The spec's `loginAs` and `registerEphemeralUser` follow the same defensive `body.data ?? body` unwrap and `access_token` field name that the Phase 2 spec uses.

**Commit:** absorbed into `696c1e7` (same caveat as Deviation 1).

### 3. [Rule 1 — Bug] ESLint `no-extra-semi` on cast-to-IIFE pattern

**Found during:** Lint gate after writing the spec.

**Issue:** Two occurrences of the defensive `;(window as unknown as WindowWithSync).__wtSyncEvents = []` pattern (copied from kodik-rpc-probe.spec.ts) triggered `no-extra-semi` because in this codebase's parser the preceding `interface` declaration is followed by the cast-assignment without ASI ambiguity.

**Fix:** Rewrote both occurrences as `const w = window as unknown as WindowWithSync; w.__wtSyncEvents = []` — equivalent runtime semantics, no leading-semicolon hack needed.

**Commit:** absorbed into `696c1e7`.

## Decisions Made

- **6 distinct test blocks, not parameterize loop:** the Kodik test diverges materially from the HTML5 tests (iframe + postMessage RPC vs. `<video>` API), and the drift-correction test diverges from both (CDP throttling, single-player). A `for` loop unifies only the 4 HTML5 sub-tests; the rest are explicit. This matches the plan's preference for "one block per player kind".
- **2000ms poll timeout, not 500ms:** the 500ms target is the user-facing "feels instant" threshold, but propagation across WS → server → WS → adapter → DOM under CI scheduling pressure routinely takes ~300-800ms. 4× slack at 2000ms catches real regressions (>2s = a bug) without flaking on CI scheduling jitter.
- **Module-level `cachedAnimeId` memoization:** avoids 6× duplicate `GET /api/users/me/anime-list?status=watching` calls. The seeded watching list doesn't change between tests in a single Playwright run.
- **Kodik fallback banner regex covers en + ru:** the spec doesn't pin a locale; both locales should pass the same test.
