<objective>
Add a new GET `/internal/anime/{shikimoriId}/episodes/validate` endpoint to the catalog service. This endpoint backs Phase 4's `state:change_*` validation in `services/watch-together/`: given an anime + player + (optional) translation + episode_id, return `{valid: bool, reason: string}`. The watch-together service will call it before broadcasting `room:state_changed`.

Purpose: provide the catalog-side validation surface mandated by WT-STATE-02. The existing `/internal/anime/{shikimoriId}/episodes` endpoint (notifications detector — NOTIF-DET-01) only supports kodik|animelib and only returns a "latest episode" int — not enough for episode-exists semantics across all 5 players. We add a sibling endpoint instead of modifying the existing one so the notifications detector's contract stays frozen.

Output: a new handler + service + router wiring + tests. Endpoint mounted at the root router (no middleware) per the existing internal-endpoint security model (Docker-network-only; gateway does not proxy `/internal/*`).
</objective>
