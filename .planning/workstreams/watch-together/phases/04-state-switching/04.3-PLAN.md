    - Call catalog.ValidateEpisode(ctx, room.AnimeID, room.Player, episode_id, room.TranslationID, "")
    - On transport error: log warn, send error{code: EPISODE_UNAVAILABLE, message: "validation upstream failed"} sender-only, do NOT mutate Redis
    - On ValidateResult{Valid: false, Reason: "EPISODE_UNAVAILABLE"}: send error{code: EPISODE_UNAVAILABLE} sender-only, do NOT mutate Redis
    - On ValidateResult{Valid: false, Reason: "TRANSLATION_UNAVAILABLE"}: send error{code: TRANSLATION_UNAVAILABLE} sender-only, do NOT mutate Redis (catalog rejected because translation no longer yields content)
    - On Valid=true: HSET episode_id=X, playback_time=0, playback_state="paused", playback_time_updated_at=now; broadcast room:state_changed{field:"episode_id", value:"X", by_user_id} to ALL

    **state:change_player {player: "kodik"}**:
    - Pre-check: payload.player MUST be in {kodik, animelib, ourenglish, hanime, raw}. If not → sendBadPayload-style "invalid player value"
    - Read current Room (need anime_id)
    - Call catalog.ValidateEpisode(ctx, room.AnimeID, payload.player, /*episode_id*/ "", /*translation_id*/ "", "")
      - translation_id="" triggers the "player-change mode" in the catalog endpoint — returns Valid if the anime has any episode for that player
    - On transport error / Valid=false: send error{code: PLAYER_UNAVAILABLE} sender-only
    - On Valid=true: HSET player=X, translation_id="" (reset — translations are player-specific), episode_id="1" (reset to first episode — see note), playback_time=0, playback_state="paused", playback_time_updated_at=now; broadcast room:state_changed{field:"player", value:"X", by_user_id} to ALL
      - **Note on episode_id reset:** the design says "reset to first episode" but we don't know what "first" is for the new player without a separate parser call. v1.0 strategy: HSET episode_id="1" (string "1") as a sensible default. Frontend players already coerce numeric episode IDs from this; if a player uses non-numeric IDs internally, the player's source-loading logic will resolve to the actual first episode on mount. Document this with an inline comment referencing CONTEXT.md §Claude's Discretion.

    **state:change_translation {translation_id: "Y"}**:
    - Read current Room (need anime_id, player, episode_id)
    - Call catalog.ValidateEpisode(ctx, room.AnimeID, room.Player, room.EpisodeID, payload.translation_id, "")
    - On transport error / Valid=false{Reason: TRANSLATION_UNAVAILABLE or EPISODE_UNAVAILABLE}: send error{code: TRANSLATION_UNAVAILABLE} sender-only
    - On Valid=true: HSET translation_id=Y, playback_time=0, playback_state="paused", playback_time_updated_at=now; broadcast room:state_changed{field:"translation_id", value:"Y", by_user_id} to ALL

    **Transport-error timeout note:** Per the locked design, transient catalog failures NEVER mutate Redis (better to send a sender-only error and let the user retry than to silently broadcast a state the catalog hasn't blessed).
