# Phase 4 Plan 4: WatchTogetherView re-mount on state change + PlayerTabBar + error toasts + i18n Summary

## One-Liner

Adds an in-room 5-tab player switcher (PlayerTabBar) and wires
WatchTogetherView to (a) re-mount the active player cleanly on `room.player`
changes via a `:key` binding, (b) route PlayerTabBar clicks through
`roomHandle.emitChangePlayer` rather than mutating local state, and
(c) surface sender-only EPISODE/PLAYER/TRANSLATION_UNAVAILABLE errors as
i18n-keyed toast notifications. Ships 2 new error code constants + 8 i18n
keys (en + ru, parity-locked).

## Files Changed

### Created

- `frontend/web/src/components/watch-together/PlayerTabBar.vue` ‚Äî 5-tab switcher SFC. Renders one `<button role="tab">` per `PlayerKind`, labels via `t('watch_together.player_tab_<kind>')`. Active tab gains `aria-selected="true"` + `bg-primary text-primary-foreground font-semibold`; inactive tabs use `bg-muted/40 text-foreground/70 font-medium`. Click handler guards same-kind clicks and `disabled` state. UI-SPEC compliant ‚Äî Tailwind utility-only, font-medium / font-semibold only, no `font-bold`.
- `frontend/web/src/components/watch-together/PlayerTabBar.spec.ts` ‚Äî 8 Vitest cases covering render count, i18n label resolution (no raw key string leaks), aria-selected on active, click‚Üíemit, same-tab no-op, disabled-suppress, font-weight guard, full-cycle emit sequence.

### Modified

- `frontend/web/src/types/watch-together.ts` ‚Äî adds `ERR_PLAYER_UNAVAILABLE` + `ERR_TRANSLATION_UNAVAILABLE` constants after the existing `ERR_EPISODE_UNAVAILABLE`; extends the `ErrorCode` union to 9 members.
- `frontend/web/src/api/watch-together.ts` ‚Äî re-exports both new constants alongside the existing error-code re-export block (single canonical import path for consumers).
- `frontend/web/src/views/WatchTogetherView.vue` ‚Äî adds `PlayerTabBar` + `useToast` imports, extends `roomHandle.onError()` with 3 new sender-only error branches BEFORE the CAPACITY_FULL / AUTH_EXPIRED checks, adds `onSelectPlayer()` handler that routes through `roomHandle.emitChangePlayer`, mounts `<PlayerTabBar>` overlaid on the player column, and adds `:key="`player-${livePlayer}`"` to all 5 player v-if/v-else-if branches.
- `frontend/web/src/views/WatchTogetherView.spec.ts` ‚Äî extends `FakeHandle` with `emitChangePlayer`, mocks `useToast` via a hoisted spy, adds a `PlayerTabBarStub` that exposes synthetic `select-player` emitters, extends the `@/api/watch-together` mock with the 3 new error codes, adds 6 new test cases (Tests 14-19) covering player remount on room.player mutation, PlayerTabBar‚ÜíemitChangePlayer routing, same-player no-op, and EPISODE/PLAYER/TRANSLATION_UNAVAILABLE toast surfacing.
- `frontend/web/src/locales/en.json` ‚Äî adds 8 new `watch_together.*` keys (5 `player_tab_*` labels + 3 `state_change_*_unavailable` toast messages).
- `frontend/web/src/locales/ru.json` ‚Äî adds the same 8 keys with Russian translations matching the existing informal-friendly tone.
- `frontend/web/src/locales/__tests__/watch-together-keys.spec.ts` ‚Äî appends 8 new keys to the `expectedKeys` array; the parity check generates `it.each` rows for both locales, so a missing key in either file fails individually rather than as a generic diff.
