# Phase 05 Plan 02: Full Prometheus metrics suite (WT-NF-06) — Summary

Five additional Prometheus metrics registered + emitted at the canonical call sites, completing the WT-NF-06 telemetry suite that the Plan 05.6 Grafana dashboard will consume. Phase 1 shipped the 9 baseline metrics (counters + WS gauges); this plan adds the gauge for live rooms, two distribution histograms (members + chat), the session duration histogram, and breaks out persistent-drift by host-vs-member.

## What landed

### New metrics (services/watch-together/internal/service/metrics.go)

| Metric | Type | Labels | Buckets / Notes |
|---|---|---|---|
| `wt_rooms_active` | Gauge | — | Live count. Inc on Create, Dec at teardown. |
| `wt_members_per_room` | Histogram | — | 1..10 (matches WT-NF-02 cap). Observed at every member:joined broadcast. |
| `wt_chat_messages_per_room` | Histogram | — | 0,1,5,10,25,50,100 (100 = LTRIM cap). Observed at teardown. |
| `wt_session_duration_seconds` | Histogram | — | 60,300,900,1800,3600,7200,14400 (test→binge). Observed at teardown. |
| `wt_persistent_drift_total` | CounterVec | user_role={host,member} | Bumped in handleTimeTick's DriftPersistent branch. |

All five share `wt_` prefix with the existing fleet so they show up grouped in `curl /metrics | grep ^wt_`.

### observeRoomTeardown: the shared seam

`observeRoomTeardown(ctx, *repo.RoomRepo, *logger.Logger, *domain.Room)` is the **single canonical** teardown observation path. It is:

- Called directly from `RoomService.Delete` (this plan).
- **NOT yet called from `GraceManager.fire`** — that's 05.1's file. The function is package-private but exported by signature so 05.1 (or a follow-up) can wire it in one line. See "Open seam for 05.1" below.

The helper is best-effort: a `MessageCount` failure logs at Debug and observes 0, never blocking the actual delete. `RoomsActive.Dec` always runs once per call so the gauge stays honest.

### PersistentDriftTotal label resolution

Done in `inbound.handleTimeTick` rather than in the DriftEngine itself:

```go
case DriftPersistent:
    role := "member"
    if room, gerr := r.repo.GetRoom(ctx, conn.RoomID); gerr == nil && room.HostUserID == conn.UserID {
        role = "host"
    }
    PersistentDriftTotal.WithLabelValues(role).Inc()
    r.sendErrorToSelf(...)
```

This keeps `DriftEngine.OnTimeTick`'s signature stable (Phase 1 tests would otherwise need a touch-up) and the extra `GetRoom` round-trip is acceptable because persistent drift is rare — 5 consecutive hard drifts (> 5s each) — per user per session. GetRoom failure falls back to `"member"` so the counter still advances.

### Open seam for 05.1: wire `observeRoomTeardown` into `GraceManager.fire`

The teardown observation helper is in place but `GraceManager.fire` (services/watch-together/internal/service/grace.go) doesn't call it yet — per the original plan scope I was instructed not to touch grace.go. The wiring is a 4-line change in `fire()`, right before `g.repo.DeleteRoom`:

```go
// before the existing DeleteRoom call:
if room, err := g.repo.GetRoom(ctx, roomID); err == nil {
    observeRoomTeardown(ctx, g.repo, g.log, room)
}
```

Without this, grace-fire teardowns will NOT contribute observations to `wt_chat_messages_per_room` / `wt_session_duration_seconds` and the `wt_rooms_active` gauge will leak (Create++ but no matching Dec when the room expires via grace). 05.1 (or a small follow-up commit) should wire this — the helper is callable as-is.
