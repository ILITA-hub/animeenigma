  <behavior>
    Two behavioral changes + one visual upgrade — all testable.

    A) `MAX_VISIBLE = 8` cap with FIFO drop:
       - Module-scope const at top of `<script setup>`: `const MAX_VISIBLE = 8`.
       - The render-loop iterates over `props.reactions.slice(-MAX_VISIBLE)` instead of the raw array. This means: when a 9th reaction arrives, the oldest (lowest id, first in chronological order) is sliced off and never rendered for this paint.
       - Note WT-POLISH-01 says "Max 8 simultaneous bursts; older bursts drop off when a 9th arrives" — `.slice(-MAX_VISIBLE)` implements this exactly.
       - The composable's own buffer is larger (~20) and lifetime is 5s. The overlay's cap is purely visual; the composable is the source of truth.

    B) 8-column stratified horizontal placement:
       - Replace the `leftFor(id): random in [5, 95]` approach with a stratified picker.
       - Columns: 8 discrete left% values [10, 20, 30, 45, 55, 70, 80, 90] (deliberately non-uniform — slight gap at center 35-45 + 55-70 to break visual symmetry; users perceive "natural" placement, not a grid).
       - Algorithm:
         1. `function columnFor(id: number): number` — uses a module-scoped `Map<number, number>` cache (same pattern as Phase 2's `positions`).
         2. On cache miss, pick a column NOT currently in use by any of the recent (last MAX_VISIBLE) cached entries. If all 8 columns are in use, pick the column with the OLDEST cached entry (FIFO replacement). Document why: this avoids two emojis in the same column when they'd visually overlap.
         3. Implementation: maintain an ordered ring `recent: number[]` (last MAX_VISIBLE ids in insertion order). On insert, push id; if length > MAX_VISIBLE, shift the oldest off and remove its entry from the position map.
       - Bound on map growth: at most MAX_VISIBLE+1 entries at any time. No unbounded growth — fixes the Phase 2 doc-noted "bounded ring buffer ~20 in practice" by making it explicit.

    C) Keyframe upgrade — scale + rise + wiggle + fade in 2.5s:
       - Replace the existing `burst-rise` keyframes with:
         ```css
         @keyframes burst-rise {
           0%   { opacity: 0; transform: translate(0, 0) scale(0.8); }
           15%  { opacity: 1; transform: translate(-4px, -20px) scale(1.2); }
           30%  { transform: translate(4px, -50px) scale(1.0); }
           60%  { transform: translate(-3px, -120px) scale(1.0); }
           100% { opacity: 0; transform: translate(2px, -200px) scale(1.0); }
         }
         ```
         (The translate(x, y) movement is the gentle wiggle on horizontal axis — ±4px maximum so it reads as "natural buoyancy", not "shaking".)
       - Update the .burst class:
         ```css
         .burst {
           bottom: 10%;
           animation: burst-rise 2.5s ease-out forwards;
           will-change: transform, opacity; /* hint the compositor */
         }
         ```
