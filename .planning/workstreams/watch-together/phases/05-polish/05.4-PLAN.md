<objective>
Implement WT-POLISH-03 mobile bottom-sheet layout: at viewport < lg the RoomSidebar collapses to a bottom-anchored panel with a 2-tab bar (Chat / Reactions). Player stays at top full-width. The sheet has collapsed (~80px) and expanded (~60% vh) states with a drag-handle gesture.

Purpose: Phase 2 deliberately deferred mobile layout polish. The result is that on mobile the sidebar stacks below the player and fills full height, pushing the player off-screen the moment the user opens chat. WT-POLISH-03 fixes this by anchoring the sidebar to the bottom and giving it a collapsed default state so the player stays visible while users glance at chat activity.

Output: Updated RoomSidebar.vue with a dual-mode template — desktop right-rail unchanged, mobile bottom-sheet with tabs. New i18n keys (bottom_sheet_tab_chat, bottom_sheet_tab_reactions). Extended spec covering both layouts.
</objective>

    State + behavior:

    - `const activeTab = ref<'chat' | 'reactions'>('chat')`

      - If `key === activeTab && sheetExpanded` → collapse (sheetExpanded = false).
      - Else: `activeTab = key; sheetExpanded = true`.
    - Drag handlers (simple Y-delta accumulator):
      - `let touchStartY = 0; let lastDeltaY = 0`
      - `onTouchStart(e)`: capture `e.touches[0].clientY` to touchStartY; lastDeltaY = 0.
      - `onTouchMove(e)`: `const dy = e.touches[0].clientY - touchStartY; lastDeltaY = dy`. If `Math.abs(dy) > 30`, e.preventDefault() to avoid native scroll fighting us.
      - `onTouchEnd()`: if `lastDeltaY < -50` (dragged UP at least 50px) → sheetExpanded = true. If `lastDeltaY > 50` → sheetExpanded = false. Otherwise no-op (small jitter ignored).
