# Phase 05 Plan 05.5: WatchTogetherView polish — capacity-full + expired redirect + auth-expired modal + i18n Summary

Polished the three exceptional WatchTogetherView states (capacity-full, room-expired redirect, auth-expired modal) per WT-POLISH-04/05/06 and completed the i18n smoke pass per WT-POLISH-07. Phase 2 shipped stub branches for capacity + room-ended; this plan made them feel intentional and added the missing modal + redirect flows.

## One-Liner

Auth-expired now shows a blocking modal (not an immediate redirect), mid-session room:closed redirects to /anime/{id}/watch with toast, REST 410 still shows the friendlier empty page, and lastAnimeId is sessionStorage-cached so WS-only close events have a redirect target.

### Task 2: Composable `onAuthExpired` subscriber (RED → GREEN)

Added `onAuthExpired(handler: () => void): () => void` to the composable's public surface as sugar over `onError` filtered for `ERR_AUTH_EXPIRED`. The view consumes it in place of branching inside the catch-all onError, keeping the modal decoupled from the state-error toast trio (EPISODE/PLAYER/TRANSLATION_UNAVAILABLE) that still flows through `onError`.

Also updated the typed fake handle in `usePlayerSyncBridge.spec.ts` to include `onAuthExpired` — the bridge doesn't subscribe to it, but the return-type contract requires it.

3 new tests pass:
- Test 20: fires when AUTH_EXPIRED frame arrives
- Test 21: does NOT fire for other error codes (CAPACITY_FULL, EPISODE_UNAVAILABLE)
- Test 22: returned unsubscriber detaches the handler

Commits: `3d7f8b8` (RED) + `37bd712` (GREEN).

### Task 3: WatchTogetherView polish branches (RED → GREEN)

Three behavior changes, all confined to `WatchTogetherView.vue`:

1. **ErrorState extended** to include `'auth-expired'`. Phase 2's immediate `router.push` for ERR_AUTH_EXPIRED is replaced by setting `errorState = 'auth-expired'` and rendering a blocking modal. The user clicks the Login button to navigate to `/auth?next=/watch/room/{roomId}` (with `returnUrl` also persisted to sessionStorage — belt-and-suspenders matching the Phase 2 pattern).

2. **Mid-session `onRoomClosed` redirect.** Previously just set `errorState = 'gone'`. Now: toast (`watch_together.room_ended_redirect_toast`, level `info`) + `router.push(/anime/{lastAnimeId}/watch)` (falling back to `/` if no anime id is known). The REST 410 bootstrap path STILL sets `errorState = 'gone'` — distinguishing the stale-URL case from the actively-watching case.

3. **`lastAnimeId` sessionStorage cache.** Persisted on bootstrap success via the per-room key `wt-last-anime-id-{roomId}` so the WS-only room:closed path has a redirect target even if the composable's `room` ref was cleared first. Removed on successful redirect. Wrapped in try/catch (privacy modes throw).

Modal a11y:
- `role="dialog"`
- `aria-modal="true"`
- `aria-labelledby` → static id `wt-auth-expired-modal-title`
- Login button has `data-testid="wt-auth-expired-login"` for e2e wiring

Tailwind utility-only; font weights restricted to `font-medium` / `font-semibold` per CLAUDE.md UI-SPEC.

7 new tests + 1 revised:
- Test 11 (revised): AUTH_EXPIRED no longer fires immediate router.push; modal is shown
- Test 20: capacity_full keys resolve (locale copy locked by parity test)
- Test 21: mid-session onRoomClosed → router.push + toast
- Test 22: onRoomClosed without animeId falls back to "/"
- Test 23: REST 410 bootstrap shows "gone" empty state (no redirect)
- Test 24: auth-expired modal a11y (role=dialog, aria-modal=true)
- Test 25: Login click pushes /auth?next=… and writes returnUrl
- Test 26: bootstrap persists lastAnimeId to sessionStorage

Commits: `25bfb13` (RED) + `68b9892` (GREEN).

## Deviations from Plan

### 1. capacity_full_title already had "(10/10)" suffix

**Found during:** Task 1 / Task 3 prep
**Issue:** Plan §Task 3.B assumed capacity_full_title might need a "(10/10)" suffix added. Actual: en.json line 1009 was already `"Room is full (10/10)"` and ru.json line 1009 was `"Комната заполнена (10/10)"` from Phase 2 (Plan 02.8) shipping.
**Fix:** No copy edit needed — the locked-in copy is already correct. Test 20 just asserts the i18n key resolves; locale-specific copy is locked by the parity test.
**Files modified:** None (this deviation REMOVED scope rather than adding it).

### 2. usePlayerSyncBridge.spec.ts FakeHandle needed onAuthExpired (tsc surface)

**Found during:** Task 2 GREEN verification (`bunx tsc --noEmit`)
**Issue:** [Rule 3 - Blocking] Adding `onAuthExpired` to `UseWatchTogetherRoomReturn` broke the type of the typed fake handle in `usePlayerSyncBridge.spec.ts` — the bridge doesn't subscribe to it, but the return-type contract is satisfied by all callers.
**Fix:** Added `onAuthExpired: vi.fn(() => () => {})` to the fake handle in `usePlayerSyncBridge.spec.ts`.
**Files modified:** `frontend/web/src/composables/__tests__/usePlayerSyncBridge.spec.ts`
**Commit:** `37bd712` (folded into GREEN since it was a 3-line type-only fix)

### 3. Plan referenced `lastError` for view composition; in practice the view uses dedicated subscribers

**Found during:** Task 3 design
**Issue:** Plan frontmatter mentioned composing view state via `room.lastError` OR `onAuthExpired`. The composable's existing `lastError` ref is fine but is a write-once-per-error state; subscribing via callbacks (`onError` + `onAuthExpired` + `onRoomClosed`) is cleaner because it fires on EACH event vs requiring a watch on lastError.
**Decision:** Used callback subscribers exclusively (matches Phase 2 pattern). `lastError` remains exposed for any future watcher consumers but not used by the view.
