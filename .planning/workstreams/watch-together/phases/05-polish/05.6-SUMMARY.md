|---|------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------|

**2. [Rule 2 - Coverage] Added 13th panel for Phase 1 baseline metrics**
- **Found during:** Task 1 metric-coverage audit
- **Issue:** The plan's 12-panel layout left `wt_ws_messages_received_total{type}` and `wt_ws_messages_dropped_total` (Phase 1 baseline metrics shipping since 01.x) without any panel coverage. The plan's own `<read_first>` validation step (`grep -oE 'wt_[a-z_]+' ... | sort -u | while read m; do ...`) would have logged these as referenced-but-then-this-was-flipped — i.e. emitted-by-code but absent from the dashboard.
- **Fix:** Added panel 13 "Inbound WS Message Rate by Type (+ dropped)" — full-width timeseries breaking out inbound message types and overlaying the drop counter. Closes the coverage gap.
- **Files modified:** `infra/grafana/dashboards/watch-together.json` (additive only)
- **Commit:** 5b1269c

### Schema version note (NOT a deviation)

The plan's `<behavior>` block specified `"schemaVersion": 39`, but both pre-existing dashboards (`scraper-provider-health.json`, `library.json`) use `38`. Locked to 38 for consistency with the codebase; Grafana 10.3.x reads both versions identically. Documented in `decisions` frontmatter.

## Auto-Provisioning Confirmation

Per `infra/grafana/dashboards/README.md` §"Adding a new dashboard", the file provisioner mounts this directory directly into the dev Grafana container (see `docker/docker-compose.yml` `grafana.volumes`). After `docker compose -f docker/docker-compose.yml restart grafana`, the new dashboard auto-imports into the **Self-Healing** folder (per existing provisioner config) under the title **"Watch Together"** with UID `watch-together`. No manual import step is required.
