# Лудка — ранбук открытия для всех пользователей

Сейчас лудка полностью работает на проде, но **видна только админам** (dark-ship,
spec §12 в `docs/superpowers/specs/2026-06-09-gacha-ludka-design.md`). Открытие —
это флип двух env-переменных, никаких изменений кода.

## Перед открытием (чек-лист контента)

1. В `/admin/gacha` создано достаточно карточек ВСЕХ четырёх редкостей
   (N/R/SR/SSR) — пустой тир перераспределяет вес вниз, но лудка без SSR скучна.
2. Существует ровно один баннер с галкой **«Стандартный»**, он включён и в нём
   лежат карты (это вечный пул; без него страница лудки пуста).
3. У включённых баннеров загружены **задники** (иначе слайд — градиентная заглушка).
4. Прокликан полный цикл под админом: крутка ×1/×10 → церемония → 3D-просмотр →
   сводка → коллекция в профиле → ежедневный бонус.

## Открытие (5 минут)

```bash
# 1. В docker/.env добавить (или раскомментировать) две строки:
GACHA_ADMIN_ONLY=false        # бэкенд-гейт: /api/gacha/* для всех авторизованных
VITE_GACHA_ADMIN_ONLY=false   # фронт-гейт: навбар/страница/коллекция для всех

# 2. Перезапустить gateway (флаг читается на старте; пересборка НЕ нужна):
make restart-gateway

# 3. Пересобрать фронт (флаг запекается в бандл на этапе сборки):
make redeploy-web
```

Админ-CRUD (`/api/gacha/admin/*`, `/admin/gacha`) при этом ОСТАЁТСЯ только для
админов — он гейтится отдельно и от этих флагов не зависит.

## Проверка после флипа

```bash
# обычный (не-админ) пользователь теперь получает 200, а не 403:
KEY=$(grep '^UI_AUDIT_API_KEY=' docker/.env | cut -d= -f2)
curl -s -o /dev/null -w "%{http_code}\n" -H "Authorization: Bearer $KEY" \
  http://localhost:8000/api/gacha/wallet        # ждём 200
# без токена по-прежнему 401; админ-роуты для не-админа по-прежнему 403:
curl -s -o /dev/null -w "%{http_code}\n" -H "Authorization: Bearer $KEY" \
  http://localhost:8000/api/gacha/admin/cards   # ждём 403
```

В инкогнито-окне: пункт «Лудка» не виден гостю, появляется после логина.

## Откат (если что-то пошло не так)

Убрать обе строки из `docker/.env` (или поставить `true`) → `make
restart-gateway` → `make redeploy-web`. Лудка снова видна только админам; данные
пользователей (кошельки/коллекции) не трогаются.
