# AnimeEnigma — Backend & Infra Audit Report

_Generated by a 27-unit multi-agent review (every Go service, shared libs, Python sidecars, docker-compose, Dockerfiles, observability stack, k8s/helm) + repo-wide security & concurrency sweeps. Each finding adversarially verified against current code. 210 confirmed findings._

## Executive summary

The backend and infra are functionally rich but carry a concentrated set of high-severity correctness, security, and resource-leak defects that cluster around a few shared seams. The single most dangerous theme is the streaming/HLS proxy auth boundary: a duplicate CORS header that breaks playback, an unanchored prefix-wildcard allowlist plus a hardcoded provenance HMAC default and no private-IP guard that together form an exploitable SSRF/open-proxy class, and a publicly-proxied unauthenticated stream-token minter. Deployment is the second fault line — production runs the un-hardened base compose with weak hardcoded secrets (postgres/minioadmin/admin), the hardened prod overlay is orphaned, container logs are uncapped and filling disk now, the k8s/helm trees are non-deployable (web crashloops, datastores and ~8 services absent, plaintext committed secrets), and a fleet-wide DBStats counter bug makes pool metrics unusable. A recurring authz gap lets unauthenticated callers reach admin/internal writes (catalog pin-translation, batch-refresh, scheduler triggers, maintenance auto-apply that can write+deploy+git-push from end-user report text). Performance debt concentrates in uncached hot paths recomputed per request (player heartbeat write-amplification, recs full-catalog ensemble with no singleflight, watch-together per-tick Redis reads) and a broad absence of supporting DB indexes on ORDER BY/filter columns across catalog/player/scheduler/library. Most issues are gated by the small self-hosted threat model, but the deploy-secrets, log-rotation, HLS-CORS, and DBStats items are live or near-live. The good news: a large fraction are 1-3 effort fixes (timeouts, drains, skip-list edits, secret env refs) with outsized impact.

**Severity counts:** critical 3 · high 34 · medium 70 · low 90 · info 13

## Cross-cutting themes

- HLS-proxy/videoutils auth boundary is structurally weak: duplicate-ACAO playback break, unanchored prefix-wildcard allowlist, hardcoded provenance HMAC default, no private-IP/loopback/metadata SSRF guard anywhere in the package, bucket-unpinned presign, and redirect targets never re-validated — together a coherent SSRF/open-proxy class on the public unauthenticated proxy.
- Deployment is a second fault line: production runs the un-hardened base compose with weak hardcoded secrets while the hardened prod overlay is orphaned; no log rotation (filling disk now); no resource limits on 31/34 services; the k8s/helm trees are non-deployable (web crashloop, missing datastores+services, committed plaintext secrets, :latest images, no pull secrets/HPA/PDB/NetworkPolicy).
- Recurring authz/exposure gap: unauthenticated or internal-but-network-exposed endpoints reach privileged actions — catalog pin-translation/batch-refresh/calendar-sync, scheduler job triggers, streaming /internal token minter through the gateway, maintenance /api/reports → autonomous write+deploy+git-push, notifications internal arbitrary user_id.
- Shared CORS middleware emits reflected-origin + Allow-Credentials:true under '*' across 13+ services (auth/recs serve per-user data) — a single shared-lib fix point.
- Uncached recompute on hot paths recomputed identically per request: player heartbeat preference write-amplification, anonymous resolve community aggregation, recs full-catalog ensemble (no singleflight, crons never write the envelope), watch-together per-tick Redis HGETALL, gateway uncached status fan-out.
- Pervasive missing DB indexes on hot ORDER BY/filter columns (catalog search/sort_priority/aired_on, player continue-watching/Tier-2, library jobs/demand, themes join, scheduler load-tasks) plus dead SQL migrations targeting the wrong table while AutoMigrate is the real schema authority.
- Resource leaks from missing http.Client timeouts (no ResponseHeaderTimeout on HLS proxy; gateway 15s total truncates streams) and undrained response bodies defeating keep-alive across player/gacha/recs producers and catalog parser fan-outs.
- Background/cron jobs run on deadline-free contexts with no per-tick timeout and (scheduler) no overlap guard or shutdown wait — one hung query stalls the ticker until restart, across recs/scheduler/notifications.
- Unbounded in-memory and metric-cardinality growth with no eviction: watch-together catalog cache + ws gauge labelled by ephemeral room UUID, anidle frozen pool, OG cache, autocache_demand table, co-occurrence table, translation_title metric label.
- String-fragile contracts repeated across services: cache-miss/error-class detection by exact error text (recs, auth unique-violation), captured-todo by exact token (maintenance), HLS rewrite skip by bare substring 'hls-proxy' — all break silently on wording drift.
- Provider-discipline drift: eighteenanime alone skips parsed-host validation, the shared BaseHTTPClient (rate-limit/retry/egress-recording), health-stage seeding, and caching that every other scraper provider follows.
- Per-service observability gaps: no domain metrics on recs/gacha hot paths, analytics flush-failure/source-label blindness, OTel pipeline self-unmonitored + sampling-biased span-metrics nobody consumes.

## Coverage gaps / caveats

- Frontend was explicitly out of scope, so client-side contributors to several findings (player heartbeat cadence, which players persist non-allowlisted combos for hotcombos, WS token handling) were inferred from cited FE files rather than fully reviewed.
- Several severities depend on deployment facts not readable from the repo: actual Postgres max_connections (DB pool exhaustion), host firewall rules (maintenance /api/reports + scheduler triggers external reachability), and ghcr package visibility (k8s imagePullSecrets) — flagged as conditional/medium-confidence by the verifiers.
- No runtime/dynamic verification was performed for any finding — all are static-analysis conclusions; the concurrency races (kodik token, maintenance active-alerts map, anidle RMW) and goroutine leaks were not reproduced under load.
- Production secret hygiene (whether docker/.env actually carries strong values in the live deployment) could not be confirmed beyond what is committed; the report assumes the committed defaults are what runs.
- Quantitative performance claims (e.g. ~48 heartbeats/episode, per-tick HGETALL load, full-catalog ensemble cost) were derived from code + memory notes, not measured against production traffic, so prioritization of perf items relative to each other is approximate.
- The k8s/helm trees may be an abandoned POC rather than a supported deploy target; the report treats them as live infra because they are committed and unmarked, but their actual deployment status was not confirmable from the repo.
- Some refuted/adjusted sub-claims (e.g. proxy.go cited as an SSRF reference, themes guest-UUID 500 reachability, prometheus k8s SD job existence) reveal that cross-file reference accuracy in the source findings was imperfect; a few low/info items may carry residual citation drift not independently re-verified here.
- Sidecar internals (Camoufox/Playwright behavior, anacrolix/torrent close semantics) were verified against pinned versions in the findings but not re-audited at the synthesis layer.

#### Provenance HMAC (the proxy's signed-URL auth boundary) silently falls back to a public hardcoded default secret
- **unit:** cross-security · **category:** security · **effort (fib):** 3 · **verdict:** confirmed
- **location:** `libs/videoutils/provenance.go:52-63 (loadProvenanceSecret default []byte("animeenigma-hls-provenance-default")); gate at libs/videoutils/proxy.go:565-568`
- **evidence:** provenance.go:52-63: loadProvenanceSecret tries STREAM_TOKEN_SECRET then JWT_SECRET, and at :60 falls back to `provenanceSecret = []byte("animeenigma-hls-provenance-default")` — a build-time constant in the public source tree. This key is used by provenanceMAC (:69) and thus validProvenanceToken (:102-112). The gate at proxy.go:565-568 grants access to an otherwise-disallowed host iff `validProvenanceToken(...)` returns true, so a known/forgeable secret lets an attacker mint tokens for arbitrary hosts. No SSRF guard (confirmed above), so a forged token authorizes fetches to 127.0.0.1, internal Docker hostnames, 169.254.169.254. Streaming requires the secret to boot (mitigating streaming-only paths), but videoutils is shared (catalog signs via SignStreamURL at :95; gacha/scraper import it); any consumer without the env var inherits the public default. The package header comment (:20-22) calls the token a graceful-degradation provenance marker, but the actual gate semantics make a valid token GRANT access — the finding correctly identifies the mismatch.
