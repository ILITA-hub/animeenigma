### Task 1: Fetch-first analytics transport with tokenized-path fallback

**Why (root cause):** EasyPrivacy-class `$ping` rules block *beacon-type* requests only. All three analytics clients try `navigator.sendBeacon` first and treat `true` as delivered — but blockers cancel beacons *after* queueing, so `true` is a lie; the fetch fallback that carries `markBlockedFromError` never runs, `blocked` never flips, and the masked (tokenized) alias is never used. Fetch is not matched by `$ping`, and when a blocker matches fetch too (`$xmlhttprequest`), the rejection is observable → flip → masked retry. `fetch(keepalive)` survives pagehide (see `src/utils/authBeacon.ts` header comment), so sendBeacon buys nothing but the blind spot.

**Files:**
- Modify: `frontend/web/src/utils/analyticsTransport.ts` (add `maskedEndpointFor`, `shipAnalyticsPayload`)
- Modify: `frontend/web/src/analytics/transport.ts` (Transport.send → shipper)
- Modify: `frontend/web/src/utils/playerTelemetry.ts:136-178` (flush → shipper)
- Modify: `frontend/web/src/utils/feErrorLog.ts:219-264` (flush → shipper)
- Test: `frontend/web/src/utils/__tests__/analyticsTransport.spec.ts`
- Test: `frontend/web/src/analytics/__tests__/transport.spec.ts`
- Test: `frontend/web/src/utils/playerTelemetry.spec.ts`, `frontend/web/src/utils/__tests__/feErrorLog.spec.ts` (beacon → fetch assertions)

**Interfaces:**
- Produces: `shipAnalyticsPayload(leaf: AnalyticsLeaf, payload: string): void` — resolves endpoint via `analyticsEndpoint(leaf)`, POSTs text/plain fetch keepalive credentials-include, on rejection runs `markBlockedFromError(err)`; if it flips, retries the same payload once at `maskedOverrideFor(leaf)`. Never throws.
- Produces: `maskedEndpointFor(leaf: AnalyticsLeaf): string | null` — masked URL whenever `maskedBase` is known, independent of `blocked` (internal reuse; `maskedOverrideFor` becomes `blocked ? maskedEndpointFor(leaf) : null`).
- Consumes: existing `analyticsEndpoint`, `markBlockedFromError`, `maskedOverrideFor` semantics (unchanged for callers).
