# Дизайн: редизайн `/schedule` в календарь

**Дата:** 2026-06-08
**Статус:** утверждён (интерактивный прототип одобрен пользователем)
**Референс-прототип:** [`2026-06-08-schedule-calendar-redesign.prototype.html`](./2026-06-08-schedule-calendar-redesign.prototype.html) — самодостаточный HTML, открывается в браузере; воспроизводит все три вида, модалку, фильтры, мобилку.

## 1. Цель и рамки

Заменить текущий список расписания (`frontend/web/src/views/Schedule.vue`, группировка онгоингов по дню недели) на **календарь** с тремя видами (Месяц / Неделя / Таблица), модалкой дня и фильтрами.

**Это чисто фронтовый редизайн — бэкенд не меняется.** Эндпоинт `GET /api/anime/schedule` (`catalog.GetSchedule`) уже отдаёт всё необходимое.

## 2. Данные (без изменений бэкенда)

`GET /api/anime/schedule` → `[]Anime` (см. `services/catalog/internal/domain/anime.go`). Используемые поля:

| Поле | Назначение |
|------|-----------|
| `id` | ссылка `/anime/{id}` |
| `name`, `name_ru`, `name_en`, `name_jp` | локализованный + оригинальный заголовок (`getLocalizedTitle`) |
| `poster_url` | обложка |
| `next_episode_at` (`*time.Time`) | якорь проекции; аниме без него **пропускается** |
| `episodes_aired` | номер следующей серии = `episodes_aired + 1` |
| `episodes_count` | верхняя граница проекции (не рисуем серии после финала); `0` = неизвестно |
| `score` | колонка «Оценка» в таблице |
| `kind` | фильтр «Тип» (TV / ONA / Фильм / OVA) |
| `genres[]` | фильтр «Жанр» |

Статус пользовательского списка для гибрид-сортировки берётся из **`useWatchlistStore`** (`watchlistMap`: `anime_id → status`), который уже есть. Гость → сортировка по времени.

### Проекция еженедельных выходов (клиент)

Онгоинги выходят раз в неделю. Для заполнения видимого окна (сетка месяца / неделя) строим вхождения:

- **Якорь** аниме = ближайшая дата на/после «сегодня», у которой день недели совпадает с днём недели `next_episode_at`. Номер серии на якоре = `episodes_aired + 1`.
- Для любой даты `D` в окне, где день недели совпадает: `weeks = round((D − anchor) / 7дней)`, `ep = episodes_aired + 1 + weeks`.
- Включаем вхождение, если `ep ≥ 1` **и** (`episodes_count ≤ 0` **или** `ep ≤ episodes_count`).
- Это даёт и прошлые серии месяца (ep < next), и будущие — месяц заполняется целиком.

Время выхода берётся из `next_episode_at` (час:минуты), отображается как `HH:MM` (по проекту — МСК, `Europe/Moscow`, как в текущем `formatTime`).

## 3. Виды

Переключатель-сегмент **[Месяц | Неделя | Таблица]** в шапке + стрелки `‹ ›` + кнопка «Сегодня». Текущий вид и опорная дата — в URL-query (`?view=month&date=2026-06-08`) для шеринга/перезагрузки. По умолчанию — **Месяц**.

### 3.1 Месяц (по умолчанию)
- Сетка 7×N, **неделя с понедельника**. Дни соседних месяцев («хвосты») приглушены, серии на них не рисуем.
- Ячейка: число дня; до **3 строк** аниме (строка = слева обложка, справа название + «**N** серия · HH:MM»); при >3 — строка «**+N ещё**».
- Какие 3 показывать (**гибрид-сортировка**): у залогиненного сначала тайтлы из «Смотрю»/«В планах» (через `watchlistMap`), затем по времени; у гостя — по времени.
- «Сегодня» — градиентная полоска cyan→violet сверху ячейки.
- Клик по дню (любому с сериями) → **модалка дня**.

### 3.2 Неделя
- 7 колонок текущей недели (Пн–Вс), высокие — показывают **все** серии дня без обрезки, отсортированы по времени.
- Клик по колонке дня → **модалка дня** (единообразно с Месяцем; решение пользователя). Строки внутри — не отдельные ссылки.

### 3.3 Таблица
- **Ограничена текущей неделей** (Пн–Вс), стрелки `‹ ›` листают по неделям; заголовок — «8–14 июня».
- Хронологический список серий, **разделители-заголовки по дням** («Суббота, 13 июня · 5 серий»).
- Колонки: **Аниме** (обложка + оригинальное название + бейдж статуса списка), **Дата · время**, **Серия** (`N / всего`), **Оценка** (★), кнопка «Смотреть».
- Сортировка кликом по заголовку любой колонки (Аниме / Дата·время / Серия / Оценка), направление переключается. Группировка по дням активна только при сортировке по дате.
- Клик по строке / «Смотреть» → `/anime/{id}`.

## 4. Модалка дня (вариант C)

- Заголовок «Суббота, 13 июня» + счётчик серий («5 серий», русская плюрализация), закрытие — ✕ / Esc / клик по затемнению.
- Карточки серий: обложка ~54×76, название + бейдж статуса списка, «Серия N», «HH:MM МСК · ★ оценка», кнопка **«Смотреть»** → `/anime/{id}`.
- Серии в модалке всегда отсортированы по времени.
- Радиус — `2xl` (модалки), скрим с blur.

## 5. Фильтры (общие для всех трёх видов)

Панель над календарём; состояние — в URL-query.

- **🔍 Поиск** по названию (рус + оригинал, подстрока, регистронезависимо).
- **★ Мой список** — тумблер: только `watching`/`planned` (через `watchlistMap`); у гостя скрыт/неактивен.
- **Жанр** ▾ — мультивыбор по `genres`.
- **Тип** ▾ — мультивыбор по `kind` (TV / ONA / Фильм / OVA).
- Активные фильтры — чипами с ✕ + «Сбросить всё» + счётчик «N из M тайтлов».
- (Фильтр «Озвучка» рассматривали и **убрали** по решению пользователя.)

## 6. Визуальный стиль — «Minimal Clean» (C) в палитре Neon Tokyo

Канон: `frontend/web/src/styles/DESIGN-SYSTEM.md`. Связываемся с **семантическими токенами**, без хардкода hex (проходим `frontend/web/scripts/design-system-lint.sh`).

- Плоские ячейки (`bg-white/2.5`, тонкий `border-white/6`, `rounded-xl`), воздух, серии разделены лёгкими линиями (не «плитки»).
- «Сегодня» — градиентная полоска `--brand-cyan → --brand-violet` сверху, без жирного глоу.
- Акцент номера серии — cyan (`--brand-cyan`/`--primary`). cyan и violet — бренд-примитивы (разрешены линтом).
- Заголовки/числа — `--font-display` (Manrope); тело — `--font-sans` (Inter).
- Радиусы: чипы→sm, кнопки/инпуты→md/lg, карточки→xl, модалка→2xl. Отступы карточек `p-4 md:p-6 lg:p-8`.
- Переиспуем ui-примитивы `@/components/ui` (Button, Badge, Input, Select/DropdownMenu, Dialog, Tabs/segmented) перед написанием нового.

### Мобилка
- **Месяц** на узком экране (<640px) — компактные обложки без названий (мини-постеры/точки); тап по дню → модалка. Сетка остаётся календарём.
- В **таблице** на мобиле скрываются колонки «Оценка» и кнопка; строка кликабельна целиком.
- Неделя — колонки сужаются/скроллятся; тап по дню → модалка.

## 7. Файлы

Новый каталог `frontend/web/src/components/schedule/`:

- `MonthView.vue` — сетка 7×N из `DayCell`.

- `TableView.vue` — недельная таблица с сортировкой и группами по дням.
- `DayCell.vue` — ячейка месяца (до 3 строк + «+N ещё»), кликабельна.

Страница:
- `views/Schedule.vue` — шелл: шапка (заголовок, переключатель, навигация, «Сегодня»), панель фильтров, синхронизация состояния с URL-query, рендер активного вида, монтирование `DayModal`.

Дни недели (полные + короткие) уже есть.

## 8. Тестирование

- **Юнит** `useScheduleCalendar`: проекция (корректные номера серий вперёд/назад, обрезка по `episodes_count`, попадание в окно месяца/недели, пропуск без `next_episode_at`); гибрид-сортировка (список юзера → верх, затем время); фильтры (поиск/мой список/жанр/тип, комбинации); группировка таблицы по дням.
- **Компонентные `.spec.ts`** (Vitest): `DayCell` (3 строки + «+N ещё», «сегодня»), `DayModal` (рендер карточек, закрытие ✕/Esc), переключение видов, навигация ‹ ›/Сегодня, сортировка таблицы.
- **i18n parity**: ключи `schedule.*` синхронны в en/ru/ja.
- **In-browser смоук** (DS-NF-06, обязательно): десктоп + мобайл; jsdom не ловит каскад Tailwind v4.
- `bunx tsc --noEmit` + `design-system-lint.sh` зелёные.

## 9. Решения (зафиксированы с пользователем)

1. Вид календаря — **месячная сетка** (не список, не таймлайн).
2. В ячейке — блок «обложка слева + название справа», крупный; до **3 строк**, затем «**+N ещё**».
3. Месяц заполняется **проекцией еженедельных выходов**.
4. **+N ещё** / клик по дню → **модалка дня** (вариант C), не поповер/инлайн-разворот.
5. Клик по дню — **всегда модалка** (и в Месяце, и в Неделе).
6. По умолчанию — **Месяц**.
7. Сортировка свёрнутой ячейки — **гибрид со списком юзера**.
8. Мобилка — **Месяц → компактные постеры**, тап → модалка.
9. Третий вид — **Таблица**, ограничена **неделей**.
10. Фильтры: **Поиск, Мой список, Жанр, Тип** (Озвучка убрана), общие для всех видов.
11. Стиль — **Minimal Clean (C)** в палитре Neon Tokyo.
