# Theater Mode для aePlayer — дизайн

**Дата:** 2026-07-15
**Статус:** согласовано владельцем (песочница `design-prototyping`, сессия `theater-mode`)

## Проблема

Запрос: «расширение плеера, чтобы не только на весь экран» — нужна середина между
окном и фуллскрином.

Театр в кодовой базе **уже написан, но до него нельзя добраться**: кнопку скрыли
2026-06-09 коммитом `94530095` («Theater-mode button hidden per request»), и тест
`PlayerControlBar.spec.ts` это запирает. При этом живы CSS, `useTheaterMode.ts`
(localStorage + Esc), объявленный emit `toggle-theater`, проводка в `Anime.vue`
и строки i18n на en/ru/ja.

Существующая реализация — **клон фуллскрина**, а не середина:

```css
body.theater-mode .navbar-root      { display: none !important }
body.theater-mode .non-player-content { display: none }
.pl--theater { aspect-ratio: auto; height: 100vh }
```

Навбар скрыт, весь контент скрыт, плеер на весь экран — от `F` не отличить.
Это наиболее правдоподобная причина, почему кнопку попросили убрать. Возвращать
её как есть бессмысленно.

Побочный дефект: `/tips` обещает несуществующее — «Кнопка в панели плеера
растягивает видео на всю ширину страницы», нарушая собственное правило страницы
«this page must never oversell».

## Решение

Театр = **полная ширина, навбар и контент под плеером остаются, высота под потолком**.

Потолок: `max-height: calc(100vh - var(--header-offset))`.

Обоснование потолка (проверено замерами в песочнице, монитор 2560×1440):

| Потолок | Видео | Выигрыш к окну | Контролы |
|---|---|---|---|
| `100vh` | 2560×1440 | ×9.5 | **срез 80px** |
| `calc(100vh - var(--header-offset))` | 2418×1360 | ×8.4 | видны |

Навбар фиксированный, страница сдвинута вниз на `--header-offset: 5rem` (80px).
При `max-height:100vh` нижние 80px плеера уходят под сгиб — а там контрол-бар.
Вычитание навбара стоит ≈12% площади и держит управление на экране всегда.
`--header-offset` — единственный источник правды для этого зазора, и
`calc(100vh - var(--header-offset))` уже идиома проекта (`Auth.vue`,
`ProfileSetup.vue`).

Базовый `aspect-ratio: 16/9` **сохраняется**, `max-height` его подрезает. Видео
остаётся `object-contain` — кадр никогда не обрезается, на широких мониторах
появляются боковые полосы. На 21:9 полосы неизбежны при любом потолке (16:9-контент
на ультраширокой матрице) — это свойство контента, не режима.

## Объём

| Файл | Изменение |
|---|---|
| `PlayerControlBar.vue` | кнопка `MonitorPlay`; пропы `theaterActive`, `canTheater`; emit `toggle-theater`; скрыта `<1024px` |
| `AePlayer.vue` | прокинуть `theater`/`canTheater` вниз, emit наверх; `.pl--theater` → `max-height` вместо `height:100vh` |
| `Anime.vue` | снять скрытие `.navbar-root` и `.non-player-content`; обнулить падинги `.player-card`; передать `:can-theater="true"` |
| `PlayerControlBar.spec.ts` | перевернуть запирающий тест |

**Не трогаем:** `useTheaterMode.ts`, локали (`theaterModeEnter`/`theaterModeExit` уже
есть на трёх языках), `/tips`, `WatchTogetherView.vue`, `DownloadsPage.vue`.

### Ловушка мёртвой кнопки

`WatchTogetherView.vue` вешает `@toggle-theater="() => {}"`, а `DownloadsPage.vue`
вообще не слушает событие. Наивно добавленная кнопка появилась бы там и молча
ничего не делала.

Поэтому проп `canTheater` **по умолчанию `false`** — кнопки нет, пока её явно не
включат. `Anime.vue` передаёт `true`; остальные точки монтирования не трогаем.
Опт-ин, а не опт-аут: новая точка монтирования по умолчанию безопасна.

## Поведение

- Состояние — `localStorage.theaterMode` (уже работает).
- <kbd>Esc</kbd> выходит из театра (уже работает).
- Горячей клавиши **нет** — решение владельца, только кнопка. `/tips` менять не нужно.
- Только десктоп (`≥1024px`): на телефоне плеер и так во всю ширину, а фуллскрин
  там и есть основная поверхность просмотра.
- Видео всегда `object-contain`.

## Non-goals

- Не возвращаем скрытие навбара и контента.
- Не добавляем ступени размера и не делаем плеер перетаскиваемым.
- Не трогаем PiP и фуллскрин.
- Не добавляем горячую клавишу.

## Тесты

- `PlayerControlBar.spec.ts`: кнопка рендерится при `canTheater: true`, эмитит
  `toggle-theater`, **отсутствует** при `canTheater: false` (дефолт) — последнее
  защищает Watch Together и Downloads от мёртвой кнопки.
- Существующий тест «does not render a theater-mode button (hidden by request)»
  переворачивается: он фиксировал июньское решение, которое мы сознательно отменяем.
