# Content-Verify: система пробинга контента (озвучка / вшитые субтитры)

- **Дата:** 2026-07-16
- **Источник:** фидбек-репорт `2026-07-16T08-50-00_tNeymik_manual` (реальный пробинг языка озвучки; снять жёсткое raw/dub-деление провайдеров; не показывать «Вшитые»/«DUB» без 95%+ уверенности) + требования владельца в сессии брейншторма.
- **Статус:** дизайн утверждён владельцем 2026-07-16.

## Проблема

Лейблы провайдеров (category=sub|dub, `sub_delivery`) регулярно врут или отсутствуют: «DUB» может оказаться RAW с вшитыми субтитрами, язык озвучки неизвестен, вшитые субтитры не задекларированы. aePlayer строит автовыбор watch combo на этих заявлениях и уверенно показывает пользователю неправду.

## Решение — обзор

Новый сервис **`services/content-verify/` (:8101)** пробивает реальный контент: скачивает фрагменты стрима и определяет язык озвучки (faster-whisper LID) и наличие/язык вшитых субтитров (pixel-эвристика + Tesseract OCR из `tools/subprobe/`). Вердикты хранятся в новой таблице (anime × provider, внутренняя структура юнитов в JSONB) и подмешиваются в capability feed; aePlayer подгружает их динамически и корректирует автовыбор combo до старта воспроизведения.

```
сигналы (визит/просмотр) ──► Redis (score-сигналы, TTL 7д)
онгоинги + топ-100 ────────► membership (запрос в catalog)
                                │
                     виртуальная очередь (score при claim)
                                │  1 юнит / 60 сек, governor-gate L≥1
                                ▼
       резолв стрима (scraper/catalog internal) → ffmpeg 3×30с фрагменты
                                │
              whisper LID (аудио) + subprobe (кадры)
                                ▼
        Postgres: content_verifications (anime × provider, units JSONB)
                                │
     catalog: подмешивает в /capabilities + проксирует /api/anime/{id}/content-verify
                                ▼
                aePlayer: маркеры, RAW-дефолт, коррекция combo
```

## Ключевые решения (утверждены владельцем)

| Вопрос | Решение |
|---|---|
| Определение языка озвучки | **faster-whisper tiny (int8, CPU)** — LID по 3 семплам аудио (до 6 при малой речи); ffprobe-метаданные не доверяются (они и есть проблема) |
| Семплинг | **1 эпизод на юнит: последний вышедший** (fallback — ближайший доступный у юнита); вердикт распространяется на весь тайтл × юнит |
| Архитектура | **Новый сервис** `services/content-verify` :8101 — Go-оркестратор + Python-анализаторы в одном контейнере; своя таблица; catalog подмешивает (паттерн Phase B playability) |
| Unverified UX | **Явный маркер «не проверено» + всё unverified = RAW по умолчанию; DUB — только verified≥95%** |
| Коррекция автовыбора | **Только до старта воспроизведения**; явный выбор юзера неприкосновенен; после первого кадра — только обновление бейджей/панели Source |

## 1. Единица работы и очередь

**Probe unit = (anime × provider × внутренняя единица)**:
- Kodik — озвучка (translation id + заявленный type voice|subtitles);
- scraper-провайдеры (gogoanime/animepahe/allanime-okru/miruro/nineanime) — server × category;
- прочие — дорожка.

**Очередь — вычисляемая (виртуальная), без таблицы состояния.** Каждые 60с воркер ранжирует кандидатов и берёт **один** самый приоритетный pending-юнит. Один юнит за такт даёт естественное вытеснение: горячий тайтл (юзер на странице сейчас) не ждёт, пока отработают 30 юнитов другого тайтла.

- **Membership**: все онгоинги ∪ топ-100 (существующий порядок `sort_priority DESC, score DESC`) ∪ визиты/просмотры за 7 дней.
- **Score(anime)** = `15 × уникальные_визитёры_за_7д` + `10 если ongoing` + `5 если топ-100`. Активный просмотр = тот же сигнал визита (+15/уникальный юзер). Дедуп: юзер/anon-сессия учитывается раз в сутки на тайтл (Redis-set `cv:visit:{anime}:{yyyymmdd}`, TTL 8 дней).
- **Pending-юниты** = дифф: текущая структура провайдеров тайтла (из catalog) минус свежие вердикты. Новая озвучка/сервер появились → юнит сам всплывает в очереди, без ручного инвалидейта.
- **Re-probe**: онгоинг — при выходе нового эпизода (сменился последний эпизод → семпл устарел); завершённые — TTL 30 дней и только пока тайтл в membership. Неудачная проба (стрим не зарезолвился) → `unreachable`, backoff 6ч × 2 до потолка 7д.
- **ae-провайдер не пробится**: язык дорожек первичен из library-инжеста (Phase C, `library_episodes.audio_lang`) — переносится в вердикты как verified с confidence 1.0.

## 2. Тротлинг и governor

Ровно **1 проба одновременно, пауза 60с между пробами**. Жёсткий бюджет юнита **~50с** (резолв → 3 фрагмента → анализ); не уложился — пишем `inconclusive`, идём дальше.

> **⚠ ПЕРЕСМОТРЕТЬ ПОСЛЕ ТЕСТОВ:** бюджет юнита ~50с — стартовая оценка. После первых живых прогонов замерить реальные тайминги (резолв медленных browser-провайдеров, whisper на 3×30с, OCR) и пересмотреть: возможно, бюджет надо поднять (и растянуть такт), разрешить перенос недоеденного юнита на следующий такт, или дифференцировать бюджет по типу провайдера (direct-CDN vs Camoufox).

- Governor: `libs/cache DegradationWatcher` — **L≥1 → воркер не берёт новые юниты** (бинарный gate, паттерн library shedGate).
- Whisper ограничен по потокам (`OMP_NUM_THREADS=2`), контейнер с mem-cap.
- Резолв «нежный»: 1/мин естественно щадит Camoufox-пул; для browser-провайдеров resolve-and-read в один заход (sid живёт ~1 мин, кэшируется мёртвым ~1ч — известная gotcha analytics-пробы). Чтение через streaming hls-proxy с Referer из stream.headers.

## 3. Модель данных

Таблица `content_verifications` (Postgres, БД сервиса): строка на **(anime_id uuid, provider varchar)**, юниты — в **JSONB**:

```json
{ "units": [ {
    "key": {"team": "AniLibria"} | {"server":"hd-2","category":"sub"} | {"track":"..."},
    "episode": 12,
    "status": "verified|inconclusive|unreachable",
    "audio":   {"lang":"ru","confidence":0.98,"verified":true},
    "hardsub": {"present":true,"lang":"en","confidence":0.97,"verified":true},
    "softsubs": [{"lang":"en","kind":"vtt"}],
    "probed_at":"2026-07-16T12:00:00Z",
    "sample":{"fragments":3,"speech_seconds":71}
} ] }
```

**Уверенность (порог 95% — из ТЗ):**
- **Аудио**: whisper LID по 3 фрагментам (25/50/75% длительности; скип OP/ED по маркерам, если стрим их отдаёт) — все фрагменты согласны и средняя вероятность ≥0.95 → `verified`. Мало речи (тишина/музыка) — добираем до 6 фрагментов, иначе `inconclusive`.
- **Hardsub**: tier1 pixel-хиты (bright-pixels-on-edges в нижней центральной полосе, агрегация по high-percentile — субтитры прерывисты) + консистентный OCR-скрипт (кириллица=RU, латиница=EN) в ≥2 окнах → `verified`.
- `verified = confidence ≥ 0.95` — единый порог.

## 4. API и интеграция

- **content-verify**: `GET /internal/verify/verdicts?anime_id=`, `POST /internal/verify/hint` (visit|watching), `GET /internal/verify/queue` (топ-50 расчётной очереди — админка/отладка), `/metrics`. Всё internal — Docker-network-only, gateway не проксирует internal-роуты.
- **catalog**: публичный `GET /api/anime/{uuid}/content-verify` (прокси к verdicts, кэш ~30с — короче интервала опроса FE). **Этот же запрос является сигналом визита** — хинт уходит асинхронно из прокси-хендлера (передаётся user-id/anon-id для дедупа). Плюс best-effort подмешивание verify-блока в `/capabilities`: 3с таймаут, swallow-all → nil, kill-switch env (паттерн Phase B `PLAYABILITY_INDEX_ENABLED`).
- **player** → hint `source=watching` при прогрессе просмотра (паттерн recs recompute-hint, Docker-network-only).

## 5. Поведение aePlayer

- **Unverified = RAW по умолчанию + явный маркер «не проверено»**: без вердикта cap показывается только под RAW, с нейтральным чипом (i18n en/ru/ja). **Под DUB попадают только verified≥95%** источники с реальным языком. Утвердительные бейджи «DUB EN» / «Вшитые (RU)» — только verified.
- Composable `useContentVerify(uuid)`: fetch при монтировании + повторный опрос каждые ~45с, пока страница открыта и воспроизведение не началось (сценарий «юзер читает описание» — данные долетают динамически).
- **Коррекция автовыбора — только до старта воспроизведения** и только если юзер не трогал combo руками. После первого кадра — никаких переключений; новые данные лишь обновляют панель Source/бейджи.
- Hacker-mode: сырые вердикты и confidence — в debugStats (правило «новый datum → debugStats»).

## 6. Наблюдаемость

Метрики: `content_verify_queue_depth`, `content_verify_probes_total{provider,result}`, `content_verify_probe_duration_seconds`, `content_verify_verdict_age_seconds`. Метрики — в пакете сервиса, **не** в libs/metrics (ловушка авторегистрации промо-метрик → перманентные нули-самозванцы из чужих сервисов).

## 7. Известные грабли (учтены в дизайне)

- Новый go.work-модуль ⇒ COPY-строка `libs/…/go.mod` во **все** Dockerfile'ы сервисов; **никогда** `go work sync`.
- Gateway-роут `/api/anime/*` уже идёт в catalog — новый публичный эндпоинт живёт там же, отдельный роут не нужен.
- Compose + k8s-манифесты (`deploy/kustomize/`), `docs/environment-variables.md`.
- Полная пробивка верхушки каталога при 1/мин займёт ~дни — ожидаемо; приоритеты решают, что пробьётся первым.
- Тяжёлые ML-зависимости (python + whisper + tesseract + ffmpeg, ~500MB+) изолированы в контейнере content-verify — корневые сервисы не раздуваются.
