# Design specs

Specs in this directory record active designs or rationale that still explains
the implementation. A spec is not automatically authoritative because it is
newer or more detailed: current reference docs, tests, and code win when they
disagree.

Retire a spec when its feature is removed, its direction is superseded, or all
durable decisions have moved into a current reference. Keep code-linked specs
until those links and the needed rationale are deliberately replaced.

Primary current references include [aePlayer](../../aeplayer-reference.md),
the [scraper framework](../../scraper-framework.md), [scraper health](../../scraper-health-reference.md),
and [Watch Together](../../watch-together-reference.md). See the
[documentation index](../../README.md) for the full map.
